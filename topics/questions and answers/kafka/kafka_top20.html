At Most Once vs At Least Once vs Exactly Once?
Kafka Concept
▾
  • 01
    At Most Once: Commit offset before processing (may lose messages). At Least Once: Process, then commit offset (may duplicate). Exactly Once: Atomic commit of output and offset (used via Kafka Transactions).
02
Auto Commit vs Manual Commit?
Kafka Concept
▾
  • 01
    Auto Commit automatically commits the highest fetched offset periodically in the background. Manual Commit requires explicitly committing the offset after processing, which is highly recommended for at-least-once delivery.
03
Describe a Kafka implementation from your project.
Kafka Concept
▾
  • 01
    We used Kafka to decouple the Order and Inventory services. OrderService publishes an OrderCreatedEvent with acks=all. InventoryService uses @KafkaListener with manual acknowledgment to ensure At Least Once delivery. A DLT was configured for unprocessable payloads.
04
Explain Producer Acknowledgment.
Kafka Concept
▾
  • 01
    Producer Acks determine when the broker responds to the producer: acks=0 (fire and forget), acks=1 (Leader writes locally), and acks=all (Leader waits for all ISRs to write). acks=all provides the highest durability.
05
How Kafka achieves scalability?
Kafka Concept
▾
  • 01
    Kafka achieves scalability through partitioning across multiple brokers, sequential disk I/O, the zero-copy principle bypassing application layer copying, and batching of messages.

How do you integrate Kafka with Spring Boot?
Kafka Concept
▾
  • 01
    Spring Boot integration uses the `spring-kafka` dependency. You use `KafkaTemplate` for producing messages and `@KafkaListener` annotations on methods to consume messages from specific topics.
07
What happens when broker crashes?
Kafka Concept
▾
  • 01
    When a broker crashes, the cluster controller detects the failure and triggers a Leader Election for all affected partitions. It picks the first available Follower in the ISR and promotes it to be the new Leader.
08
What is Consumer Group?
Kafka Concept
▾
  • 01
    A Consumer Group is a collection of consumer instances sharing the same Group ID. They cooperate to consume a topic. Kafka ensures that each partition is read by exactly one consumer instance within the group.
09
What is Consumer Lag?
Kafka Concept
▾
  • 01
    Consumer Lag is the difference between the latest offset written by the producer and the latest offset committed by the consumer. High lag indicates the consumer is too slow.
10
What is DLT?
Kafka Concept
▾
  • 01
    A Dead Letter Topic (DLT) is a separate topic where unprocessable "poison pill" messages are sent. This prevents a failing message from blocking the entire partition indefinitely.
11
What is ISR?
Kafka Concept
▾
  • 01
    ISR stands for In-Sync Replicas. It is the subset of replicas (Followers) that are fully caught up with the Leader partition. Only a broker in the ISR list can be elected as the new Leader if the current one fails.
12
What is Idempotent Producer?
Kafka Concept
▾
  • 01
    An Idempotent Producer prevents duplicate messages during network retries by assigning a unique Producer ID and Sequence Number to each message, ensuring the broker only accepts it once.
13
What is Kafka?
Kafka Concept
▾
  • 01
    Apache Kafka is an open-source distributed event streaming platform used for high-performance data pipelines, streaming analytics, data integration, and mission-critical applications. Unlike traditional message queues, Kafka acts as an append-only distributed commit log.
14
What is Log Compaction?
Kafka Concept
▾
  • 01
    Log Compaction ensures Kafka retains at least the latest known value for each message key rather than deleting based on time, useful for maintaining state (e.g., database row updates).
15
What is Offset?
Kafka Concept
▾
  • 01
    An Offset is a simple integer identifier assigned to each message as it arrives in a partition. It is a logical sequence number used by consumers to track their position in the partition.
16
What is Partition?
Kafka Concept
▾
  • 01
    A Partition is the physical manifestation of a topic. Topics are divided into multiple partitions to allow for scalability. Partitions allow multiple consumers to read from a topic in parallel.
17
What is Rebalancing?
Kafka Concept
▾
  • 01
    Rebalancing is the process of reassigning a topic's partitions among available consumers in a Consumer Group when a consumer joins or leaves. Consumption stops temporarily during a rebalance.
18
What is Replication?
Kafka Concept
▾
  • 01
    Replication ensures high availability by copying partition data to multiple brokers. One broker acts as the Leader for reads/writes, while others act as Followers.
19
What is Topic?
Kafka Concept
▾
  • 01
    A Topic is a logical name (like a database table) where records are published. It acts as a category or feed name to which records are stored and published.
20
Why Kafka over RabbitMQ?
Kafka Concept
▾
  • 01
    RabbitMQ is a message broker. Kafka is an event streaming platform. RabbitMQ drops messages once consumed, while Kafka retains them. RabbitMQ excels at complex routing, whereas Kafka excels at massive horizontal scalability and high throughput.
