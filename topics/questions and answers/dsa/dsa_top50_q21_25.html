Climbing Stairs — How many ways to reach the top?
Easy1D DPFibonacciGateway DP Problem
▾
Pattern — 1D DP (Bottom-Up Fibonacci)
To reach step n, you came from step n-1 (took 1 step) or step n-2 (took 2 steps). So dp[n] = dp[n-1] + dp[n-2]. This is exactly the Fibonacci recurrence. Base cases: dp[1]=1, dp[2]=2. Can be space-optimised to O(1) by keeping only last two values.
P
Problem: You are climbing a staircase with n steps. Each time you can either climb 1 or 2 steps. How many distinct ways can you climb to the top?

Input: n=5 → Output: 8
1
DP Framework — 3 Questions to Ask: (1) What is the subproblem? "How many ways to reach step i?" (2) What is the recurrence? "dp[i] = dp[i-1] + dp[i-2]" — came from one step below or two steps below. (3) What are the base cases? "dp[1]=1 (one way: step 1), dp[2]=2 (1+1 or 2)." Answer these three questions for EVERY DP problem.
2
Top-Down (Memoization) vs Bottom-Up (Tabulation): Top-down: recursive with a cache array. Natural to write, matches the recurrence directly. Bottom-Up: iterative, fills a table from base cases forward. No stack overflow risk. For this problem both are O(n) time and O(n) space — but bottom-up can be reduced to O(1) space.
3
Space Optimisation: Since dp[i] only depends on dp[i-1] and dp[i-2], we don't need the full array. Keep two variables: prev2 and prev1. At each step compute curr = prev1 + prev2, then shift: prev2=prev1, prev1=curr. O(1) space.
4
Generalisation — k Steps Allowed: If you can take 1 to k steps, dp[i] = sum(dp[i-1] + dp[i-2] + ... + dp[i-k]). Maintain a sliding window sum of the last k values. This generalisation is asked at Razorpay.
Time
O(n)
Space
O(1)
Pattern
1D DP
// Bottom-up DP — O(n) time, O(n) space
public int climbStairs(int n) {
    if (n <= 2) return n;
    int[] dp = new int[n + 1];
    dp[1] = 1; dp[2] = 2;
    for (int i = 3; i <= n; i++)
        dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
}

// Space-optimised — O(1) space (best solution)
public int climbStairsOpt(int n) {
    if (n <= 2) return n;
    int prev2 = 1, prev1 = 2;
    for (int i = 3; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

// Top-down memoization
public int climbStairsMemo(int n) {
    int[] memo = new int[n + 1];
    return solve(n, memo);
}
private int solve(int n, int[] memo) {
    if (n <= 2) return n;
    if (memo[n] != 0) return memo[n];
    return memo[n] = solve(n - 1, memo) + solve(n - 2, memo);
}

// Java 8 Stream — for showing stream knowledge:
public int climbStairsStream(int n) {
    int[] fib = {1, 2};
    IntStream.range(3, n + 1).forEach(i -> {
        int next = fib[0] + fib[1];
        fib[0] = fib[1]; fib[1] = next;
    });
    return n == 1 ? 1 : fib[1];
}

Always start DP answers with: "Let dp[i] = the answer for subproblem of size i. The recurrence is dp[i] = dp[i-1] + dp[i-2] because I can arrive at step i from step i-1 or step i-2." This structured approach signals DP mastery to interviewers.

House Robber — Maximum money without robbing adjacent houses
Medium1D DPDecision DPClassic Pattern
▾
Pattern — Include/Exclude Decision DP
At each house i, make a binary decision: rob it (add nums[i] to dp[i-2]) or skip it (take dp[i-1]). Take the max. dp[i] = max(nums[i] + dp[i-2], dp[i-1]). Space-optimise to O(1) with two variables — same trick as climbing stairs.
P
Problem: An array represents money in each house. Adjacent houses have security systems — cannot rob two adjacent houses. Maximise the total amount robbed.

Input: [2,7,9,3,1] → Output: 12 (rob houses 0,2,4: 2+9+1=12)
1
The Decision at Each Step: For house i, you have exactly 2 choices: (1) Rob house i — earn nums[i] plus best from houses 0..i-2 (can't include i-1). (2) Skip house i — take best from houses 0..i-1. Take the maximum: dp[i] = max(nums[i] + dp[i-2], dp[i-1]). This "include/exclude" structure appears in 0/1 knapsack and many string DP problems.
2
Space Optimisation: dp[i] only depends on dp[i-1] and dp[i-2]. Use two variables rob2 (dp[i-2]) and rob1 (dp[i-1]). At each step: curr = max(nums[i] + rob2, rob1), then shift. O(1) space.
3
Follow-Up — House Robber II (Circular, LC 213): Houses in a circle — first and last are adjacent. Solution: run House Robber on nums[0..n-2] (exclude last), run on nums[1..n-1] (exclude first), return max of both. Two passes with the same algorithm.
4
Follow-Up — House Robber III (Binary Tree, LC 337): Houses arranged as a binary tree — can't rob parent and child. Use tree DP: return [robbed, skipped] for each node. Robbed = node.val + left.skipped + right.skipped. Skipped = max(left) + max(right). O(n) with post-order traversal.
Time
O(n)
Space
O(1)
Pattern
Include/Exclude
// Space-optimised O(1) — best solution
public int rob(int[] nums) {
    if (nums.length == 1) return nums[0];
    int rob2 = 0, rob1 = 0;           // dp[i-2], dp[i-1]
    for (int num : nums) {
        int curr = Math.max(num + rob2, rob1);
        rob2 = rob1;
        rob1 = curr;
    }
    return rob1;
}

// With full dp array (clearer, same O(n) time, O(n) space)
public int robDP(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    int[] dp = new int[n];
    dp[0] = nums[0];
    dp[1] = Math.max(nums[0], nums[1]);
    for (int i = 2; i < n; i++)
        dp[i] = Math.max(nums[i] + dp[i - 2], dp[i - 1]);
    return dp[n - 1];
}

// Follow-up: House Robber II (circular array)
public int robII(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    // Try excluding first or last house, take max
    return Math.max(
        robRange(nums, 0, n - 2),   // exclude last
        robRange(nums, 1, n - 1)    // exclude first
    );
}
private int robRange(int[] nums, int lo, int hi) {
    int rob2 = 0, rob1 = 0;
    for (int i = lo; i <= hi; i++) {
        int curr = Math.max(nums[i] + rob2, rob1);
        rob2 = rob1; rob1 = curr;
    }
    return rob1;
}
nums = [2, 7, 9, 3, 1] ────────────────────────────────────────────── rob2=0, rob1=0 num=2: curr=max(2+0,0)=2 rob2=0, rob1=2 num=7: curr=max(7+0,2)=7 rob2=2, rob1=7 num=9: curr=max(9+2,7)=11 rob2=7, rob1=11 num=3: curr=max(3+7,11)=11 rob2=11,rob1=11 num=1: curr=max(1+11,11)=12 rob2=11,rob1=12 ────────────────────────────────────────────── Answer: 12 ✓ (rob houses 0,2,4: 2+9+1=12)
The decision pattern "dp[i] = max(include_i, exclude_i)" is the backbone of many DP problems. House Robber is the simplest — master it, then recognise it in harder variants like Weighted Job Scheduling and Maximum Non-Adjacent Sum.

Coin Change — Minimum coins to make a target amount
MediumUnbounded Knapsack1D DPMust Know
▾
Pattern — Unbounded Knapsack DP
For each amount from 1 to target, try every coin: dp[amount] = min(dp[amount], dp[amount - coin] + 1). Each coin can be used unlimited times (unbounded). Initialize dp[0]=0, all others = infinity. If dp[target] is still infinity at end → impossible.
P
Problem: Given coin denominations and an amount, find minimum number of coins to make that amount. Return -1 if impossible.

Input: coins=[1,5,6,9], amount=11 → Output: 2 (5+6 or 2+9)
1
Why Greedy Fails: Greedy (always take largest coin ≤ remaining) fails for some coin sets. Example: coins=[1,3,4], amount=6. Greedy: 4+1+1=3 coins. Optimal: 3+3=2 coins. DP considers ALL possibilities and picks the minimum — that's the key advantage over greedy.
2
Recurrence — Bottom-Up: dp[i] = minimum coins to make amount i. For each amount i (1 to target), for each coin c: if i >= c and dp[i-c] != INF, then dp[i] = min(dp[i], dp[i-c] + 1). This says: "if I use coin c, the remaining amount is i-c, and I need one more coin."
3
Unbounded vs 0/1 Knapsack: In unbounded knapsack (this problem), each item can be used multiple times. The inner loop iterates over amounts bottom-up, which naturally allows the same coin to be reused. In 0/1 knapsack (each item once), the loop order must be top-down to prevent reuse.
4
Follow-Up — Coin Change II (Count ways, LC 518): Count number of COMBINATIONS (not permutations) to reach amount. dp[i] += dp[i-coin] for each coin (for ways, not min). Order matters for permutations vs combinations — process coins in outer loop for combinations.
Time
O(n×m)
Space
O(n)
Pattern
Unbound Knapsack
// Bottom-up DP — O(amount × coins) time, O(amount) space
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1);       // fill with "infinity"
    dp[0] = 0;                           // base: 0 coins for amount 0

    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i) {
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
            }
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}

// Top-down memoization
public int coinChangeMemo(int[] coins, int amount) {
    int[] memo = new int[amount + 1];
    Arrays.fill(memo, -1);
    memo[0] = 0;
    return solve(coins, amount, memo);
}
private int solve(int[] coins, int rem, int[] memo) {
    if (rem < 0) return -1;
    if (memo[rem] != -1) return memo[rem];
    int min = Integer.MAX_VALUE;
    for (int c : coins) {
        int res = solve(coins, rem - c, memo);
        if (res >= 0) min = Math.min(min, res + 1);
    }
    return memo[rem] = (min == Integer.MAX_VALUE) ? -1 : min;
}

// Follow-up: Coin Change II (number of combinations)
public int change(int amount, int[] coins) {
    int[] dp = new int[amount + 1];
    dp[0] = 1;                           // 1 way to make 0: use no coins
    for (int coin : coins)               // outer = coins (for combinations)
        for (int i = coin; i <= amount; i++)
            dp[i] += dp[i - coin];
    return dp[amount];
}
coins=[1,5,6,9], amount=11 dp[0]=0, all others = 12 (infinity) ────────────────────────────────────────────────────── i=1: coin=1: dp[1]=min(12,dp[0]+1)=1 i=5: coin=1: dp[5]=min(12,dp[4]+1)=5 (via 1s) coin=5: dp[5]=min(5,dp[0]+1)=1 i=6: coin=1: dp[6]=min(12,dp[5]+1)=2 coin=5: dp[6]=min(2,dp[1]+1)=2 coin=6: dp[6]=min(2,dp[0]+1)=1 i=9: coin=9: dp[9]=min(...,dp[0]+1)=1 i=10: coin=1: dp[10]=min(12,dp[9]+1)=2 i=11: coin=1: dp[11]=min(12,dp[10]+1)=3 coin=5: dp[11]=min(3,dp[6]+1)=2 coin=6: dp[11]=min(2,dp[5]+1)=2 ────────────────────────────────────────────────────── Answer: dp[11] = 2 ✓ (5+6 = 11, 2 coins)
Using Integer.MAX_VALUE as infinity and then doing +1 causes integer overflow. Always use amount + 1 or a large-but-safe sentinel like 99999. This is a real bug that trips candidates in live coding.

Longest Common Subsequence — Classic 2D DP
Medium2D DPString DPFoundation Problem
▾
Pattern — 2D DP on Two Sequences
dp[i][j] = LCS length of text1[0..i-1] and text2[0..j-1]. If characters match: dp[i][j] = dp[i-1][j-1] + 1. If not: dp[i][j] = max(dp[i-1][j], dp[i][j-1]). Build the table bottom-up. Answer is at dp[m][n].
P
Problem: Given two strings, return the length of their longest common subsequence. A subsequence is formed by deleting characters without changing relative order.

Input: text1="abcde", text2="ace" → Output: 3 ("ace")
1
The 2D DP Table: dp[i][j] answers: "what is the LCS of the first i characters of text1 and first j characters of text2?" Base cases: dp[0][j]=0 (empty text1 → LCS=0) and dp[i][0]=0. The recurrence covers all cases: if chars match, extend LCS by 1 from the diagonal. If not, take the best from either dropping a char from text1 or text2.
2
Recurrence Explained: If text1[i-1]==text2[j-1]: this character is part of LCS → dp[i][j] = dp[i-1][j-1] + 1. Else: either text1[i-1] is not in LCS (skip it → dp[i-1][j]) or text2[j-1] is not in LCS (skip it → dp[i][j-1]). Take max.
3
Space Optimisation: Each row only depends on the previous row. Use two rows: current and previous. Or a single array with careful update order. Reduces from O(m×n) to O(min(m,n)) space.
4
Related Problems Using Same Table: Edit Distance (LC 72): 3 operations instead of 1. Longest Common Substring: dp[i][j] = dp[i-1][j-1]+1 if match, else 0 (no carry). Shortest Common Supersequence: m+n-LCS. Knowing LCS deeply unlocks an entire family of string DP problems.
Time
O(m×n)
Space
O(m×n)
Pattern
2D DP
// Bottom-up 2D DP — O(m×n) time and space
public int longestCommonSubsequence(String text1, String text2) {
    int m = text1.length(), n = text2.length();
    int[][] dp = new int[m + 1][n + 1];  // +1 for empty string base case

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (text1.charAt(i - 1) == text2.charAt(j - 1))
                dp[i][j] = dp[i - 1][j - 1] + 1; // match: extend diagonal
            else
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]); // skip one
        }
    }
    return dp[m][n];
}

// Space-optimised: O(n) space using 1D rolling array
public int lcsOptimised(String text1, String text2) {
    int m = text1.length(), n = text2.length();
    int[] dp = new int[n + 1];
    for (int i = 1; i <= m; i++) {
        int prev = 0;                        // represents dp[i-1][j-1]
        for (int j = 1; j <= n; j++) {
            int temp = dp[j];               // save before overwrite
            if (text1.charAt(i-1) == text2.charAt(j-1))
                dp[j] = prev + 1;
            else
                dp[j] = Math.max(dp[j], dp[j-1]);
            prev = temp;
        }
    }
    return dp[n];
}

// Edit Distance (LC 72) — same table, 3 operations
public int minDistance(String w1, String w2) {
    int m = w1.length(), n = w2.length();
    int[][] dp = new int[m+1][n+1];
    for (int i=0;i<=m;i++) dp[i][0]=i;    // delete all of w1
    for (int j=0;j<=n;j++) dp[0][j]=j;    // insert all of w2
    for (int i=1;i<=m;i++)
        for (int j=1;j<=n;j++)
            dp[i][j] = w1.charAt(i-1)==w2.charAt(j-1) ? dp[i-1][j-1] :
                1 + Math.min(dp[i-1][j-1], Math.min(dp[i-1][j],dp[i][j-1]));
    return dp[m][n];
}

Build the 2D table on paper during the interview — it shows the interviewer your thinking process clearly. For each cell explain: "If characters match, I extend the diagonal. Otherwise I take the max from above or left — representing skipping a character from either string."

Word Break — Can the string be segmented using dictionary words?
Medium1D DPString DPJuspay / Groww
▾
Pattern — 1D DP with String Suffix Check
dp[i] = true if s[0..i-1] can be segmented using words from the dictionary. For each position i, check all previous positions j: if dp[j] is true AND s[j..i-1] is a word in the dictionary → dp[i] = true. Use a HashSet for O(1) word lookup.
P
Problem: Given string s and dictionary wordDict, return true if s can be segmented into a space-separated sequence of dictionary words.

Input: s="leetcode", wordDict=["leet","code"] → Output: true ("leet"+"code")
1
DP Definition: dp[i] = "can the first i characters of s be segmented?" Base case: dp[0] = true (empty string is always valid). For each i from 1 to n, check all j from 0 to i-1: if dp[j]==true and s.substring(j,i) is in the dictionary → set dp[i]=true and break.
2
Use HashSet for O(1) Lookup: Convert wordDict list to a HashSet. String contains() on a list is O(n×m) per lookup. HashSet.contains() is O(m) (just hashing the substring). This reduces overall complexity from O(n³) to O(n²) effectively.
3
Optimisation — Check Only Valid Word Lengths: Instead of checking all j from 0 to i, only check values of j such that (i-j) equals a word length in the dictionary. Collect all distinct word lengths — this prunes unnecessary checks when dictionary words are short but string is long.
4
Follow-Up — Word Break II (Return All Sentences, LC 140): Instead of boolean dp, store all possible previous break points. Use backtracking + memoization. O(n² × output size) — exponential in worst case, but memoization prunes dramatically.
Time
O(n²)
Space
O(n)
Pattern
1D DP + Set
public boolean wordBreak(String s, List<String> wordDict) {
    Set<String> dict = new HashSet<>(wordDict);  // O(1) lookup
    int n = s.length();
    boolean[] dp = new boolean[n + 1];
    dp[0] = true;                                  // empty string always valid

    for (int i = 1; i <= n; i++) {
        for (int j = 0; j < i; j++) {
            if (dp[j] && dict.contains(s.substring(j, i))) {
                dp[i] = true;
                break;                           // found one valid split, stop
            }
        }
    }
    return dp[n];
}

// Optimised: only check valid word-length substrings
public boolean wordBreakOpt(String s, List<String> wordDict) {
    Set<String> dict = new HashSet<>(wordDict);
    Set<Integer> lengths = wordDict.stream()
        .map(String::length).collect(Collectors.toSet()); // Java 8
    int n = s.length();
    boolean[] dp = new boolean[n + 1];
    dp[0] = true;
    for (int i = 1; i <= n; i++)
        for (int len : lengths)
            if (i >= len && dp[i - len] && dict.contains(s.substring(i - len, i)))
                { dp[i] = true; break; }
    return dp[n];
}

// Follow-up: Word Break II — return all possible sentences
public List<String> wordBreakII(String s, List<String> wordDict) {
    Set<String> dict = new HashSet<>(wordDict);
    Map<Integer, List<String>> memo = new HashMap<>();
    return dfs(s, 0, dict, memo);
}
private List<String> dfs(String s, int start, Set<String> dict,
                          Map<Integer,List<String>> memo) {
    if (memo.containsKey(start)) return memo.get(start);
    List<String> res = new ArrayList<>();
    if (start == s.length()) { res.add(""); return res; }
    for (int end = start + 1; end <= s.length(); end++) {
        String word = s.substring(start, end);
        if (dict.contains(word))
            for (String rest : dfs(s, end, dict, memo))
                res.add(word + (rest.isEmpty() ? "" : " " + rest));
    }
    memo.put(start, res);
    return res;
}

State the DP clearly: "dp[i] means the first i characters can be broken into dictionary words. I check for each split point j: if dp[j] is true AND the suffix s[j..i] is a word, then dp[i] is true." This precise verbal definition is what earns full marks.
