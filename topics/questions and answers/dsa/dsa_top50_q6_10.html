3Sum — Find all unique triplets that sum to zero
MediumTwo PointersSortingTop 10 Most Asked
▾
Pattern — Sort + Two Pointers (reduce to 2Sum)
Fix one element at a time (the outer loop). For the remaining elements, use Two Pointers to find pairs that sum to -nums[i]. Sorting enables Two Pointers AND makes duplicate skipping simple — adjacent duplicates can be skipped with a single comparison.
P
Problem: Given integer array nums, return all unique triplets [a, b, c] such that a + b + c = 0. No duplicate triplets allowed.

Input: nums = [-1, 0, 1, 2, -1, -4]
Output: [[-1,-1,2], [-1,0,1]]
1
Sort First: Sorting is the key enabler. It lets us use Two Pointers (monotone property — moving right increases, moving left decreases), AND it puts duplicates adjacent so we can skip them with a simple check: if (i > 0 && nums[i] == nums[i-1]) continue;
2
Fix + 2Sum Structure: For each index i, the problem becomes: find two numbers in nums[i+1..n-1] that sum to -nums[i]. This is exactly 2Sum with Two Pointers on a sorted array. L = i+1, R = n-1. If sum too small → L++. If too large → R--. If equal → record triplet.
3
Duplicate Skipping — Three Places: (a) Skip outer loop duplicates: if (i>0 && nums[i]==nums[i-1]) continue. (b) After finding a triplet, skip duplicate L values: while (L<R && nums[L]==nums[L-1]) L++. (c) Skip duplicate R values: while (L<R && nums[R]==nums[R+1]) R--. Miss any one → duplicates in output.
4
Early Exit: Since array is sorted, if nums[i] > 0, no triplet summing to 0 is possible (all remaining values are ≥ nums[i] > 0). Break early for O(n) best case.
Time
O(n²)
Space
O(1)*
Pattern
Sort+2ptr
public List<List<Integer>> threeSum(int[] nums) {
    List<List<Integer>> result = new ArrayList<>();
    Arrays.sort(nums);                           // MUST sort first

    for (int i = 0; i < nums.length - 2; i++) {
        if (nums[i] > 0) break;                 // early exit
        if (i > 0 && nums[i] == nums[i - 1]) continue; // skip outer dups

        int L = i + 1, R = nums.length - 1;
        while (L < R) {
            int sum = nums[i] + nums[L] + nums[R];
            if (sum == 0) {
                result.add(Arrays.asList(nums[i], nums[L], nums[R]));
                while (L < R && nums[L] == nums[L + 1]) L++; // skip dup L
                while (L < R && nums[R] == nums[R - 1]) R--; // skip dup R
                L++; R--;
            } else if (sum < 0) L++;
            else R--;
        }
    }
    return result;
}

// Java 8 stream variant — elegant but harder to skip duplicates cleanly:
public List<List<Integer>> threeSumStream(int[] nums) {
    Arrays.sort(nums);
    return IntStream.range(0, nums.length - 2)
        .filter(i -> i == 0 || nums[i] != nums[i - 1])
        .boxed()
        .flatMap(i -> {
            List<List<Integer>> pairs = new ArrayList<>();
            int L = i + 1, R = nums.length - 1;
            while (L < R) {
                int s = nums[i] + nums[L] + nums[R];
                if (s == 0) { pairs.add(Arrays.asList(nums[i],nums[L++],nums[R--])); }
                else if (s < 0) L++; else R--;
            }
            return pairs.stream();
        })
        .distinct()
        .collect(Collectors.toList());
}

Forgetting to skip duplicates is the #1 bug in 3Sum. There are 3 places to skip: after outer loop increment, after L moves past matched pair, after R moves. Missing any one gives duplicate triplets.

Maximum Subarray — Kadane's Algorithm
MediumKadane'sDynamic ProgrammingClassic Must Know
▾
Pattern — Kadane's: Local Max vs Global Max
At each index, decide: should I extend the current subarray, or start fresh from this element? If the running sum becomes negative, it can only hurt future subarrays — reset to 0 and start fresh. Track the maximum sum seen at any point.
P
Problem: Given integer array nums, find the contiguous subarray with the largest sum and return its sum.

Input: nums = [-2,1,-3,4,-1,2,1,-5,4]
Output: 6 — subarray [4,-1,2,1] has sum = 6
1
The Core Insight: currentSum = max(nums[i], currentSum + nums[i]). This single line captures the decision: either start a new subarray at i (take just nums[i]), or extend the existing subarray (currentSum + nums[i]). Take whichever is larger.
2
Equivalent Formulation: If currentSum drops below 0, reset it to 0. A negative prefix can only reduce any future subarray's sum — throw it away. This is the classic "drain and reset" version that's slightly easier to explain verbally.
3
All Negative Array: If all numbers are negative, the maximum subarray is the single largest (least negative) element. The max(nums[i], currentSum + nums[i]) formulation handles this correctly — it never goes below the largest single element. Verify mentally: [-3, -1, -2] → answer = -1.
4
Follow-Up — Return the Subarray Itself: Track start, end, and tempStart indices. When we "start fresh" at index i (because nums[i] alone is better), update tempStart = i. When maxSum is updated, set start = tempStart, end = i.
Time
O(n)
Space
O(1)
Pattern
Kadane's DP
// Kadane's Algorithm — O(n) time, O(1) space
public int maxSubArray(int[] nums) {
    int maxSum = nums[0];
    int currentSum = nums[0];

    for (int i = 1; i < nums.length; i++) {
        currentSum = Math.max(nums[i], currentSum + nums[i]);
        maxSum = Math.max(maxSum, currentSum);
    }
    return maxSum;
}

// Java 8 — using IntStream.reduce elegantly
public int maxSubArrayStream(int[] nums) {
    int[] maxSum = {nums[0]}, cur = {nums[0]};
    IntStream.range(1, nums.length).forEach(i -> {
        cur[0] = Math.max(nums[i], cur[0] + nums[i]);
        maxSum[0] = Math.max(maxSum[0], cur[0]);
    });
    return maxSum[0];
}

// Follow-up: Return the actual subarray [start, end]
public int[] maxSubArrayWithIndices(int[] nums) {
    int maxSum = nums[0], curSum = nums[0];
    int start = 0, end = 0, tempStart = 0;

    for (int i = 1; i < nums.length; i++) {
        if (nums[i] > curSum + nums[i]) { // start fresh
            curSum = nums[i];
            tempStart = i;
        } else { curSum += nums[i]; }

        if (curSum > maxSum) {
            maxSum = curSum;
            start = tempStart;
            end = i;
        }
    }
    return new int[]{start, end}; // indices of max subarray
}

Verbalize the key decision: "At each element I ask: is it better to start fresh here or extend the previous sum? If extending gives a smaller result, I start fresh. I track the global best along the way."

Merge Intervals — Merge all overlapping intervals
MediumSortingIntervalsGoogle / Razorpay
▾
Pattern — Sort by Start + Greedy Merge
After sorting by start time, overlapping intervals are always adjacent. Walk through sorted intervals — if current interval overlaps with last in result (current.start <= last.end), extend the end to max(last.end, current.end). Otherwise add as new interval. Sorting makes this O(n log n) total, O(n) for merge step.
P
Problem: Given array of intervals where intervals[i] = [start, end], merge all overlapping intervals and return the resulting array.

Input: [[1,3],[2,6],[8,10],[15,18]]
Output: [[1,6],[8,10],[15,18]] — [1,3] and [2,6] overlap → [1,6]
1
Why Sort by Start? Unsorted intervals: you'd need to compare every pair O(n²). After sorting by start time, any overlapping interval is necessarily adjacent to its neighbor in the sorted array. Proof: if interval A starts before B, and B overlaps A, then B.start <= A.end. If C overlaps with the merged [A,B] result but not with B alone, C must start between B.start and merged.end — still adjacent after sorting.
2
Overlap Condition: Two intervals [a,b] and [c,d] overlap if and only if c <= b (after sorting, a <= c is guaranteed). When they overlap, merged interval is [a, max(b,d)]. Use max(b,d) not just d — consider [1,10],[2,3] where inner interval is fully contained.
3
Result Building: Keep a result list. For each interval, either extend the last interval in result (if overlap) or add new entry. At the end, convert list to int[][] array.
4
Follow-Up — Insert Interval: Insert a new interval into an already-sorted, non-overlapping list. Three phases: (1) Add all intervals that end before newInterval.start. (2) Merge all overlapping intervals with newInterval. (3) Add remaining. O(n) time, no sort needed.
Time
O(n log n)
Space
O(n)
Pattern
Sort+Merge
public int[][] merge(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]); // sort by start
    LinkedList<int[]> result = new LinkedList<>();

    for (int[] interval : intervals) {
        // No overlap: add as new interval
        if (result.isEmpty() || result.getLast()[1] < interval[0]) {
            result.add(interval);
        } else {
            // Overlap: extend end if needed
            result.getLast()[1] = Math.max(result.getLast()[1], interval[1]);
        }
    }
    return result.toArray(new int[0][]);
}

// Java 8 Stream version:
public int[][] mergeStream(int[][] intervals) {
    LinkedList<int[]> result = new LinkedList<>();
    Arrays.stream(intervals)
          .sorted(Comparator.comparingInt(a -> a[0]))
          .forEach(iv -> {
              if (result.isEmpty() || result.getLast()[1] < iv[0])
                  result.add(iv);
              else
                  result.getLast()[1] = Math.max(result.getLast()[1], iv[1]);
          });
    return result.toArray(new int[0][]);
}

// Follow-up: Insert Interval into sorted non-overlapping list
public int[][] insert(int[][] intervals, int[] newInterval) {
    List<int[]> res = new ArrayList<>();
    int i = 0, n = intervals.length;
    while (i < n && intervals[i][1] < newInterval[0]) res.add(intervals[i++]);
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i++][1]);
    }
    res.add(newInterval);
    while (i < n) res.add(intervals[i++]);
    return res.toArray(new int[0][]);
}

Key insight to say: "After sorting by start, overlapping intervals are always adjacent — so I only need to compare each interval with the LAST merged interval, not all previous ones. This is the O(n) merge step after O(n log n) sort."

Find the Duplicate Number — No extra space, no modifying array
MediumFloyd's CycleTwo PointersTricky — PhonePe
▾
Pattern — Floyd's Cycle Detection (Tortoise & Hare)
Treat the array as a linked list where each value is the "next" pointer: index i points to index nums[i]. Since there is a duplicate, two indices point to the same value — creating a cycle. Floyd's algorithm finds the cycle entry point, which is the duplicate number. O(n) time, O(1) space, no modification to array.
P
Problem: Given array nums of n+1 integers where each integer is in [1, n], exactly one value is repeated. Find the duplicate without modifying the array and using only O(1) extra space.

Input: nums = [1,3,4,2,2]
Output: 2
1
Key Mapping — Array as Linked List: Think of index i as a node, and nums[i] as its "next" pointer. Start at index 0. Since values are in [1,n] and n+1 elements exist, one value appears twice — meaning two different indices point to the same "next". This creates a cycle. The cycle's entry point is the duplicate value.
2
Phase 1 — Detect Cycle (Find Intersection): Slow pointer moves one step at a time (slow = nums[slow]). Fast pointer moves two steps (fast = nums[nums[fast]]). They meet inside the cycle. This is guaranteed since a cycle exists.
3
Phase 2 — Find Cycle Entry (The Duplicate): Reset one pointer to index 0. Move both pointers one step at a time. Where they meet = the cycle entry = the duplicate number. Mathematical proof: distance from head to cycle entry equals distance from intersection point to cycle entry (within the cycle).
4
Why Not Simpler Approaches? HashSet: O(n) space — violates constraint. Sort + check adjacent: modifies array — violates constraint. Binary search on value: O(n log n) but valid. Sum formula: only works if exactly one duplicate. Floyd's: O(n) time, O(1) space, handles all cases cleanly.
Time
O(n)
Space
O(1)
Pattern
Floyd's Cycle
public int findDuplicate(int[] nums) {
    // Phase 1: Find intersection point inside cycle
    int slow = nums[0];
    int fast = nums[0];
    do {
        slow = nums[slow];           // move 1 step
        fast = nums[nums[fast]];     // move 2 steps
    } while (slow != fast);

    // Phase 2: Find cycle entry (= duplicate number)
    slow = nums[0];                 // reset slow to start
    while (slow != fast) {
        slow = nums[slow];
        fast = nums[fast];           // BOTH move 1 step now
    }
    return slow;                     // meeting point = duplicate
}

// Alternative: Binary Search on value range O(n log n), O(1)
public int findDuplicateBinarySearch(int[] nums) {
    int lo = 1, hi = nums.length - 1;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        // Count how many nums <= mid
        long count = Arrays.stream(nums).filter(n -> n <= mid).count();
        // If count > mid, duplicate is in [lo, mid] (pigeonhole)
        if (count > mid) hi = mid;
        else lo = mid + 1;
    }
    return lo;
}

// Simple but O(n) space — mention this as suboptimal approach first:
public int findDuplicateHashSet(int[] nums) {
    Set<Integer> seen = new HashSet<>();
    return Arrays.stream(nums)
        .filter(n -> !seen.add(n))
        .findFirst().orElse(-1);
}

This problem requires understanding Floyd's cycle detection at a deep level. In interviews, first offer the HashSet O(n) space solution, then say "but with the O(1) space constraint, I'd use Floyd's cycle detection — treating array indices as a linked list." This shows progressive thinking.

Rotate Array — Rotate right by k steps in O(1) space
MediumArrayReverse TrickClean O(1) Solution
▾
Pattern — Three Reverses (The Elegant Trick)
To rotate right by k: (1) Reverse entire array. (2) Reverse first k elements. (3) Reverse remaining n-k elements. This works because rotating right by k places the last k elements at the front — and three reverses achieve exactly this in O(n) time, O(1) space.
P
Problem: Given an integer array nums, rotate the array to the right by k steps in-place.

Input: nums = [1,2,3,4,5,6,7], k = 3
Output: [5,6,7,1,2,3,4] — last 3 elements move to front
1
Critical: k = k % n: If k ≥ n, rotating n times is a full cycle (no change). Use k = k % nums.length first. If k = 0 after mod, array is already correct — can return early.
2
The Three-Reverse Trick Explained: Original: [1,2,3,4,5,6,7]. After rotating right 3: [5,6,7,1,2,3,4]. Notice: the last k=3 elements [5,6,7] come first, then [1,2,3,4]. Three reverses achieve this: Reverse all → [7,6,5,4,3,2,1]. Reverse first k=3 → [5,6,7,4,3,2,1]. Reverse last n-k=4 → [5,6,7,1,2,3,4]. ✓
3
Why It Works (Intuition): Reversing the whole array "breaks" the original order. Then reversing each half "fixes" the order within each half. The net effect is exactly a rotation. Mathematical proof via string rotation equivalences — but the pattern is the key takeaway.
4
Brute Force — O(n) time, O(n) space: Copy last k elements to a temp array, shift remaining n-k elements right by k, copy temp back to front. Simpler but uses O(k) extra space. The three-reverse trick is O(n) time, O(1) space — the optimal solution.
Time
O(n)
Space
O(1)
Pattern
3 Reverses
public void rotate(int[] nums, int k) {
    k = k % nums.length;          // handle k >= n
    if (k == 0) return;            // no rotation needed

    reverse(nums, 0, nums.length - 1); // step 1: reverse all
    reverse(nums, 0, k - 1);           // step 2: reverse first k
    reverse(nums, k, nums.length - 1); // step 3: reverse rest
}

private void reverse(int[] nums, int start, int end) {
    while (start < end) {
        int temp = nums[start];
        nums[start++] = nums[end];
        nums[end--] = temp;
    }
}

// Java 8 — rotation using ArrayDeque (readable but O(k) space):
public void rotateDeque(int[] nums, int k) {
    k = k % nums.length;
    ArrayDeque<Integer> dq = IntStream.of(nums)
        .boxed().collect(Collectors.toCollection(ArrayDeque::new));
    IntStream.range(0, k).forEach(i -> dq.addFirst(dq.removeLast()));
    int[] i = {0};
    dq.forEach(v -> nums[i[0]++] = v);
}

// Left rotation by k (rotate left instead of right):
public void rotateLeft(int[] nums, int k) {
    k = k % nums.length;
    reverse(nums, 0, k - 1);           // reverse first k
    reverse(nums, k, nums.length - 1); // reverse rest
    reverse(nums, 0, nums.length - 1); // reverse all
}
nums = [1,2,3,4,5,6,7], k=3, n=7 (k%n = 3) ───────────────────────────────────────────────── Step 1 — Reverse entire array [0..6]: [1,2,3,4,5,6,7] → [7,6,5,4,3,2,1] Step 2 — Reverse first k=3 elements [0..2]: [7,6,5,4,3,2,1] → [5,6,7,4,3,2,1] Step 3 — Reverse remaining n-k=4 elements [3..6]: [5,6,7,4,3,2,1] → [5,6,7,1,2,3,4] ───────────────────────────────────────────────── Answer: [5,6,7,1,2,3,4] ✓ Verification: Original: [1,2,3,4,5,6,7] Rotate 1: [7,1,2,3,4,5,6] Rotate 2: [6,7,1,2,3,4,5] Rotate 3: [5,6,7,1,2,3,4] ✓
Interviewers love this answer: "The most elegant O(1) space solution uses three reverses. Reversing the whole array, then reversing the first k and last n-k subarrays separately, achieves exactly a right rotation. It's a well-known trick worth memorizing — the same reverse helper is used in string rotation problems too."
