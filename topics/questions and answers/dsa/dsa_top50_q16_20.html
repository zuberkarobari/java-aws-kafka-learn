Linked List Cycle — Detect a cycle using O(1) space
EasyFast/Slow PointersFloyd'sClassic
▾
Pattern — Floyd's Tortoise & Hare
Slow pointer moves 1 step, fast pointer moves 2 steps. If there's a cycle, fast will eventually lap slow and they'll meet inside the cycle. If there's no cycle, fast reaches null. O(n) time, O(1) space — no HashSet needed.
P
Problem: Given head of a linked list, determine if the list has a cycle. A cycle means some node's next pointer points back to a previous node.

Input: 3→1→2→4→(back to 1) → Output: true
1
Why Fast/Slow Pointers? If there's no cycle, fast reaches null — exit immediately. If there's a cycle, both pointers enter the cycle. Fast gains 1 step per iteration on slow. Since the cycle is finite, fast MUST eventually catch slow. They meet when (fast_pos - slow_pos) is divisible by cycle length. Guaranteed meeting in O(n).
2
Follow-Up — Find the Cycle Start (LC 142): After detecting meeting point, reset one pointer to head. Move both pointers one step at a time. Where they meet = cycle start. Mathematical proof: distance(head→entry) = distance(meeting→entry going forward). This is exactly what we used in Q9 (Find Duplicate Number) — same algorithm, different context.
3
Find Cycle Length: Once meeting point found, keep one pointer fixed and move the other until it returns to meeting point, counting steps. That count = cycle length.
4
HashSet Alternative (O(n) space): Add each visited node to a HashSet. If you encounter a node already in the set, there's a cycle. Simple but uses O(n) space — mention it as brute force before presenting Floyd's.
Time
O(n)
Space
O(1)
Pattern
Fast/Slow
// ListNode definition (given in interviews):
class ListNode { int val; ListNode next; }

// Cycle Detection — O(n) time, O(1) space
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;           // 1 step
        fast = fast.next.next;      // 2 steps
        if (slow == fast) return true; // met inside cycle
    }
    return false;                   // fast hit null — no cycle
}

// Follow-up: Find cycle start node (LC 142)
public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;

    // Phase 1: find meeting point
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) break;
    }
    if (fast == null || fast.next == null) return null; // no cycle

    // Phase 2: find cycle start
    slow = head;                    // reset to head
    while (slow != fast) {
        slow = slow.next;
        fast = fast.next;           // both move 1 step now
    }
    return slow;                    // meeting point = cycle start
}
List: 3→1→2→4→(→1 cycle) Nodes: head=3, 3.next=1, 1.next=2, 2.next=4, 4.next=1 ───────────────────────────────────────────────── Start: slow=3, fast=3 step1: slow=1, fast=2 (fast skipped 1→2) step2: slow=2, fast=1 (fast went 2→4→1) step3: slow=4, fast=4 (fast went 1→2→4... wait) slow=4, fast=4 EQUAL → cycle detected! ───────────────────────────────────────────────── return true ✓ (For detectCycle Phase 2:) reset slow=head(3), fast stays at meeting(4) step1: slow=1, fast=1 EQUAL → cycle starts at node 1
Always check fast != null && fast.next != null — you need both. If you only check fast.next != null, you'll get NullPointerException when fast is null and you try fast.next.

Reverse Linked List — Iterative and Recursive
EasyPointer ManipulationRecursiveFoundation Problem
▾
Pattern — Three-Pointer In-Place Reversal
Keep three pointers: prev (initially null), curr (initially head), next (temp storage). At each step: save curr.next → point curr.next to prev → advance prev to curr → advance curr to saved next. After loop, prev = new head.
P
Problem: Given head of a singly linked list, reverse the list and return the new head.

Input: 1→2→3→4→5 → Output: 5→4→3→2→1
1
Iterative — Three Pointers: The key danger: before you reverse curr.next, you must save the original curr.next pointer (otherwise you lose the rest of the list). The order is strict: save next → reverse pointer → advance prev → advance curr.
2
Recursive — Trust the Recursion: Assume reverseList(head.next) correctly reverses the sub-list and returns the new head. Now you need to attach head to the end of the reversed sub-list: head.next.next = head (the node after head now points back to head), head.next = null (head becomes the new tail). Return the new head (unchanged from recursion).
3
Reverse a Sublist (LC 92): Reverse from position left to right. Find the node before left, reverse the sublist [left..right], reconnect. Uses the same three-pointer technique with careful boundary tracking.
4
Check Understanding: "What if list has only one node?" — Both iterative and recursive handle it: iterative: curr=head, next=null, curr.next=prev(null), prev=head, curr=null, return prev(head). Recursive: base case fires, return head directly.
Time
O(n)
Space (iter)
O(1)
Space (rec)
O(n) stack
// Iterative — O(n) time, O(1) space (preferred)
public ListNode reverseList(ListNode head) {
    ListNode prev = null;
    ListNode curr = head;
    while (curr != null) {
        ListNode next = curr.next;  // 1. save next
        curr.next = prev;            // 2. reverse pointer
        prev = curr;                 // 3. advance prev
        curr = next;                 // 4. advance curr
    }
    return prev;                    // prev = new head
}

// Recursive — O(n) time, O(n) space (call stack)
public ListNode reverseListRec(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode newHead = reverseListRec(head.next); // reverse rest
    head.next.next = head;  // attach head to reversed tail
    head.next = null;       // head is now the new tail
    return newHead;         // newHead stays the same throughout
}

// Reverse sublist from position left to right (1-indexed, LC 92)
public ListNode reverseBetween(ListNode head, int left, int right) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode pre = dummy;
    for (int i = 1; i < left; i++) pre = pre.next; // reach node before left

    ListNode curr = pre.next;
    for (int i = 0; i < right - left; i++) {
        ListNode next = curr.next;
        curr.next = next.next;
        next.next = pre.next;
        pre.next = next;
    }
    return dummy.next;
}
List: 1 → 2 → 3 → 4 → 5 → null ───────────────────────────────────────────────── Initial: prev=null, curr=1 iter1: next=2, curr.next=null, prev=1, curr=2 null ← 1 2 → 3 → 4 → 5 iter2: next=3, curr.next=1, prev=2, curr=3 null ← 1 ← 2 3 → 4 → 5 iter3: next=4, curr.next=2, prev=3, curr=4 null ← 1 ← 2 ← 3 4 → 5 iter4: next=5, curr.next=3, prev=4, curr=5 iter5: next=null, curr.next=4, prev=5, curr=null null ← 1 ← 2 ← 3 ← 4 ← 5 curr==null → loop ends. return prev = 5 ✓
Draw the pointer diagram when explaining — interviewers appreciate it. The three-pointer dance: "save next, reverse, advance prev, advance curr" — say it like a rhythm to show you have it memorized.

Merge Two Sorted Lists — Merge in O(1) space
EasyMergeTwo PointersFoundation for Merge Sort
▾
Pattern — Dummy Head + Two Pointer Merge
Use a dummy head node to avoid special-casing the result head. Maintain a current pointer on the result list. At each step, attach the smaller of the two current nodes to result.next, then advance that list. When one list is exhausted, attach the rest of the other directly.
P
Problem: Merge two sorted linked lists and return the merged list (sorted).

Input: l1=1→2→4, l2=1→3→4
Output: 1→1→2→3→4→4
1
Dummy Head Trick: Without a dummy head, you'd need special logic to set the head of the result list (before the loop). With a dummy: result always has a valid head (dummy), and curr starts at dummy. The real result starts at dummy.next. This pattern appears in many linked list problems — remember it.
2
Attach Remainder: When one list is exhausted, the other list is already sorted — attach it directly to curr.next. This is O(1) — just one pointer assignment, not a loop.
3
Recursive Version: Elegant but uses O(n+m) stack space. Choose the smaller head, and recursively solve for the rest. Base case: either list is null → return the other. In interviews, both versions are acceptable — mention both, prefer iterative for O(1) space.
4
Extension — Merge K Sorted Lists (LC 23): Use a Min Heap (PriorityQueue) of size K. Initially add head of all K lists. Each iteration: poll minimum node, add to result, push its next node. O(n log k) — a must-know for senior interviews.
Time
O(m+n)
Space
O(1)
Pattern
Dummy Head
// Iterative — O(m+n) time, O(1) space
public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);  // dummy head
    ListNode curr = dummy;

    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            curr.next = l1;
            l1 = l1.next;
        } else {
            curr.next = l2;
            l2 = l2.next;
        }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2; // attach remainder
    return dummy.next;
}

// Recursive — elegant but O(m+n) stack space
public ListNode mergeTwoListsRec(ListNode l1, ListNode l2) {
    if (l1 == null) return l2;
    if (l2 == null) return l1;
    if (l1.val <= l2.val) {
        l1.next = mergeTwoListsRec(l1.next, l2);
        return l1;
    } else {
        l2.next = mergeTwoListsRec(l1, l2.next);
        return l2;
    }
}

// Merge K Sorted Lists (LC 23) — O(n log k) with PriorityQueue
public ListNode mergeKLists(ListNode[] lists) {
    PriorityQueue<ListNode> pq = new PriorityQueue<>(
        Comparator.comparingInt(n -> n.val));
    for (ListNode node : lists)
        if (node != null) pq.offer(node);

    ListNode dummy = new ListNode(0), curr = dummy;
    while (!pq.isEmpty()) {
        curr.next = pq.poll();
        curr = curr.next;
        if (curr.next != null) pq.offer(curr.next);
    }
    return dummy.next;
}
l1: 1→2→4 l2: 1→3→4 dummy→ (curr starts here) ──────────────────────────────────────────── l1.val=1 <= l2.val=1 → pick l1 curr→1(l1), l1=2, curr=1 l1.val=2 > l2.val=1 → pick l2 curr→1(l2), l2=3, curr=1 l1.val=2 <= l2.val=3 → pick l1 curr→2, l1=4, curr=2 l1.val=4 > l2.val=3 → pick l2 curr→3, l2=4, curr=3 l1.val=4 <= l2.val=4 → pick l1 curr→4(l1), l1=null, curr=4 l1==null → exit loop curr.next = l2(4) (attach remainder) ──────────────────────────────────────────── Result: dummy.next = 1→1→2→3→4→4 ✓

Reorder List — L0→L1→…→Ln-1→Ln to L0→Ln→L1→Ln-1…
MediumFast/SlowReverse + Merge3-Step Pattern
▾
Pattern — Find Middle + Reverse Second Half + Merge
Three steps: (1) Find the middle of the list using slow/fast pointers. (2) Reverse the second half in-place. (3) Merge the two halves by interleaving. Each step uses a technique you already know — the power is combining them.
P
Problem: Given the head of a linked list, reorder it such that nodes alternate from front and back: L0→Ln→L1→Ln-1→L2→Ln-2→… Do this in-place.

Input: 1→2→3→4→5 → Output: 1→5→2→4→3
1
Step 1 — Find Middle: Use slow/fast pointers. When fast reaches end, slow is at middle. Split list: slow.next = null cuts it into two halves. First half: [1→2→3], second half: [4→5].
2
Step 2 — Reverse Second Half: Apply our reverseList from Q17 to the second half. [4→5] becomes [5→4].
3
Step 3 — Interleave Merge: Two pointers: p1 on first half [1→2→3], p2 on reversed second half [5→4]. Each iteration: save p1.next and p2.next, insert p2 after p1, advance both. Careful with the termination condition — when p2 is null or p2.next is null.
4
Why This Works: After reversing the second half, merging front and reversed-back gives exactly the required order. The three-step structure (find middle, reverse, merge) is a reusable pattern — it also appears in Palindrome Linked List (LC 234).
Time
O(n)
Space
O(1)
Pattern
3-Step
public void reorderList(ListNode head) {
    if (head == null || head.next == null) return;

    // Step 1: Find the middle
    ListNode slow = head, fast = head;
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    // slow is now at middle; cut the list
    ListNode secondHalf = slow.next;
    slow.next = null;

    // Step 2: Reverse second half
    secondHalf = reverse(secondHalf);

    // Step 3: Merge two halves
    ListNode p1 = head, p2 = secondHalf;
    while (p2 != null) {
        ListNode tmp1 = p1.next, tmp2 = p2.next;
        p1.next = p2;
        p2.next = tmp1;
        p1 = tmp1;
        p2 = tmp2;
    }
}

private ListNode reverse(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}
Input: 1→2→3→4→5 ────────────────────────────────────────────────── Step 1 — Find Middle: slow/fast start at 1 fast.next!=null && fast.next.next!=null: slow=2, fast=3 slow=3, fast=5 (fast.next=null → stop) Middle = slow = 3 First half: 1→2→3 Second half: 4→5 Step 2 — Reverse second half: 4→5 becomes 5→4 Step 3 — Interleave: p1=1, p2=5 iter1: p1.next=5, 5.next=2, p1=2, p2=4 list so far: 1→5→2→... iter2: p1.next=4, 4.next=3, p1=3, p2=null list so far: 1→5→2→4→3 p2==null → stop ────────────────────────────────────────────────── Result: 1→5→2→4→3 ✓
The key insight to say: "This is a combination of three primitives I already know — find middle with slow/fast, reverse a linked list, and merge two lists. Breaking a complex problem into known sub-problems is the skill being tested."

Remove Nth Node From End — One-pass with two pointers
MediumTwo PointersOne PassClever Gap Trick
▾
Pattern — N-Gap Two Pointers (Dummy Head)
Place fast pointer n+1 steps ahead of slow (using dummy head). Move both together until fast reaches null. Now slow is right before the node to delete — set slow.next = slow.next.next. The n+1 gap ensures slow stops one node BEFORE the target, not at it.
P
Problem: Given head of a linked list, remove the nth node from the end in one pass and return the head.

Input: 1→2→3→4→5, n=2 → Output: 1→2→3→5 (node 4 removed)
1
The Gap Insight: If fast is exactly n+1 steps ahead of slow, when fast reaches null, slow is at the node JUST BEFORE the one to delete. This gap is maintained throughout — both pointers move at the same speed. n+1 (not n) because we need slow to be BEFORE the target for deletion.
2
Dummy Head is Essential: What if we need to delete the first node (n = list length)? Without dummy, fast would advance past null before slow starts, and slow would never reach position 0. With dummy as the starting point, slow = dummy when we need to delete node 1. slow.next = slow.next.next correctly removes it.
3
Two-Pass Alternative: First pass — count total length L. Second pass — go to node (L-n), delete next. Simpler to understand but two passes. The one-pass solution is what interviewers want — it demonstrates insight.
4
Edge Cases: n = list length (delete head) → handled by dummy head. n = 1 (delete tail) → slow lands at second-to-last node, slow.next = null. Single node list, n=1 → slow=dummy, dummy.next=null, return dummy.next=null.
Time
O(n)
Space
O(1)
Passes
1 Pass
public ListNode removeNthFromEnd(ListNode head, int n) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode fast = dummy, slow = dummy;

    // Advance fast by n+1 steps to create n-gap
    for (int i = 0; i <= n; i++) {     // n+1 steps (note: <= not <)
        fast = fast.next;
    }

    // Move both until fast == null
    while (fast != null) {
        slow = slow.next;
        fast = fast.next;
    }

    // slow is now just before the node to delete
    slow.next = slow.next.next;

    return dummy.next;
}

// Why n+1 not n?
// If fast is n steps ahead and we stop when fast reaches LAST node,
// slow is at target (need prev). With n+1, fast reaches null, slow = prev.
// Both are correct — the choice depends on while condition (last vs null).

// Alternative using n (fast stops at last node, not null):
public ListNode removeNthV2(ListNode head, int n) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode fast = head, slow = dummy;
    for (int i = 0; i < n; i++) fast = fast.next;
    while (fast != null) { slow = slow.next; fast = fast.next; }
    slow.next = slow.next.next;
    return dummy.next;
}
List: dummy→1→2→3→4→5, n=2 ───────────────────────────────────────────────── Step 1 — Advance fast by n+1=3 steps from dummy: fast: dummy→1→2→3 (fast is at node 3) Step 2 — Move both until fast==null: fast=3, slow=dummy: fast→4, slow→1 fast=4, slow=1: fast→5, slow→2 fast=5, slow=2: fast→null, slow→3 fast==null → stop slow = node 3 (just before node 4, which is 2nd from end) Step 3 — Delete: slow.next = slow.next.next node3.next = node5 (skip node4) ───────────────────────────────────────────────── Result: 1→2→3→5 ✓ (node 4 removed) Edge case: n=5 (delete head, list=1→2→3→4→5) fast advances 6 steps from dummy → fast=null immediately after 5 steps slow stays at dummy → slow.next(=1) gets skipped Return dummy.next = 2→3→4→5 ✓
The most common mistake: advancing fast by n steps instead of n+1 from dummy, then using while(fast.next != null). Both approaches are equivalent — pick one and be consistent. The n+1-from-dummy + while(fast!=null) version is cleaner.
