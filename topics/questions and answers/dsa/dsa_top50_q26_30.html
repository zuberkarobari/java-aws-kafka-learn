Number of Islands — Count connected components in a grid
MediumGraph DFSBFSMost Asked Graph Q
▾
Pattern — DFS/BFS Flood Fill to Count Components
Iterate through every cell. When you find an unvisited land cell ('1'), increment island count and DFS/BFS from that cell to mark all connected land cells as visited. After the traversal, the count equals the number of islands. In-place marking ('1'→'0') avoids a separate visited array.
P
Problem: Given an m×n grid of '1' (land) and '0' (water), return the number of islands. An island is surrounded by water and formed by connecting adjacent land cells horizontally or vertically.

Input: grid with two islands → Output: 2
1
DFS Approach — Sink the Island: When you find '1', increment count, then DFS in all 4 directions, marking each visited '1' as '0' (sinking it). This prevents counting the same island twice. Time: O(m×n) — each cell visited at most once. Space: O(m×n) in worst case for recursion stack (when entire grid is land).
2
BFS Alternative: Same logic with a queue instead of recursion. Add the starting cell to queue, mark visited, then process each cell's 4 neighbors. BFS avoids stack overflow for very large grids — prefer BFS in production/interviews when grid can be huge.
3
Union-Find Alternative: Each land cell is a node. Connect adjacent land cells. Count connected components. More complex to implement but good to mention for follow-up: "How would you handle dynamic land/water changes?" — Union-Find handles incremental merges efficiently.
4
Variations asked in fintech interviews: Number of distinct islands (need to track shape), Max area of island (track size during DFS), Number of Enclaves (islands touching border vs not). All use the same DFS/BFS flood-fill pattern.
Time
O(m×n)
Space
O(m×n)
Pattern
Flood Fill
// DFS — in-place marking, O(m×n) time/space
public int numIslands(char[][] grid) {
    int count = 0;
    for (int r = 0; r < grid.length; r++) {
        for (int c = 0; c < grid[0].length; c++) {
            if (grid[r][c] == '1') {
                count++;
                dfs(grid, r, c);           // sink entire island
            }
        }
    }
    return count;
}

private void dfs(char[][] grid, int r, int c) {
    if (r < 0 || r >= grid.length || c < 0 || c >= grid[0].length
        || grid[r][c] == '0') return;
    grid[r][c] = '0';               // mark visited by sinking
    dfs(grid, r + 1, c); dfs(grid, r - 1, c);
    dfs(grid, r, c + 1); dfs(grid, r, c - 1);
}

// BFS — avoids recursion stack overflow for huge grids
public int numIslandsBFS(char[][] grid) {
    int m = grid.length, n = grid[0].length, count = 0;
    int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
    for (int r = 0; r < m; r++) {
        for (int c = 0; c < n; c++) {
            if (grid[r][c] == '1') {
                count++;
                Queue<int[]> q = new LinkedList<>();
                q.offer(new int[]{r, c});
                grid[r][c] = '0';
                while (!q.isEmpty()) {
                    int[] cell = q.poll();
                    for (int[] d : dirs) {
                        int nr = cell[0] + d[0], nc = cell[1] + d[1];
                        if (nr>=0&&nr<m&&nc>=0&&nc<n&&grid[nr][nc]=='1') {
                            grid[nr][nc] = '0';
                            q.offer(new int[]{nr, nc});
                        }
                    }
                }
            }
        }
    }
    return count;
}

// Follow-up: Max Area of Island — track size during DFS
public int maxAreaOfIsland(int[][] grid) {
    int max = 0;
    for (int r = 0; r < grid.length; r++)
        for (int c = 0; c < grid[0].length; c++)
            if (grid[r][c] == 1)
                max = Math.max(max, dfsArea(grid, r, c));
    return max;
}
private int dfsArea(int[][] g, int r, int c) {
    if (r<0||r>=g.length||c<0||c>=g[0].length||g[r][c]!=1) return 0;
    g[r][c] = 0;
    return 1 + dfsArea(g,r+1,c) + dfsArea(g,r-1,c) + dfsArea(g,r,c+1) + dfsArea(g,r,c-1);
}

DFS vs BFS — both work. In interviews, DFS is faster to write. But mention: "For very large grids I'd prefer BFS to avoid stack overflow from deep recursion." This shows production awareness interviewers at senior level appreciate.

Clone Graph — Deep copy of an undirected graph
MediumGraph DFSHashMapPointer Mapping
▾
Pattern — HashMap as Old→New Node Registry
Use a HashMap<Node, Node> mapping original nodes to their clones. During DFS/BFS, if a node is already cloned (exists in map), return the clone. Otherwise create the clone, add to map, then recursively clone all neighbors. The map prevents infinite loops on cycles AND provides O(1) lookup of already-cloned nodes.
P
Problem: Given a reference to a node in an undirected graph, return a deep copy (clone) of the graph. Each node contains val and a list of neighbors.

Input: node with val=1, neighbors=[2,4], and the connected graph → Output: cloned graph (all new objects)
1
Why HashMap is Essential: Graphs can have cycles (1→2→3→1). Without tracking cloned nodes, DFS would recurse infinitely. The HashMap serves two purposes: (1) Cycle detection — if node already in map, return existing clone. (2) Pointer consistency — multiple nodes pointing to the same original must point to the SAME clone, not different copies.
2
DFS Clone Pattern: Base case: if node is null, return null. If node already cloned (in map), return map.get(node). Otherwise: create a new node with same val, add to map BEFORE recursing (prevents infinite loop on cycles), then recursively clone each neighbor and add to clone's neighbor list.
3
BFS Alternative: Queue-based. Add original node to queue, create its clone, put in map. For each node dequeued, iterate neighbors: if not in map, create clone, add to map, enqueue. Add clone of neighbor to current clone's list either way.
4
General Pattern: This "HashMap as node registry during graph traversal" pattern applies to: clone any graph structure (including doubly linked lists, N-ary trees with back-pointers), copy constructors for cyclic object graphs, serializing graphs.
Time
O(V+E)
Space
O(V)
Pattern
DFS+HashMap
class Node {
    int val;
    List<Node> neighbors;
    Node(int v) { val=v; neighbors=new ArrayList<>(); }
}

// DFS with HashMap registry
private Map<Node, Node> cloneMap = new HashMap<>();

public Node cloneGraph(Node node) {
    if (node == null) return null;
    if (cloneMap.containsKey(node)) return cloneMap.get(node); // already cloned

    Node clone = new Node(node.val);
    cloneMap.put(node, clone);          // register BEFORE recursing (breaks cycles)

    for (Node neighbor : node.neighbors)
        clone.neighbors.add(cloneGraph(neighbor));

    return clone;
}

// BFS version — no recursion stack risk
public Node cloneGraphBFS(Node node) {
    if (node == null) return null;
    Map<Node,Node> map = new HashMap<>();
    Queue<Node> q = new LinkedList<>();

    map.put(node, new Node(node.val));
    q.offer(node);

    while (!q.isEmpty()) {
        Node curr = q.poll();
        for (Node nb : curr.neighbors) {
            if (!map.containsKey(nb)) {
                map.put(nb, new Node(nb.val));
                q.offer(nb);
            }
            map.get(curr).neighbors.add(map.get(nb));
        }
    }
    return map.get(node);
}

You MUST add the node to the map BEFORE recursing into its neighbors — not after. If you add it after, a cycle (1→2→1) will cause infinite recursion because when you reach node 1 again, it's not yet in the map.

Pacific Atlantic Water Flow — Multi-source BFS from both oceans
MediumMulti-Source BFSGraphReverse Thinking
▾
Pattern — Reverse Multi-Source BFS (Flow Backwards)
Instead of simulating water flowing downhill from every cell (expensive), think in reverse: start BFS/DFS from both ocean borders and flow UPHILL (to cells with equal or greater height). Mark cells reachable from Pacific and cells reachable from Atlantic. The intersection is the answer.
P
Problem: A rectangular island has Pacific Ocean on top/left edges and Atlantic Ocean on bottom/right edges. Water flows to adjacent cells with equal or lower height. Find all cells that can flow to BOTH oceans.

Input: m×n height matrix → Output: list of [row,col] positions
1
The Reverse Insight: Forward simulation: for each cell, DFS downhill to see which oceans it reaches — O(m²×n²), too slow. Reverse: from each ocean border, BFS uphill (can go to equal or higher cells) to find all cells that drain to that ocean. O(m×n) total — the key insight that makes this solvable efficiently.
2
Two BFS Passes: Pass 1: seed Pacific queue with top row + left column. BFS uphill, mark reachable cells in pacific boolean array. Pass 2: seed Atlantic queue with bottom row + right column. BFS uphill, mark in atlantic array. Final: collect cells where both arrays are true.
3
Uphill BFS Condition: Move to neighbor if: not visited AND neighbor height ≥ current height. This reverses the downhill water flow — in the forward direction, water flows to equal or lower cells. In the reverse (uphill) direction, water could have come from equal or higher cells.
4
DFS Alternative: Same logic with recursion instead of queue. Both have same complexity. BFS is preferred to avoid stack overflow on large grids. The multi-source start (initialising with all border cells at once) is the key efficiency — O(m×n) instead of O(m²×n²).
Time
O(m×n)
Space
O(m×n)

public List<List<Integer>> pacificAtlantic(int[][] heights) {
    int m = heights.length, n = heights[0].length;
    boolean[][] pac = new boolean[m][n];
    boolean[][] atl = new boolean[m][n];

    // Seed border cells for each ocean
    Queue<int[]> pacQ = new LinkedList<>();
    Queue<int[]> atlQ = new LinkedList<>();

    for (int i = 0; i < m; i++) {
        pacQ.offer(new int[]{i, 0});   pac[i][0] = true;  // left col
        atlQ.offer(new int[]{i, n-1}); atl[i][n-1] = true; // right col
    }
    for (int j = 0; j < n; j++) {
        pacQ.offer(new int[]{0, j});   pac[0][j] = true;  // top row
        atlQ.offer(new int[]{m-1, j}); atl[m-1][j] = true; // bottom row
    }

    bfs(heights, pacQ, pac);
    bfs(heights, atlQ, atl);

    // Collect intersection
    List<List<Integer>> result = new ArrayList<>();
    for (int r = 0; r < m; r++)
        for (int c = 0; c < n; c++)
            if (pac[r][c] && atl[r][c])
                result.add(Arrays.asList(r, c));
    return result;
}

private void bfs(int[][] h, Queue<int[]> q, boolean[][] visited) {
    int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
    int m = h.length, n = h[0].length;
    while (!q.isEmpty()) {
        int[] cell = q.poll();
        for (int[] d : dirs) {
            int nr = cell[0]+d[0], nc = cell[1]+d[1];
            if (nr>=0&&nr<m&&nc>=0&&nc<n
                && !visited[nr][nc]
                && h[nr][nc] >= h[cell[0]][cell[1]]) { // uphill!
                visited[nr][nc] = true;
                q.offer(new int[]{nr, nc});
            }
        }
    }
}

The reverse thinking is the key insight. Say: "Instead of simulating water flowing down from every cell — O(m²n²) — I reverse the problem. I ask: which cells can reach each ocean by flowing uphill? I start BFS from the ocean borders and expand to cells with equal or greater height."

Course Schedule — Detect cycle in directed graph (Topological Sort)
MediumTopological SortCycle DetectionMust Know
▾
Pattern — Kahn's Algorithm (BFS Topological Sort)
Build adjacency list and in-degree array. Add all nodes with in-degree 0 to queue (can be taken first — no prerequisites). Process queue: for each course, reduce in-degree of its dependents. If a dependent's in-degree drops to 0, add to queue. If all courses are processed → no cycle → return true.
P
Problem: There are numCourses courses. Given prerequisites list where [a,b] means "to take a, must first take b", determine if it's possible to finish all courses (i.e., no circular dependency).

Input: numCourses=2, prerequisites=[[1,0],[0,1]] → Output: false (cycle: 0→1→0)
1
The Core Insight: A valid course schedule exists if and only if the prerequisite graph has no cycles (it's a DAG — Directed Acyclic Graph). Two approaches: Kahn's BFS (count processed nodes) or DFS with 3-state coloring (WHITE/GRAY/BLACK). Both detect cycles in O(V+E).
2
Kahn's Algorithm (BFS): Repeatedly remove nodes with no incoming edges (in-degree 0). If at the end, all nodes were removed → no cycle. If some nodes couldn't be removed (they still had incoming edges from cycle partners) → cycle exists. Count of processed nodes tells you: if count == numCourses → no cycle.
3
DFS with 3-State Coloring: WHITE (unvisited), GRAY (currently in DFS stack), BLACK (fully processed). If during DFS you encounter a GRAY node → back edge → cycle. If BLACK → already fully processed, safe. Mark WHITE→GRAY when entering, GRAY→BLACK when exiting DFS.
4
Course Schedule II (Return the Order, LC 210): Same algorithm — for Kahn's, the order courses are dequeued is the valid topological order. For DFS, add to result list when marking GRAY→BLACK (post-order), then reverse.
Time
O(V+E)
Space
O(V+E)
Pattern

// Kahn's Algorithm — BFS Topological Sort
public boolean canFinish(int n, int[][] prereqs) {
    List<List<Integer>> adj = new ArrayList<>();
    int[] inDegree = new int[n];
    for (int i = 0; i < n; i++) adj.add(new ArrayList<>());

    for (int[] pre : prereqs) {
        adj.get(pre[1]).add(pre[0]); // pre[1] → pre[0]
        inDegree[pre[0]]++;
    }

    Queue<Integer> q = new LinkedList<>();
    for (int i = 0; i < n; i++)
        if (inDegree[i] == 0) q.offer(i);  // no prerequisites

    int processed = 0;
    while (!q.isEmpty()) {
        int course = q.poll();
        processed++;
        for (int next : adj.get(course)) {
            if (--inDegree[next] == 0) q.offer(next);
        }
    }
    return processed == n;             // all processed = no cycle
}

// DFS with 3-state coloring
public boolean canFinishDFS(int n, int[][] prereqs) {
    List<List<Integer>> adj = buildAdj(n, prereqs);
    int[] state = new int[n]; // 0=white, 1=gray, 2=black
    for (int i = 0; i < n; i++)
        if (state[i] == 0 && hasCycle(adj, state, i)) return false;
    return true;
}
private boolean hasCycle(List<List<Integer>> adj, int[] state, int node) {
    state[node] = 1;               // mark gray (in progress)
    for (int next : adj.get(node)) {
        if (state[next] == 1) return true;  // back edge = cycle
        if (state[next] == 0 && hasCycle(adj, state, next)) return true;
    }
    state[node] = 2;               // mark black (done)
    return false;
}

Know both Kahn's BFS and DFS 3-state. Kahn's is easier to explain verbally ("repeatedly remove nodes with no prerequisites"). DFS is more natural if you've already written DFS for other graph problems. Interviewers accept either — pick the one you can explain most clearly.

Surrounded Regions — Capture all 'O's not connected to the border
MediumGraph DFSReverse ThinkingFlood Fill Variant
▾
Pattern — Mark Safe Cells from Border, Then Flip the Rest
Any 'O' connected to a border 'O' cannot be captured (it's safe). Start DFS/BFS from every border 'O' and mark all connected 'O's as safe (e.g., 'S'). Then do a single pass: 'O' → 'X' (captured), 'S' → 'O' (restore safe). This reverse-thinking approach is O(m×n) with one pass per step.
P
Problem: Given an m×n board with 'X' and 'O', capture all regions of 'O' that are completely surrounded by 'X'. A region is captured by flipping all 'O's to 'X'.
Note: 'O's on the border and connected to border 'O's cannot be captured.

Input: board with inner 'O's and border 'O's → flip only the inner ones
1
The Reverse Insight: Forward approach: for each 'O', DFS to check if it reaches the border — complex and expensive. Reverse: start from border 'O's and mark all 'O's reachable from borders as safe. Everything remaining is surrounded. This reduces the problem to two simple passes.
2
Three Steps: (1) For each 'O' on the 4 borders, DFS/BFS and mark all connected 'O's as 'S' (safe). (2) Single board pass: flip 'O'→'X' (captured regions), flip 'S'→'O' (restore safe). (3) Done — O(m×n) total.
3
Why This Works: 'O's that survive (not captured) are exactly those connected to the border. By flooding from border 'O's first, we identify ALL survivors in O(m×n). The final sweep correctly flips all non-survivors to 'X'.
4
Pattern Reuse: This "mark safe from border, then flip rest" pattern also appears in: Number of Enclaves (count 1s not connected to border), Walls and Gates (BFS from gates). The key insight — working backwards from the boundary — is reusable across multiple graph problems.
Time
O(m×n)
Space
O(m×n)
Pattern

public void solve(char[][] board) {
    int m = board.length, n = board[0].length;

    // Step 1: Mark all 'O's connected to border as safe ('S')
    for (int r = 0; r < m; r++) {
        markSafe(board, r, 0);       // left border
        markSafe(board, r, n - 1);   // right border
    }
    for (int c = 0; c < n; c++) {
        markSafe(board, 0, c);       // top border
        markSafe(board, m - 1, c);   // bottom border
    }

    // Step 2: Flip — 'O'→'X' (captured), 'S'→'O' (restore)
    for (int r = 0; r < m; r++)
        for (int c = 0; c < n; c++) {
            if (board[r][c] == 'O') board[r][c] = 'X'; // captured
            if (board[r][c] == 'S') board[r][c] = 'O'; // restore safe
        }
}

private void markSafe(char[][] board, int r, int c) {
    if (r < 0 || r >= board.length || c < 0 || c >= board[0].length
        || board[r][c] != 'O') return;
    board[r][c] = 'S';             // mark as safe
    markSafe(board, r+1, c); markSafe(board, r-1, c);
    markSafe(board, r, c+1); markSafe(board, r, c-1);
}

// BFS version of markSafe (avoids stack overflow on large boards)
private void markSafeBFS(char[][] b, int r, int c) {
    if (b[r][c] != 'O') return;
    Queue<int[]> q = new LinkedList<>();
    q.offer(new int[]{r,c}); b[r][c]='S';
    int[][] dirs={{1,0},{-1,0},{0,1},{0,-1}};
    while(!q.isEmpty()){
        int[] cur=q.poll();
        for(int[] d:dirs){
            int nr=cur[0]+d[0],nc=cur[1]+d[1];
            if(nr>=0&&nr<b.length&&nc>=0&&nc<b[0].length&&b[nr][nc]=='O')
                {b[nr][nc]='S'; q.offer(new int[]{nr,nc});}
        }
    }
}
