Arithmetic & Relational Operators
The foundation of computation and comparison in Java. Beyond simple math, understanding how prefix/postfix evaluation and object comparisons work is crucial.
Math Comparisons Increment

Java provides standard arithmetic operators: +, -, *, /, and % (modulo). Note that integer division drops the fractional part (e.g., 5 / 2 evaluates to 2).

java — Increment & Decrement Traps
int x = 5;
int y = ++x; // Prefix: Increment first, then assign. x=6, y=6.

int a = 5;
int b = a++; // Postfix: Assign first, then increment. a=6, b=5.

// String Concatenation using +
System.out.println("Total: " + 5 + 4);   // "Total: 54" (Left-to-right String conversion)
System.out.println(5 + 4 + " Total");   // "9 Total" (Math first, then String conversion)

Relational operators (==, !=, >, <, >=, <=) always return a boolean.

The Relational == vs .equals() Trap For primitives, == compares values. For Objects, == compares memory references.

new String("A") == new String("A") is false because they are two distinct objects in memory. Always use .equals() for object content comparison!
Logical & Ternary Operators
Logical operators control the flow of decision-making. The ternary operator offers a concise way to handle simple if-else assignments.

The standard logical operators are && (AND), || (OR), and ! (NOT). They are known as "short-circuit" operators because they stop evaluating as soon as the outcome is certain.

java
User user = null;

// Short-circuit AND (&&) saves us from a NullPointerException here:
if (user != null && user.isActive()) {
    // If user is null, the first condition is false. 
    // Java skips the second condition entirely.
}

// Short-circuit OR (||) stops if the first is true:
if (isValid() || heavyDatabaseCheck()) {
    // If isValid() is true, heavyDatabaseCheck() is NEVER executed.
}
Single & and | Operators Java also has non-short-circuiting logical operators: & and |. If you use them in an if statement, Java evaluates both sides regardless of the first side's result. This is rarely what you want in conditional logic, but they double as Bitwise operators for numeric types.

The Ternary operator (? :) is Java's only operator that takes three operands. It functions as a single-line if-else statement that returns a value.

Syntax: condition ? valueIfTrue : valueIfFalse;

java
int age = 20;

// Traditional if-else
String status;
if (age >= 18) {
    status = "Adult";
} else {
    status = "Minor";
}

// Ternary equivalent
String status = (age >= 18) ? "Adult" : "Minor";
Ternary Null Checks A common production pattern is using the ternary operator for safe variable assignment to avoid nulls:
String displayName = (user.getName() != null) ? user.getName() : "Guest";
Bitwise & Shift Operators
These operators manipulate the individual bits of integer types. While rare in high-level web dev, they are essential for performance optimization, cryptography, networking, and reading source code of low-level libraries.
Bitwise Logic
& (AND): 1 if both bits are 1.
| (OR): 1 if either bit is 1.
^ (XOR): 1 if bits are different.
~ (NOT): Inverts all bits (1s complement).
Bit Shifts
<< (Left Shift): Shifts bits left, fills with 0. Effectively multiplies by 2^n.
>> (Right Shift): Shifts right, preserves the sign bit (fills with 0 if positive, 1 if negative). Effectively divides by 2^n.
>>> (Unsigned Right): Shifts right, ALWAYS fills with 0, ignoring the sign.
java
int a = 5;  // Binary: 0000 0101
int b = 3;  // Binary: 0000 0011

System.out.println(a & b);  // 0000 0001 (1)
System.out.println(a | b);  // 0000 0111 (7)
System.out.println(a ^ b);  // 0000 0110 (6)

int c = 8;  // Binary: 0000 1000
System.out.println(c << 1); // 0001 0000 (16) - Multiplied by 2
System.out.println(c >> 2); // 0000 0010 (2)  - Divided by 4
Operator Precedence & Evaluation
When multiple operators exist in a single statement, Java relies on precedence rules to determine what gets evaluated first.

Similar to PEMDAS in mathematics, Java has a hierarchy. Highest precedence evaluates first:

  1. Postfix (expr++, expr--)
  2. Unary (++expr, -expr, !, ~)
  3. Multiplicative (*, /, %)
  4. Additive (+, -)
  5. Shift (<<, >>, >>>)
  6. Relational (<, >, <=, >=, instanceof)
  7. Equality (==, !=)
  8. Logical AND (&&) then Logical OR (||)
  9. Ternary (? :)
  10. Assignment (=, +=, -=, etc.)
Assignment Evaluates Right-to-Left While most operators evaluate left-to-right (e.g., 10 - 5 - 2 is (10-5) - 2 = 3), assignment evaluates right-to-left.

This allows chaining: int a, b, c; a = b = c = 100;
c = 100 evaluates first, returning 100, which is then assigned to b, which returns 100, which is assigned to a.
java — Compound Assignment Implicit Cast
byte b = 10;

// This causes a COMPILE ERROR:
// b = b + 5; 
// Reason: 'b + 5' automatically promotes to an 'int'. 
// You cannot assign an int back into a byte without explicit casting.

// This WORKS:
b += 5; 
// Reason: Compound assignment operators implicitly cast the result back
// to the target type. It is equivalent to: b = (byte)(b + 5);
Interview Prep
Operators seem basic, but interviewers use them to test deep JVM understanding, short-circuit logic, and casting rules.
Q1 What is the difference between && and & operators? Medium ▼

&& is the Logical AND operator. It is a short-circuit operator. If the left operand evaluates to false, it immediately returns false without evaluating the right operand. This is crucial for preventing NullPointerExceptions (e.g., if (obj != null && obj.isValid())).

& is a Bitwise AND operator, but it can also be used as a Logical AND for booleans. When used with booleans, it is a non-short-circuit operator. It evaluates both the left and right operands regardless of the left operand's outcome. It is rarely used for conditional logic.

Q2 If x = 5, what is the value of x and the result of the expression (x++ + ++x)? Hard ▼

Let's break down the evaluation left-to-right:

1. x++ evaluates to the current value of x (which is 5), and then increments x to 6.

2. The addition operator + waits for the right side.

3. ++x increments x first (from 6 to 7), and then evaluates to 7.

4. The expression resolves to 5 + 7, which equals 12.

The final value of the variable x is 7.

Pro Tip: Compilers evaluate operands strictly from left to right in Java, resolving side effects as they go.
