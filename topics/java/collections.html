Java Collections Framework

The Java Collections Framework (JCF) is a unified architecture for storing, retrieving, and manipulating groups of objects. It provides a rich set of interfaces and classes — List, Set, Map, Queue, Deque — along with powerful algorithms for sorting and searching. Mastering collections is essential for writing real-world, performant Java.

1. Collections Hierarchy Overview

The framework is built around two root interfaces: Collection (for sequences and sets of elements) and Map (for key-value pairs). They live in java.util and are generic, so you always parameterise them with a type. 

// The two root interface families
java.util.Collection<E>
  ├── List<E>        // ordered, index-based, duplicates allowed
  │     ├── ArrayList<E>
  │     └── LinkedList<E>
  ├── Set<E>         // no duplicates
  │     ├── HashSet<E>
  │     ├── LinkedHashSet<E>
  │     └── TreeSet<E>     (SortedSet)
  └── Queue<E>       // FIFO / priority ordering
        ├── PriorityQueue<E>
        └── Deque<E>        // double-ended queue
              └── ArrayDeque<E>

java.util.Map<K, V>
  ├── HashMap<K, V>
  ├── LinkedHashMap<K, V>
  └── TreeMap<K, V>         (SortedMap)

Every collection is also an Iterable<E>, so you can use the enhanced for-each loop on any of them. Map is not a subtype of Collection, but you can obtain collection views of its keys, values, or entries.

2. List — Ordered, Index-Based

A List maintains insertion order, allows duplicates, and provides positional access by index. The two most-used implementations are ArrayList and LinkedList.

2.1 ArrayList

Backed by a resizable array. Offers O(1) random access and fast iteration. Appending to the end is amortised O(1). Insertion or removal in the middle is O(n) because elements must be shifted. 

import java.util.ArrayList;
import java.util.List;

public class ArrayListDemo {
    public static void main(String[] args) {
        // Create — always program to the interface
        List<String> fruits = new ArrayList<>();

        // Add
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Cherry");
        fruits.add(1, "Blueberry"); // insert at index 1

        // Access
        String first = fruits.get(0);         // "Apple"
        int size      = fruits.size();         // 4
        int idx       = fruits.indexOf("Banana"); // 2

        // Update
        fruits.set(0, "Avocado");

        // Remove
        fruits.remove("Cherry");              // by object
        fruits.remove(0);                     // by index

        // Bulk operations
        List<String> more = List.of("Grape", "Mango");
        fruits.addAll(more);

        System.out.println(fruits); // [Banana, Blueberry, Grape, Mango]
    }
}

2.2 LinkedList

Backed by a doubly-linked list. O(1) insertion and removal at the head or tail, but O(n) random access. Use it when you frequently add or remove from either end, or when you need a Deque / stack behaviour. 

import java.util.LinkedList;
import java.util.Deque;

public class LinkedListDemo {
    public static void main(String[] args) {
        LinkedList<Integer> list = new LinkedList<>();

        // Efficient head / tail operations
        list.addFirst(10);   // [10]
        list.addLast(20);    // [10, 20]
        list.addFirst(5);    // [5, 10, 20]

        System.out.println(list.peekFirst()); // 5  — no removal
        System.out.println(list.pollFirst()); // 5  — removes it
        System.out.println(list.peekLast());  // 20

        // Also usable as a Deque
        Deque<Integer> deque = new LinkedList<>();
        deque.push(1);  // push = addFirst
        deque.push(2);
        deque.pop();    // pop  = removeFirst → 2
    }
}

2.3 ArrayList vs LinkedList — Quick Reference

Operation ArrayList LinkedList
get(i) / set(i)O(1) ✅O(n) ❌
add(element) — endO(1) amortised ✅O(1) ✅
add(i, element) — middleO(n)O(n) (traversal)
remove(i) — middleO(n)O(n) (traversal)
addFirst / addLastO(n) / O(1)O(1) ✅
Memory overheadLowHigh (node pointers)
Cache localityExcellent ✅Poor ❌

Rule of thumb: default to ArrayList. Switch to LinkedList only when you have heavy head/tail insertions and removals, or when you need Deque behaviour without a backing array.

3. Set — Unique Elements

A Set guarantees no duplicate elements. Attempting to add an existing element is a no-op that returns false. Three implementations cover the common cases.

3.1 HashSet

Backed by a HashMap. Offers O(1) average add, remove, and contains. Order is not guaranteed — elements can appear in any sequence. 

import java.util.HashSet;
import java.util.Set;

public class HashSetDemo {
    public static void main(String[] args) {
        Set<String> colors = new HashSet<>();

        colors.add("Red");
        colors.add("Green");
        colors.add("Blue");
        colors.add("Red");   // duplicate — silently ignored

        System.out.println(colors.size());         // 3
        System.out.println(colors.contains("Blue")); // true

        colors.remove("Green");

        // Set operations
        Set<String> warm = new HashSet<>(Set.of("Red", "Orange", "Yellow"));
        Set<String> all  = new HashSet<>(colors);

        all.addAll(warm);              // union
        colors.retainAll(warm);        // intersection (mutates colors)
        all.removeAll(warm);           // difference
    }
}

3.2 LinkedHashSet

Maintains insertion order by combining a hash table with a linked list. Slightly slower than HashSet due to the extra bookkeeping, but still O(1) operations. Use when iteration order matters. 

import java.util.LinkedHashSet;
import java.util.Set;

public class LinkedHashSetDemo {
    public static void main(String[] args) {
        Set<String> langs = new LinkedHashSet<>();
        langs.add("Java");
        langs.add("Kotlin");
        langs.add("Scala");
        langs.add("Java");  // duplicate ignored

        // Iteration respects insertion order
        for (String lang : langs) {
            System.out.print(lang + " ");  // Java Kotlin Scala
        }
    }
}

3.3 TreeSet

Backed by a Red-Black tree. Elements are kept in sorted order (natural ordering or a custom Comparator). Operations are O(log n). Implements NavigableSet, offering range-query methods like floor(), ceiling(), headSet(), and tailSet(). 

import java.util.TreeSet;

public class TreeSetDemo {
    public static void main(String[] args) {
        TreeSet<Integer> numbers = new TreeSet<>();
        numbers.add(50);
        numbers.add(20);
        numbers.add(80);
        numbers.add(10);
        numbers.add(40);

        System.out.println(numbers);          // [10, 20, 40, 50, 80]
        System.out.println(numbers.first());  // 10
        System.out.println(numbers.last());   // 80
        System.out.println(numbers.floor(35));   // 20 — largest ≤ 35
        System.out.println(numbers.ceiling(35)); // 40 — smallest ≥ 35

        // Subsets
        System.out.println(numbers.headSet(40));         // [10, 20]     — exclusive
        System.out.println(numbers.tailSet(40));         // [40, 50, 80] — inclusive
        System.out.println(numbers.subSet(20, true, 60, false)); // [20, 40, 50]
    }
}

4. Map — Key-Value Pairs

A Map associates unique keys to values. Keys must be unique; values may repeat. The three canonical implementations mirror the Set family.

4.1 HashMap

The go-to general-purpose map. Average O(1) get and put. No ordering guarantee. Allows one null key and multiple null values. 

import java.util.HashMap;
import java.util.Map;

public class HashMapDemo {
    public static void main(String[] args) {
        Map<String, Integer> scores = new HashMap<>();

        // Put / update
        scores.put("Alice", 95);
        scores.put("Bob",   88);
        scores.put("Carol", 72);
        scores.put("Alice", 97); // updates existing key

        // Read
        System.out.println(scores.get("Alice"));            // 97
        System.out.println(scores.getOrDefault("Dave", 0)); // 0

        // Conditional put — only if key absent
        scores.putIfAbsent("Dave", 85);

        // Compute patterns
        scores.compute("Bob", (k, v) -> v == null ? 1 : v + 10); // 98
        scores.merge("Carol", 5, Integer::sum);                    // 77

        // Check
        System.out.println(scores.containsKey("Bob"));   // true
        System.out.println(scores.containsValue(100));   // false

        // Remove
        scores.remove("Carol");
        scores.remove("Bob", 99); // removes only if value matches

        System.out.println(scores);
    }
}

4.2 Iterating a Map

Map<String, Integer> map = Map.of("A", 1, "B", 2, "C", 3);

// 1. Entry set — most common, access both key and value
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.printf("%s → %d%n", entry.getKey(), entry.getValue());
}

// 2. Key set
for (String key : map.keySet()) {
    System.out.println(key + " = " + map.get(key));
}

// 3. Values only
for (int value : map.values()) {
    System.out.println(value);
}

// 4. forEach lambda (cleanest)
map.forEach((k, v) -> System.out.println(k + ": " + v));

4.3 LinkedHashMap

Maintains insertion order (or optionally access order, useful for LRU caches). Slightly higher memory cost but preserves predictable iteration. 

import java.util.LinkedHashMap;
import java.util.Map;

public class LinkedHashMapDemo {
    public static void main(String[] args) {
        Map<String, String> capitals = new LinkedHashMap<>();
        capitals.put("France",  "Paris");
        capitals.put("Germany", "Berlin");
        capitals.put("Japan",   "Tokyo");

        // Iteration order == insertion order
        capitals.forEach((country, capital) ->
            System.out.println(country + " → " + capital));
        // France → Paris
        // Germany → Berlin
        // Japan → Tokyo
    }

    // LRU Cache example (access-order LinkedHashMap)
    static <K, V> Map<K, V> lruCache(int capacity) {
        return new LinkedHashMap<K, V>(capacity, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
                return size() > capacity;
            }
        };
    }
}

4.4 TreeMap

Backed by a Red-Black tree; keys are kept in sorted order. O(log n) operations. Implements NavigableMap — exposes firstKey(), lastKey(), floorKey(), ceilingKey(), headMap(), and tailMap(). 

import java.util.TreeMap;

public class TreeMapDemo {
    public static void main(String[] args) {
        TreeMap<String, Integer> wordCount = new TreeMap<>();
        wordCount.put("banana", 3);
        wordCount.put("apple",  7);
        wordCount.put("cherry", 2);
        wordCount.put("date",   5);

        // Keys in natural (alphabetical) order
        System.out.println(wordCount.keySet()); // [apple, banana, cherry, date]

        System.out.println(wordCount.firstKey());        // apple
        System.out.println(wordCount.lastKey());         // date
        System.out.println(wordCount.floorKey("c"));     // banana
        System.out.println(wordCount.ceilingKey("c"));   // cherry

        // Descending view
        wordCount.descendingMap().forEach((k, v) ->
            System.out.println(k + ": " + v));
    }
}

5. Queue and Deque

5.1 Queue — FIFO

Queue<E> models a first-in, first-out line. Use the pair-method API — the throwing version (add, remove, element) vs the safe version (offer, poll, peek). 

import java.util.LinkedList;
import java.util.Queue;

public class QueueDemo {
    public static void main(String[] args) {
        Queue<String> queue = new LinkedList<>();

        // Enqueue
        queue.offer("Task-1");
        queue.offer("Task-2");
        queue.offer("Task-3");

        // Peek — inspect head without removing
        System.out.println(queue.peek());  // Task-1

        // Dequeue — remove head
        while (!queue.isEmpty()) {
            System.out.println("Processing: " + queue.poll());
        }
        // Processing: Task-1
        // Processing: Task-2
        // Processing: Task-3
    }
}

5.2 PriorityQueue

Elements are dequeued in priority order (natural ordering or a custom Comparator), not insertion order. Internally a min-heap. 

import java.util.PriorityQueue;
import java.util.Comparator;

public class PriorityQueueDemo {
    public static void main(String[] args) {
        // Min-heap by default (lowest integer first)
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();
        minHeap.offer(30);
        minHeap.offer(10);
        minHeap.offer(20);

        while (!minHeap.isEmpty()) {
            System.out.print(minHeap.poll() + " "); // 10 20 30
        }

        // Max-heap with Comparator.reverseOrder()
        PriorityQueue<Integer> maxHeap =
            new PriorityQueue<>(Comparator.reverseOrder());
        maxHeap.offer(30);
        maxHeap.offer(10);
        maxHeap.offer(20);

        while (!maxHeap.isEmpty()) {
            System.out.print(maxHeap.poll() + " "); // 30 20 10
        }
    }
}

5.3 Deque — Double-Ended Queue

Deque<E> supports insertion and removal at both ends. ArrayDeque is the preferred implementation — faster than LinkedList as a deque and also doubles as a stack. 

import java.util.ArrayDeque;
import java.util.Deque;

public class DequeDemo {
    public static void main(String[] args) {
        Deque<String> deque = new ArrayDeque<>();

        // Add at both ends
        deque.offerFirst("B");
        deque.offerFirst("A");  // [A, B]
        deque.offerLast("C");   // [A, B, C]

        System.out.println(deque.peekFirst()); // A
        System.out.println(deque.peekLast());  // C

        deque.pollFirst(); // removes A → [B, C]
        deque.pollLast();  // removes C → [B]

        // Use as a stack
        Deque<Integer> stack = new ArrayDeque<>();
        stack.push(1);  // addFirst
        stack.push(2);
        stack.push(3);
        System.out.println(stack.pop());  // 3 — LIFO
        System.out.println(stack.pop());  // 2
    }
}

6. Iterating Collections

Java offers several ways to walk through a collection. Choose the style that best fits your use case.

6.1 Enhanced for-each

The simplest and most readable form. Works on anything that is Iterable.

List<String> names = List.of("Alice", "Bob", "Carol");

for (String name : names) {
    System.out.println(name);
}

6.2 Iterator

Use the explicit Iterator when you need to remove elements during iteration (the for-each loop does not support this safely). 

import java.util.ArrayList;
import java.util.Iterator;
import java.util.List;

public class IteratorDemo {
    public static void main(String[] args) {
        List<Integer> nums = new ArrayList<>(List.of(1, 2, 3, 4, 5, 6));
        Iterator<Integer> it = nums.iterator();

        while (it.hasNext()) {
            int n = it.next();
            if (n % 2 == 0) {
                it.remove(); // safe removal during iteration
            }
        }

        System.out.println(nums); // [1, 3, 5]
    }
}

6.3 forEach with Lambda

Clean and concise. Defined on Iterable and available on every collection. Cannot break early (use a stream with findFirst() / anyMatch() for that). 

List<String> fruits = List.of("Apple", "Banana", "Cherry");

// Basic lambda
fruits.forEach(f -> System.out.println(f));

// Method reference — identical behaviour
fruits.forEach(System.out::println);

// On a Map
Map<String, Integer> prices = Map.of("Apple", 3, "Banana", 1);
prices.forEach((fruit, price) ->
    System.out.printf("%s costs $%d%n", fruit, price));

6.4 ListIterator — Bidirectional

Only available on List. Allows forward and backward traversal, plus in-place set() and add(). 

import java.util.ArrayList;
import java.util.List;
import java.util.ListIterator;

public class ListIteratorDemo {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>(List.of("a", "b", "c"));
        ListIterator<String> li = list.listIterator(list.size()); // start at end

        // Iterate backwards
        while (li.hasPrevious()) {
            System.out.print(li.previous() + " "); // c b a
        }

        // Forward iteration with in-place replacement
        ListIterator<String> li2 = list.listIterator();
        while (li2.hasNext()) {
            li2.set(li2.next().toUpperCase());
        }
        System.out.println(list); // [A, B, C]
    }
}

7. Sorting — Comparable and Comparator

7.1 Comparable — Natural Ordering

Implement Comparable<T> in your class to define its natural ordering. Collections.sort() and TreeSet / TreeMap use this by default. 

public class Student implements Comparable<Student> {
    private final String name;
    private final double gpa;

    public Student(String name, double gpa) {
        this.name = name;
        this.gpa  = gpa;
    }

    // Natural order: descending GPA
    @Override
    public int compareTo(Student other) {
        return Double.compare(other.gpa, this.gpa); // reversed for highest-first
    }

    @Override
    public String toString() {
        return name + "(" + gpa + ")";
    }
}

// Usage
import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

List<Student> students = new ArrayList<>();
students.add(new Student("Alice", 3.8));
students.add(new Student("Bob",   3.5));
students.add(new Student("Carol", 3.9));

Collections.sort(students);
System.out.println(students); // [Carol(3.9), Alice(3.8), Bob(3.5)]

7.2 Comparator — External / Ad-Hoc Ordering

Comparator<T> lets you define multiple orderings without touching the class. Since Java 8, you can build sophisticated comparators using the fluent factory methods on Comparator. 

import java.util.ArrayList;
import java.util.Comparator;
import java.util.List;

public class ComparatorDemo {
    record Person(String name, int age, String city) {}

    public static void main(String[] args) {
        List<Person> people = new ArrayList<>(List.of(
            new Person("Alice", 30, "New York"),
            new Person("Bob",   25, "Boston"),
            new Person("Carol", 30, "Austin"),
            new Person("Dave",  25, "Denver")
        ));

        // Sort by age ascending, then name ascending
        Comparator<Person> byAgeTheByName =
            Comparator.comparingInt(Person::age)
                      .thenComparing(Person::name);

        people.sort(byAgeTheByName);
        people.forEach(System.out::println);
        // Person[name=Bob,  age=25, city=Boston]
        // Person[name=Dave, age=25, city=Denver]
        // Person[name=Alice, age=30, city=New York]
        // Person[name=Carol, age=30, city=Austin]

        // Reversed: oldest first
        people.sort(Comparator.comparingInt(Person::age).reversed());

        // Sort strings case-insensitively
        List<String> names = new ArrayList<>(List.of("banana", "Apple", "cherry"));
        names.sort(String.CASE_INSENSITIVE_ORDER);
        System.out.println(names); // [Apple, banana, cherry]
    }
}

7.3 Sorting Arrays

import java.util.Arrays;

int[]    ints   = {5, 3, 8, 1};
String[] words  = {"banana", "apple", "cherry"};

Arrays.sort(ints);                                    // [1, 3, 5, 8]
Arrays.sort(words);                                   // [apple, banana, cherry]
Arrays.sort(words, Comparator.reverseOrder());        // [cherry, banana, apple]
Arrays.sort(words, 0, 2, String.CASE_INSENSITIVE_ORDER); // partial range sort

8. Collections Utility Class

java.util.Collections is a utility class (all static methods) that acts as an algorithm library for collections.

8.1 Sorting and Searching

import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

List<Integer> nums = new ArrayList<>(List.of(3, 1, 4, 1, 5, 9, 2, 6));

Collections.sort(nums);                          // [1, 1, 2, 3, 4, 5, 6, 9]
Collections.sort(nums, Comparator.reverseOrder()); // descending

// Binary search — list MUST be sorted first
Collections.sort(nums);
int idx = Collections.binarySearch(nums, 5);    // index of 5

8.2 Min, Max, Frequency, Disjoint

List<Integer> data = List.of(3, 7, 2, 9, 4, 7, 7);

System.out.println(Collections.min(data));         // 2
System.out.println(Collections.max(data));         // 9
System.out.println(Collections.frequency(data, 7)); // 3

List<String> a = List.of("cat", "dog");
List<String> b = List.of("fish", "bird");
System.out.println(Collections.disjoint(a, b));    // true — no common elements

8.3 Shuffle, Reverse, Rotate, Swap

List<Integer> list = new ArrayList<>(List.of(1, 2, 3, 4, 5));

Collections.shuffle(list);                   // random permutation
Collections.reverse(list);                   // reverses in place
Collections.rotate(list, 2);                 // rotates right by 2 positions
Collections.swap(list, 0, list.size() - 1); // swap first and last
Collections.fill(list, 0);                   // fill all with 0

8.4 Unmodifiable and Synchronized Wrappers

import java.util.Collections;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

// Unmodifiable wrappers — defensive copies for API boundaries
List<String> mutable = new ArrayList<>(List.of("a", "b", "c"));
List<String> readOnly = Collections.unmodifiableList(mutable);
// readOnly.add("d"); // throws UnsupportedOperationException

// Synchronized wrappers — for basic thread safety
// (prefer java.util.concurrent classes for real concurrency)
Map<String, Integer> syncMap =
    Collections.synchronizedMap(new HashMap<>());

// Singleton and empty collections — efficient immutable stubs
List<String> single  = Collections.singletonList("only");
List<String> empty   = Collections.emptyList();
Map<String, Integer> emptyMap = Collections.emptyMap();

8.5 nCopies and addAll

// Create a list of n identical elements
List<String> zeros = Collections.nCopies(5, "zero");
System.out.println(zeros); // [zero, zero, zero, zero, zero]

// Bulk-add array elements to a collection
List<String> target = new ArrayList<>();
Collections.addAll(target, "x", "y", "z");

9. Immutable Collections (Java 9+)

Java 9 introduced concise factory methods on List, Set, and Map for creating compact, immutable collections. These are ideal for constants, test data, and read-only API results. 

// Immutable List — preserves order, allows duplicates
List<String> colors = List.of("Red", "Green", "Blue");

// Immutable Set — no duplicates, no guaranteed order
Set<Integer> primes = Set.of(2, 3, 5, 7, 11);

// Immutable Map — up to 10 pairs via Map.of(), more via Map.ofEntries()
Map<String, String> iso = Map.of(
    "US", "United States",
    "DE", "Germany",
    "JP", "Japan"
);

// For more than 10 entries:
Map<String, Integer> large = Map.ofEntries(
    Map.entry("a", 1),
    Map.entry("b", 2),
    Map.entry("c", 3)
    // ... up to any size
);

// Mutations throw UnsupportedOperationException
// colors.add("Yellow"); // ❌ throws

// Copying to a mutable collection
List<String> mutableColors = new ArrayList<>(colors);

10. Choosing the Right Collection

Need Best Choice Why
General ordered list ArrayList O(1) random access, compact memory
Frequent head/tail inserts ArrayDeque O(1) at both ends, better cache than LinkedList
Unique elements, O(1) ops HashSet Hash-based, fastest membership test
Unique + insertion order LinkedHashSet Ordered iteration with O(1) ops
Sorted unique elements TreeSet O(log n), range queries via NavigableSet
Key-value, O(1) ops HashMap General-purpose map
Key-value + insertion order LinkedHashMap Predictable iteration; LRU cache pattern
Sorted key-value TreeMap O(log n), range queries via NavigableMap
Priority-based processing PriorityQueue Min-heap; configurable via Comparator
Stack (LIFO) ArrayDeque Prefer over legacy Stack class
Immutable constant data List.of() / Set.of() Thread-safe, minimal overhead (Java 9+)

Summary

  • The JCF revolves around two root interfaces — Collection and Map.
  • Use ArrayList by default for lists; switch to ArrayDeque for stack/queue behaviour.
  • HashSet for fast uniqueness checks, TreeSet when you need sorted order.
  • HashMap for general key-value storage; TreeMap for sorted keys.
  • Prefer forEach with lambdas for clean iteration; use Iterator.remove() when modifying during traversal.
  • Implement Comparable for natural order; use Comparator for flexible, ad-hoc ordering.
  • The Collections utility class provides algorithms (sort, shuffle, min, max, frequency) and wrappers (unmodifiable, synchronized).
  • Use List.of(), Set.of(), and Map.of() for concise immutable collections in Java 9+.
