Classes, Objects & Constructors 🧬
Java is an Object-Oriented Programming (OOP) language. Everything in Java is associated with classes and objects, along with its attributes and methods.

A Class is a blueprint or template. It defines the state (fields/variables) and behavior (methods) of a type. An Object is a physical instance of a class, taking up memory on the Heap.

A Constructor is a special method used to initialize objects. It is called when an object of a class is created using the new keyword. It must have the exact same name as the class and no return type.

java — Class & Constructor Example
public class Car {
    // Fields (State)
    String brand;
    int year;

    // Default Constructor (Provided implicitly if no constructors exist, but explicit here)
    public Car() {
        this.brand = "Unknown";
        this.year = 0;
    }

    // Parameterized Constructor
    public Car(String brand, int year) {
        // 'this' refers to the current object instance
        this.brand = brand;
        this.year = year;
    }

    // Method (Behavior)
    public void startEngine() {
        System.out.println(brand + " engine started!");
    }
}

// Creating objects
Car myCar = new Car("Toyota", 2023);
myCar.startEngine();
Encapsulation 🔒
The mechanism of wrapping the data (variables) and code acting on the data (methods) together as a single unit. In encapsulation, the variables of a class are hidden from other classes, preventing direct access.

To achieve encapsulation in Java: declare the variables of a class as private, and provide public setter and getter methods to modify and view the variables values.

This provides Data Hiding. The external world doesn't know *how* the class stores its data; it only knows the public interface to interact with it. This allows you to add validation logic inside setters later without breaking external code.

Access Modifiers
private: Accessible ONLY within the same class.
(default): Accessible within the same package.
protected: Accessible within the same package AND subclasses in different packages.
public: Accessible everywhere.
Why Encapsulate?
If balance is public, anyone can write account.balance = -10000;.

By making it private and using a setter, you can add rules:
if (amount > 0) { this.balance += amount; }
Inheritance 🦋
Inheritance is a mechanism where a new class is derived from an existing class. It promotes code reusability and establishes an IS-A relationship.

In Java, we use the extends keyword to inherit from a class. The class being inherited from is the Superclass (or parent), and the class inheriting is the Subclass (or child).

A subclass inherits all non-private members of the superclass. Every class in Java implicitly inherits from java.lang.Object.

java — Extending Classes & super keyword
class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    public void makeSound() {
        System.out.println("Some generic animal sound");
    }
}

// Dog IS-A Animal
class Dog extends Animal {
    
    public Dog(String name) {
        // 'super' calls the constructor of the parent class
        // It MUST be the first statement in the constructor
        super(name); 
    }

    // Method Overriding
    @Override
    public void makeSound() {
        System.out.println(name + " barks: Woof!");
    }
}
Multiple Inheritance in Java Java does NOT support multiple inheritance with classes (a class cannot extend more than one class). This avoids the "Diamond Problem" of ambiguity. However, a class *can* implement multiple interfaces.
Polymorphism 🎭
Polymorphism means "many forms". It allows us to perform a single action in different ways. In Java, this is achieved via method overloading and overriding.
Compile-Time (Static)
Achieved via Method Overloading.

Multiple methods in the same class have the same name but different parameters (type, number, or order).

The compiler decides which method to call based on the arguments passed.
Runtime (Dynamic)
Achieved via Method Overriding and Upcasting.

A subclass provides a specific implementation of a method that is already provided by its parent class.

The JVM decides which method to execute at runtime based on the actual object created, not the reference variable type.
java — Dynamic Method Dispatch
Animal myPet = new Dog("Buddy"); // Upcasting

// At compile time, the compiler checks if 'makeSound' exists in Animal. It does.
// At runtime, the JVM looks at the ACTUAL object (Dog) and calls Dog's overridden method.
myPet.makeSound(); // Outputs: Buddy barks: Woof!
Abstraction ☁️
Abstraction is the process of hiding the implementation details and showing only functionality to the user. It lets you focus on what the object does instead of how it does it.

In Java, abstraction is achieved using Abstract classes and Interfaces.

An abstract class (declared with the abstract keyword) cannot be instantiated (you cannot use new on it). It can have both abstract methods (methods without a body) and regular methods. It serves as a strict template for subclasses.

java — Abstract Classes
abstract class Shape {
    String color;
    
    // Abstract method: No body. Subclasses MUST implement this.
    abstract double getArea();

    // Concrete method: Shared by all subclasses.
    public void setColor(String color) {
        this.color = color;
    }
}

class Circle extends Shape {
    double radius;
    
    public Circle(double radius) { this.radius = radius; }

    @Override
    double getArea() {
        return Math.PI * radius * radius;
    }
}
Interfaces 🔌
An interface is a completely "abstract class" that is used to group related methods with empty bodies. It establishes a contract: any class implementing the interface MUST fulfill the contract.

A class uses the implements keyword to inherit an interface. Unlike classes, a single Java class can implement multiple interfaces, bypassing the single-inheritance restriction.

Historically, interfaces could only have abstract methods and constants (public static final). Since Java 8, interfaces can also have default and static methods with concrete implementations.

java — Interfaces & Default Methods
interface PaymentProcessor {
    // Implicitly public and abstract
    void processPayment(double amount);

    // Default method (Java 8+). Provides a fallback implementation.
    default void issueReceipt() {
        System.out.println("Receipt sent via email.");
    }
}

interface Refundable {
    void processRefund(double amount);
}

// Implementing multiple interfaces
class StripeGateway implements PaymentProcessor, Refundable {
    @Override
    public void processPayment(double amount) {
        System.out.println("Processing $" + amount + " via Stripe");
    }

    @Override
    public void processRefund(double amount) {
        System.out.println("Refunding $" + amount + " via Stripe");
    }
}
Why did Java 8 add Default Methods? They were added to support backward compatibility. When Java developers wanted to add the .stream() method to the Collection interface, adding an abstract method would have broken millions of custom classes worldwide that implement Collection. By making it a default method, old code still compiled without needing modifications!
Interview Prep
OOP questions are guaranteed in any Java interview. Focus on the subtle differences between similar concepts.
Q1 What is the difference between an Abstract Class and an Interface? Medium ▼

1. Inheritance: A class can implement multiple interfaces, but can extend only one abstract class.

2. State: Abstract classes can have instance variables (state) and constructors. Interfaces cannot have instance variables (only public static final constants) and cannot have constructors.

3. Use Case: Use an Abstract Class when you want to share code among closely related classes (an IS-A relationship with shared state). Use an Interface to define a contract or capability (a CAN-DO relationship) across completely unrelated classes.

Q2 Can you override a static method in Java? Hard ▼

No. You cannot override static methods. Static methods are bound to the class, not the instance.

If a subclass defines a static method with the exact same signature as a static method in the superclass, the subclass method hides the superclass method. This is called Method Hiding, not overriding.

Because it's resolved at compile-time (early binding), polymorphism does not apply. If you call the static method via a superclass reference holding a subclass object, the superclass's static method will execute.

Q3 What is the difference between Overloading and Overriding? Easy ▼

Overloading happens in the same class. Methods have the same name but different parameters. It is an example of compile-time polymorphism.

Overriding happens between a superclass and subclass. The subclass method has the exact same name, return type, and parameters as the superclass method. It is an example of runtime polymorphism.

The Abstract Parent Class
Create an abstract BankAccount class. Notice the encapsulation on the balance field.
abstract class BankAccount { private String accountNumber; protected double balance; // protected allows subclasses to access it public BankAccount(String accountNumber, double initialBalance) { this.accountNumber = accountNumber; this.balance = initialBalance; } public double getBalance() { return balance; } public String getAccountNumber() { return accountNumber; } public void deposit(double amount) { if (amount > 0) balance += amount; } // Abstract method forces subclasses to define withdrawal rules public abstract void withdraw(double amount); }
2
The Concrete Subclass
Extend the abstract class and implement the abstract method.
class SavingsAccount extends BankAccount { private double interestRate; public SavingsAccount(String accNum, double balance, double rate) { super(accNum, balance); // Initialize parent fields this.interestRate = rate; } @Override public void withdraw(double amount) { if (amount > 0 && balance - amount >= 0) { balance -= amount; } else { System.out.println("Insufficient funds!"); } } }
