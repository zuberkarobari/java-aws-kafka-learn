19. Docker & Kubernetes

How does a Spring Boot JAR running on your laptop become a globally scaled, self-healing service serving millions of requests? Containers and Kubernetes.

1. Docker vs Kubernetes

Docker

Packages your Spring Boot application and ALL its dependencies (Java runtime, libraries) into a single, portable Container image. "It works on my machine" becomes "it works on every machine". A Docker container is like a standardized shipping container — same box fits on any truck or ship.

Kubernetes (K8s)

The container orchestrator. If Docker is the shipping container, Kubernetes is the entire port logistics system. It decides which physical machines (Nodes) to run the containers on, restarts crashed containers, scales up during traffic spikes, and rolls out new deployments with zero downtime.

2. Kubernetes Architecture

In production, the Kubernetes cluster consists of Control Plane (brain) and Worker Nodes (muscles). Your Spring Boot container runs inside a Pod on a Worker Node.

Kubernetes Cluster

Control Plane (Brain)
API Server
The Gateway for all K8s operations. `kubectl` talks to this.
Scheduler
Decides which Node has spare CPU/RAM to run a new Pod.
Controller Manager
Ensures the actual state matches desired state. If 3 replicas requested but only 2 running, it creates a 3rd.
Worker Node (Muscle)
Pod
The smallest K8s unit. Wraps 1+ containers. Has its own IP address.
Kubelet (Agent)
Reports to Control Plane. Starts/Stops containers on this node.
Kube-proxy
Handles network routing between pods. Implements K8s Services.
order-service-deployment.yaml 
apiVersion: apps/v1
kind: Deployment
spec:
  replicas: 3   # Keep 3 pods running ALWAYS
  template:
    spec:
      containers:
      - name: order-service
        image: myregistry/order-service:v2.1.0
        ports:
        - containerPort: 8080

3. Rolling Deployments (Zero Downtime)

Blue-Green Deployment with K8s

How Uber pushes code 100 times a day without any downtime.

1

Start with 3 Pods on v1

All 3 pods are running the old version. 100% of traffic hits v1.

2

Rolling Replace: One at a time

K8s kills Pod-1 (v1), starts Pod-1 (v2). Now 2 Pods on v1, 1 Pod on v2. Traffic continues uninterrupted across the 2 remaining v1 pods. K8s waits for v2 to pass its readiness probe before continuing.

3

Complete: All 3 Pods on v2

Zero downtime. Users never experienced an outage. If v2 fails health checks at any step, K8s automatically rolls back to v1.

A Container is a running instance of that image — like an object instantiated from a class. You can run 50 containers from the same image simultaneously. Each container has its own writable layer on top and runs as an isolated process.
