1. The Browser

What exactly happens inside the browser engine when a user types a URL and hits enter? Let's break down the DOM, CSSOM, Render Tree, and network initialization.

1. Definition

Simple English

The browser is the translator between human intentions and computer networks. It takes a typed address, fetches the corresponding code from across the world, and paints it on your screen as a beautiful, interactive page.

Technical

A software application consisting of a UI, a browser engine, a rendering engine (like Blink or WebKit), a networking layer, and a JavaScript engine (like V8) that retrieves, presents, and traverses information resources on the WWW.

Purpose

To provide a secure sandbox for executing untrusted code (JS) while rendering standardized markup (HTML/CSS) into an interactive graphical interface, handling all underlying network protocols transparently.

Amazon Homepage Load

Why frontend optimization dictates revenue.

The Parsing Blockers

When Amazon's HTML arrives, the browser starts parsing. If it encounters a synchronous `<script src="...">`, DOM parsing halts completely. The browser must download and execute the JS before continuing. This is why Amazon puts non-critical scripts at the bottom or uses `defer`/`async` attributes—to ensure the user sees products immediately.

Reflow Costs

If a user resizes their browser window, or if a carousel image loads dynamically pushing content down, the browser must recalculate positions (Reflow) and redraw pixels (Repaint). Amazon minimizes layout shifts to keep the CPU cool and the UI buttery smooth.

4. Caching & Storage

Cookies

Max 4KB. Sent to the server automatically with every HTTP request. Used for session management (Session IDs) and tracking. Secured via `HttpOnly` and `Secure` flags.

Local Storage

~5MB capacity. Persists even after the browser is closed. Never sent to the server automatically. Accessed via JavaScript. Ideal for user preferences (e.g., dark mode).

Session Storage

~5MB capacity. Cleared as soon as the specific browser tab is closed. Ideal for temporary state like multi-step form data.

Browser Cache

Stores HTTP responses (images, CSS, JS) based on HTTP headers (`Cache-Control`, `ETag`). Prevents re-downloading static assets, drastically improving load times.
