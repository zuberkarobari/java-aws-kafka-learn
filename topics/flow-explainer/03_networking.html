3. Internet & Networking

Now that we have the IP address, how does the actual data travel across the ocean? Welcome to TCP/IP, Sockets, and the 3-Way Handshake.

1. Definition

Simple English

Imagine sending a 1,000-page book in the mail. You can't fit it in one envelope, so you tear out the pages, number them, put them in 1,000 envelopes, and mail them. The receiver reassembles them in order. That's TCP/IP.

Technical

The TCP/IP suite is the foundational communication protocol of the Internet. The IP layer routes packets from source IP to destination IP. The TCP layer ensures reliable, ordered, and error-checked delivery of a stream of octets.

Purpose

To provide a standardized way for disparate hardware systems across the globe (routers, switches, fiber optic cables) to speak a common language and guarantee data delivery.

2. Internal Working: TCP Handshake

Before any HTTP data (like your HTML file) can be sent, the browser and server must establish a reliable connection. This is the famous TCP 3-Way Handshake.

The TCP 3-Way Handshake

Client
1. SYN (Sequence: X)
"Hey, are you there? Let's connect."
Server
2. SYN-ACK (Seq: Y, Ack: X+1)
"Yes, I'm here! Acknowledged X. Here is my sequence."
3. ACK (Ack: Y+1)
"Acknowledged Y. Connection established."
Connection Established
Now the actual HTTP Request (GET /index.html) can be sent over this TCP socket.

Answer: TCP (Transmission Control Protocol) is connection-oriented, reliable, and ordered. If a packet drops, TCP retransmits it. It is used for Web (HTTP), Email (SMTP), and File Transfer (FTP).

UDP (User Datagram Protocol) is connectionless and unreliable ("fire and forget"). It does not check if packets arrived or re-order them. Because it skips the 3-way handshake overhead, it is faster and used for Live Video Streaming, Online Gaming, and DNS queries.

Answer: A Socket is one endpoint of a two-way communication link between two programs running on the network. It is formed by concatenating an IP Address and a Port Number.

For example, if the server is 142.250.190.46, and the web server process is listening on Port 443 (HTTPS), the socket address is `142.250.190.46:443`. Sockets allow the Operating System to route incoming packets not just to the correct computer, but to the correct specific application running on that computer.
