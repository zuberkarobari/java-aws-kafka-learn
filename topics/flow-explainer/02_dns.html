2. DNS Resolution

How the internet translates human-readable domain names into machine-readable IP addresses using a globally distributed hierarchy.

1. Definition

Simple English

DNS is the phonebook of the internet. You memorize a name (google.com), but computers need a number (IP Address like 142.250.190.46) to dial the connection. DNS makes the translation.

Technical

A hierarchical, decentralized naming system that resolves human-readable hostnames to IP addresses. It uses UDP port 53 for queries, falling back to TCP if the response is too large.

Purpose

To decouple logical names from physical infrastructure. Servers change IP addresses frequently due to load balancing or failure; DNS ensures the user's bookmark never breaks.

2. Internal Working: The DNS Lookup Flow

If the IP is not in the browser, OS, or router cache, your computer contacts your ISP's Recursive Resolver to do the heavy lifting.

3. Real-World Cloud Usage

AWS Route 53 (Netflix/Uber)

DNS isn't just for IP translation—it's a global traffic controller.

Latency-Based Routing

If a user in Tokyo requests netflix.com, Route 53 (the Authoritative Server) looks at their location. Instead of returning the IP of a US server, it returns the IP of an AWS region in Tokyo, ensuring minimal latency.

Failover Routing (Active-Passive)

If Uber's primary data center goes down, Route 53 detects failed health checks. It immediately stops returning the primary IP and starts returning the disaster-recovery IP. Users experience a minor blip instead of a full outage.

Answer: Time To Live (TTL) dictates how long DNS resolvers (like your ISP) should cache a record before querying the authoritative server again.

If you set a high TTL (e.g., 24 hours), it reduces load on your DNS servers and speeds up user requests. However, the danger is during an emergency: if your server crashes and you switch your IP to a backup server, users worldwide won't see the change for up to 24 hours because their ISPs are holding the cached (old) IP address. Production applications often keep TTLs low (e.g., 60 seconds) for dynamic records.
