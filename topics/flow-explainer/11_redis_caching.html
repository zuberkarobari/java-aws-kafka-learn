11. Redis Caching

Why hit the slow hard drive of a database when you can fetch data from lightning-fast RAM? Learn how Redis saves the database from crashing.

1. Definition

Simple English

If someone asks you what 342 * 987 is, you have to do the math (hitting the database). But if they ask you 50 times a day, you just memorize the answer and tell them instantly (caching).

Technical

Redis is an open-source, in-memory data structure store used as a database, cache, message broker, and streaming engine. It stores data in RAM (Random Access Memory) rather than on an SSD/HDD.

Purpose

To drastically reduce latency (from ~10ms down to ~1ms) and reduce the load on the primary SQL database for read-heavy operations, session storage, and rate-limiting.

2. The Cache Aside Pattern

The most common pattern in Spring Boot is "Cache Aside". The application code is responsible for checking the cache first before hitting the database.

3. Interview Questions

Answer: Spring provides an abstraction layer.
1. Add @EnableCaching to your main application class.
2. Add @Cacheable(value = "users", key = "#id") to your repository or service method.

Spring will automatically intercept the method call. If the key exists in Redis, it returns it instantly (bypassing the method). If not, it runs the method (hitting the DB), puts the result in Redis, and returns it.

Answer:
Cache Penetration: A hacker constantly queries an ID that doesn't exist in the cache OR the DB (e.g., ID=-999). Every request misses the cache and hits the DB, potentially crashing it. Fix: Cache null values with a short TTL, or use a Bloom Filter.

Cache Breakdown (Thundering Herd): A highly popular key (e.g., a viral tweet) suddenly expires. In that exact millisecond, 10,000 requests hit the cache, see a miss, and all 10,000 hit the database simultaneously to regenerate the cache. Fix: Use a Mutex lock so only the first request hits the DB while others wait.
