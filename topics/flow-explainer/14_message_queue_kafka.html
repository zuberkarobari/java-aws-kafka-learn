14. Message Queue (Kafka)

When processing an order takes 5 seconds, the user will leave. Instead, we drop the task in a queue, return a success message instantly, and process it later.

1. Definition

Simple English

At a busy coffee shop, the cashier doesn't make your coffee. They take your order, write it on a cup, put it in a queue, and shout "Next!". The barista reads the queue and makes the coffee. Kafka is that queue.

Technical

Apache Kafka is a distributed event streaming platform. It acts as an immutable append-only log that allows systems to produce (write) and consume (read) massive streams of records in real-time, completely decoupling the sender from the receiver.

Purpose

To enable asynchronous processing. It prevents the user interface from blocking while slow tasks (sending emails, generating PDFs, charging credit cards) happen in the background.

2. Decoupling Microservices

Instead of the Order Service calling the Email Service via a slow REST API, it drops an event into Kafka and forgets about it.

3. Interview Questions

Answer: RabbitMQ is a traditional "smart broker, dumb consumer" message queue. It pushes messages to consumers. Once a consumer acknowledges a message, RabbitMQ deletes it from the queue forever.

Kafka is a "dumb broker, smart consumer" event streaming platform. It acts as an immutable log file. It does not push; consumers must Poll/Pull. When a consumer reads a message, Kafka does NOT delete it. The message stays on disk for a configured retention period (e.g., 7 days). This allows new microservices to spin up and replay history from the beginning of time.

If enable.auto.commit=true (default), the consumer might commit the offset before it finishes sending the email. If it crashes, the message is lost forever because Kafka thinks it was processed.

In production, we set auto-commit to false and use Manual Acknowledgment. The Email Service reads the message, actually sends the email, and *then* manually commits the offset. If it crashes mid-process, the offset is never committed. When the service reboots, it will read the same message again. (Note: This requires your services to be Idempotent).
