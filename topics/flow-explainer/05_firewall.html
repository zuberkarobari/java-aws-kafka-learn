5. Network Firewall

Before the request even reaches the application server, it must pass through the bouncers: WAFs, Security Groups, and NACLs.

1. Definition

Simple English

A firewall is the bouncer at an exclusive club. It checks everyone's ID (IP address, Port, HTTP headers) against a VIP list. If you aren't on the list, or if you look suspicious, you are dropped immediately.

Technical

A network security device or software that monitors and filters incoming and outgoing network traffic based on predetermined security rules. In modern clouds, this is handled by Security Groups (L4) and Web Application Firewalls (L7).

Purpose

To establish a barrier between your secure internal networks and outside untrusted networks (the Internet), blocking DDoS attacks, SQL injections, and unauthorized port access.

2. Security Layers in Production

Enterprise applications use a defense-in-depth strategy. A request must pass through multiple firewall layers before it ever reaches the Spring Boot code.

Layer 7: Web Application Firewall (WAF)
Inspects the actual HTTP content. Blocks SQL Injections, Cross-Site Scripting (XSS), and malicious bots before they hit the load balancer.
Layer 4: Network ACL (Subnet Level)
Stateless firewall for the entire subnet. E.g., explicitly DENY all traffic from a known block of malicious IP addresses.
Layer 4: Security Group (Instance Level)
Stateful firewall acting directly on the EC2 instance. E.g., ALLOW incoming TCP traffic ONLY on Port 443 (HTTPS) and Port 22 (SSH).

3. Interview Questions

Answer: A Layer 4 firewall (like AWS Security Groups) only looks at IPs and Ports. It can block traffic from an IP, but it doesn't know what data is inside the packets.

A Layer 7 firewall (WAF) unpacks the HTTP request and reads the actual data. It can look at the URL, the HTTP Headers, and the JSON body. If it sees `SELECT * FROM Users` hidden inside a login form (SQL Injection), the WAF blocks it, whereas a Layer 4 firewall would let it through because the port (443) was open.

Answer: A Distributed Denial of Service (DDoS) attack is when an attacker uses thousands of compromised computers (a botnet) to flood a server with fake traffic, exhausting its CPU or bandwidth so real users can't connect.

To protect against it, you use a CDN (like Cloudflare) or a managed DDoS protection service (like AWS Shield) at the edge of the network. Because they have massive bandwidth capacity, they absorb the flood of fake requests and use WAF rules/rate-limiting to drop malicious packets before they ever reach your origin Spring Boot servers.
