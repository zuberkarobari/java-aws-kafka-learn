12. Database

The heart of the application. If it isn't in the cache, we go to the source of truth. Let's understand ACID, B-Trees, and Master-Slave replication.

1. Definition

Simple English

A database is an electronic filing cabinet. Instead of tossing papers in randomly, it organizes them in folders with strict rules, indexes them alphabetically so you can find them in seconds, and keeps them locked in a fireproof safe.

Technical

A Relational Database Management System (RDBMS) like PostgreSQL stores data in structured tables using rows and columns, enforcing strict schemas and maintaining ACID properties to ensure data integrity during hardware failures.

Purpose

To provide durable, persistent storage. RAM clears when power is lost; hard drives (SSDs) do not. The DB is the ultimate source of truth for the application's state.

2. Master-Slave Replication

In a production environment, you never have just one database server. To handle massive read traffic and provide disaster recovery, we use replication.

Answer: An index is a separate data structure (usually a B-Tree) created from a specific column (like `email`) in a table.

Without an index, the DB must perform a Full Table Scan, reading every single row to find a matching email (O(N) time). With an index, the DB traverses the B-Tree and finds the exact disk location of the row in logarithmic time (O(log N)). The downside is that every INSERT/UPDATE is now slower, because the DB must update both the table and the B-Tree.

Answer: In a Master-Slave setup, replication is usually asynchronous. The Master commits a write, immediately responds "Success" to the user, and *then* sends the new data to the Slaves.

Replication Lag is the delay (usually milliseconds, but under heavy load, can be seconds) before the Slaves receive the update. If a user updates their profile picture on the Master, and immediately refreshes the page, their browser queries the Slave. Because of the lag, the Slave returns the old picture, confusing the user. This is a trade-off of Eventual Consistency.
