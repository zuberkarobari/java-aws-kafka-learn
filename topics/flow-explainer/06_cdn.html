6. Content Delivery Network

Why fetch an image from a server 10,000 miles away when you can fetch it from a server 10 miles away? Let's understand CDNs.

1. Definition

Simple English

Imagine ordering a book from a publisher in New York to your home in London. It takes days. If the publisher kept a copy in a local London warehouse, you'd get it instantly. A CDN is that local warehouse for website files.

Technical

A geographically distributed network of proxy servers and their data centers (Edge Locations). It caches static assets (HTML, CSS, JS, Images, Videos) closer to the end user to minimize latency and origin server load.

Purpose

To overcome the physical limits of the speed of light over fiber optic cables. It drastically improves website load times, reduces bandwidth costs, and provides global availability.

2. Internal Working: Cache Miss vs Hit

When a user requests logo.png, DNS routes them to the nearest CDN Edge Location instead of the main server.

Scenario A: Cache Miss
User (Tokyo)
1. Request `logo.png`
Edge (Tokyo) Doesn't have it!
2. Forward to Origin
Origin (New York)
*Edge fetches from Origin, caches it locally, and sends to User. (Slow)*
Scenario B: Cache Hit
User 2 (Tokyo)
1. Request `logo.png`
← 2. Return from Cache
Edge (Tokyo) I have it!
*Origin server in NY is never contacted. Response is instant. (Fast)*

3. Real-World Examples

Netflix Open Connect

How Netflix streams 4K video without crashing the internet.

ISP Embedded Appliances

Netflix accounts for up to 15% of all global internet traffic. If everyone streamed directly from AWS US-East, the ocean cables would melt. Instead, Netflix gives specialized hardware boxes (Open Connect Appliances) directly to your local ISP (like Comcast or Airtel). When you hit play on Stranger Things, the video streams from a box located in your own city, completely bypassing the public internet backbone.

For example, a GET request for a product catalog (`/api/products`) can be cached at the Edge for 5 minutes (`Cache-Control: max-age=300`).
