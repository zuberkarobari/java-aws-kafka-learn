Java Basics

Java is one of the most widely used programming languages in the world. It powers everything from Android apps and enterprise back-ends to embedded systems and scientific tools. This topic gives you a solid foundation — understanding what Java is, how it runs, and how to write your first programs with correct structure, data types, and user input.

What Is Java?

Java is a high-level, object-oriented, class-based programming language designed by James Gosling at Sun Microsystems (now Oracle) and released in 1995. Its most famous promise is "Write Once, Run Anywhere" (WORA) — a Java program compiled on one platform can run on any other platform without recompilation, as long as a Java runtime is installed.

The Java Platform: JDK, JRE, and JVM

Three acronyms appear constantly in Java documentation. Here is exactly what each one means and how they relate to each other.

JVM — Java Virtual Machine

The JVM is an abstract computing machine that executes Java bytecode. When you compile a .java source file the compiler produces a .class file containing platform- neutral bytecode. The JVM on the target machine translates that bytecode into native machine instructions at runtime.

  • The JVM is platform-specific — there is a different JVM implementation for Windows, macOS, Linux, etc.
  • Your compiled bytecode is platform-neutral — the same .class file runs on every JVM.
  • The JVM also manages memory automatically through garbage collection.

JRE — Java Runtime Environment

The JRE bundles the JVM together with the standard Java class libraries (like java.lang, java.util, java.io, etc.) that your programs depend on at runtime. If someone just wants to run a Java application — not develop one — installing the JRE is sufficient.

JDK — Java Development Kit

The JDK is the full developer toolkit. It includes everything in the JRE plus the Java compiler (javac), the documentation generator (javadoc), the archiver (jar), debuggers, and other development tools. If you are writing Java code, you must install the JDK.

A simple way to remember the relationship:

// Containment relationship (not real Java — just illustration)
JDK  ⊇  JRE  ⊇  JVM

Modern JDK distributions (JDK 9+) ship with a built-in JRE, so you only need to install the JDK. Popular distributions include Oracle JDK, OpenJDK, Eclipse Temurin, and Amazon Corretto.

Hello, World!

Every Java program lives inside a class. The entry point of any Java application is the main method. Let's look at the simplest possible program: 

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Running the Program

Save the file as HelloWorld.java (the filename must match the class name exactly, including case). Then from the terminal:

// Step 1 — compile the source file into bytecode
javac HelloWorld.java     // produces HelloWorld.class

// Step 2 — run the bytecode on the JVM
java HelloWorld           // prints: Hello, World!

Line-by-Line Explanation

  • public class HelloWorld — declares a public class named HelloWorld. Every Java file must contain exactly one public class whose name matches the filename.
  • public static void main(String[] args) — the JVM looks for this exact method signature to start execution.
    • public — accessible from outside the class (the JVM needs this).
    • static — belongs to the class itself, not an instance; the JVM can call it without creating an object.
    • void — the method returns nothing.
    • String[] args — an array of command-line arguments passed to the program.
  • System.out.println("Hello, World!") — prints the string followed by a newline. System is a class in java.lang; out is a static field of type PrintStream; println is a method on that stream.

Java Program Structure

A well-organised Java source file follows this top-down order:

// 1. Package declaration (optional but recommended)
package com.example.basics;

// 2. Import statements
import java.util.Scanner;

// 3. Class declaration
public class ProgramStructure {

    // 4. Fields (class-level variables)
    static int counter = 0;

    // 5. The main method — entry point
    public static void main(String[] args) {

        // 6. Local variables
        String greeting = "Hello, Java!";

        // 7. Statements
        System.out.println(greeting);
        counter++;
        System.out.println("Counter: " + counter);
    }
}

Key Structural Rules

  • Java is case-sensitive — Main and main are different identifiers.
  • Every statement ends with a semicolon (;).
  • Code blocks are delimited by curly braces ({ }).
  • The public class name must exactly match the .java filename.
  • Java uses 4-space indentation by convention (though 2 spaces is also common in some style guides).

Data Types

Java is a statically typed language — every variable must have a declared type that is known at compile time. Java provides two categories of types: primitive and reference types.

Primitive Data Types

Java has exactly eight primitive types. They are not objects; they hold their values directly in memory, making them fast and memory-efficient.

Integer Types

byte  b = 100;          // 8-bit  signed integer: -128 to 127
short s = 30_000;       // 16-bit signed integer: -32,768 to 32,767
int   i = 2_000_000;    // 32-bit signed integer: -2^31 to 2^31-1  (most common)
long  l = 9_000_000_000L; // 64-bit signed integer: -2^63 to 2^63-1
                           // Note the 'L' suffix — required for long literals

Underscores in numeric literals (2_000_000) are allowed from Java 7 onwards and improve readability — they have no effect on the value.

Floating-Point Types

float  f = 3.14f;       // 32-bit IEEE 754 float  — note the 'f' suffix
double d = 3.14159265;  // 64-bit IEEE 754 double — default for decimal literals

Prefer double over float for most calculations. float has only ~7 significant decimal digits of precision, while double has ~15-16.

Character Type

char letter = 'A';      // 16-bit Unicode character (single quotes)
char digit  = '7';
char symbol = '\u00A9'; // Unicode escape for © (copyright sign)

A char holds a single Unicode code unit. Use single quotes for char literals and double quotes for String literals.

Boolean Type

boolean isJavaFun    = true;
boolean isComplicated = false;

A boolean can only hold true or false. It is not interchangeable with integers (unlike in C/C++).

Quick Reference — All 8 Primitive Types

// Type        | Size    | Default | Range / Notes
// ------------|---------|---------|-----------------------------
// byte        | 8-bit   | 0       | -128 to 127
// short       | 16-bit  | 0       | -32,768 to 32,767
// int         | 32-bit  | 0       | -2,147,483,648 to 2,147,483,647
// long        | 64-bit  | 0L      | -9.2×10^18 to 9.2×10^18
// float       | 32-bit  | 0.0f    | ~7 decimal digits of precision
// double      | 64-bit  | 0.0d    | ~15-16 decimal digits of precision
// char        | 16-bit  | '\u0000'| Single Unicode character
// boolean     | 1-bit*  | false   | true or false only
//   * JVM-dependent actual size

Variables

A variable is a named container that holds a value of a specific type. In Java you must declare a variable before you can use it.

Declaration and Initialisation

// Declaration only — variable holds the default value (0, false, null, etc.)
int age;

// Declaration + initialisation in one line
int score = 95;

// Multiple variables of the same type
int x = 10, y = 20, z = 30;

// Assigning a value after declaration
age = 25;
System.out.println("Age: " + age);      // Age: 25
System.out.println("Score: " + score);  // Score: 95

Naming Conventions

Java follows camelCase for variable and method names. Following these conventions makes your code readable to every other Java developer: 

// ✅ Good — camelCase, descriptive names
int studentAge = 20;
double accountBalance = 1500.75;
boolean isLoggedIn = false;
String firstName = "Alice";

// ❌ Bad — hard to read, not idiomatic
int a = 20;
double AB = 1500.75;
boolean logged_in = false;   // underscore style is not Java convention
String FIRSTNAME = "Alice";  // all-caps is reserved for constants

Naming Rules (enforced by the compiler)

  • Must start with a letter, $, or _ (but starting with $ or _ is discouraged).
  • After the first character, digits are also allowed.
  • Cannot be a Java reserved keyword (int, class, for, etc.).
  • Case-sensitive — count and Count are different variables.

Local vs Instance vs Class Variables

public class VariableScopes {

    // Instance variable — each object has its own copy
    String name = "Default";

    // Class (static) variable — shared across all instances
    static int objectCount = 0;

    public static void main(String[] args) {
        // Local variable — exists only inside this method block
        int localValue = 42;
        System.out.println(localValue);   // 42
        System.out.println(objectCount);  // 0
    }
}

Constants with the final Keyword

A constant is a variable whose value cannot change after it has been assigned. Declare constants using the final keyword. By convention, constant names are written in UPPER_SNAKE_CASE. 

public class Constants {

    // Class-level constant — public, static, and final
    public static final double PI           = 3.141592653589793;
    public static final int    MAX_STUDENTS = 30;
    public static final String APP_NAME     = "JavaLearn";

    public static void main(String[] args) {
        final int DAYS_IN_WEEK = 7;  // local constant

        System.out.println("PI = " + PI);
        System.out.println("Max students: " + MAX_STUDENTS);
        System.out.println("Days in a week: " + DAYS_IN_WEEK);

        // ❌ This would cause a compile-time error:
        // PI = 3.0;  // cannot assign a value to final variable PI
    }
}

Why Use Constants?

  • Readability — MAX_STUDENTS is far clearer than a magic number 30 scattered through the code.
  • Maintainability — change the value in one place and it updates everywhere.
  • Safety — the compiler prevents accidental reassignment.

Type Casting

Type casting converts a value from one data type to another. Java supports two kinds: implicit (widening) and explicit (narrowing).

Implicit Casting — Widening Conversion

Java automatically converts a smaller type to a larger type because no data can be lost. This happens silently — no special syntax required. 

// Widening order: byte → short → int → long → float → double
int    myInt    = 9;
double myDouble = myInt;     // int automatically widened to double

System.out.println(myInt);    // 9
System.out.println(myDouble); // 9.0

// More examples
byte  b = 42;
short sh = b;    // byte → short
int   i  = sh;   // short → int
long  l  = i;    // int → long
float fv = l;    // long → float
double dv = fv;  // float → double

Explicit Casting — Narrowing Conversion

When converting a larger type to a smaller one, data may be lost (truncation). Java requires you to write an explicit cast to acknowledge the risk. 

double price = 9.99;
int    dollars = (int) price;  // explicit cast: drops the decimal part

System.out.println(price);    // 9.99
System.out.println(dollars);  // 9  ← fractional part is truncated, NOT rounded

// Another example — narrowing can overflow
int  bigNumber = 300;
byte smallByte  = (byte) bigNumber; // 300 doesn't fit in a byte
System.out.println(smallByte);      // 44  (300 mod 256 = 44, then signed)

Casting Between char and int

// char to int — gives the Unicode code point
char letter = 'A';
int  code   = letter;
System.out.println(code);   // 65  (Unicode value of 'A')

// int to char — gives the character at that code point
int  num  = 66;
char ch   = (char) num;
System.out.println(ch);     // B

String to Number Conversions

Strings are not primitives, so casting doesn't apply. Use the wrapper class parse methods instead: 

String numStr = "42";
int    parsed = Integer.parseInt(numStr);   // String → int
double dParsed = Double.parseDouble("3.14"); // String → double

System.out.println(parsed  + 8);   // 50  (arithmetic, not string concat)
System.out.println(dParsed + 1.0); // 4.140000000000001

// Number → String
String s1 = String.valueOf(parsed);   // "42"
String s2 = Integer.toString(100);    // "100"
String s3 = "" + 99;                  // "99" — quick but less explicit

Basic Input with the Scanner Class

Java's standard library provides java.util.Scanner to read input from the keyboard (or any other input stream). You must import it before using it.

Reading Different Types

import java.util.Scanner;

public class UserInput {
    public static void main(String[] args) {
        // Create a Scanner attached to standard input (keyboard)
        Scanner scanner = new Scanner(System.in);

        System.out.print("Enter your name: ");
        String name = scanner.nextLine();   // reads a full line of text

        System.out.print("Enter your age: ");
        int age = scanner.nextInt();        // reads the next integer token

        System.out.print("Enter your GPA: ");
        double gpa = scanner.nextDouble();  // reads the next decimal token

        System.out.println("---");
        System.out.println("Name : " + name);
        System.out.println("Age  : " + age);
        System.out.println("GPA  : " + gpa);

        // Always close the scanner when you're done
        scanner.close();
    }
}

Common Scanner Methods

Scanner sc = new Scanner(System.in);

String  line    = sc.nextLine();    // reads the rest of the line (includes spaces)
String  word    = sc.next();        // reads the next whitespace-delimited token
int     integer = sc.nextInt();     // reads the next int
long    bigNum  = sc.nextLong();    // reads the next long
double  decimal = sc.nextDouble();  // reads the next double
float   flt     = sc.nextFloat();   // reads the next float
boolean bool    = sc.nextBoolean(); // reads "true" or "false" (case-insensitive)

The nextLine() Pitfall

A very common beginner mistake: calling nextLine() immediately after nextInt() (or any other nextXxx() method) skips the next line unexpectedly. This happens because nextInt() does not consume the trailing newline character. 

import java.util.Scanner;

public class ScannerPitfall {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter your age: ");
        int age = sc.nextInt();

        // ⚠️ Consume the leftover newline before reading a string
        sc.nextLine();

        System.out.print("Enter your full name: ");
        String name = sc.nextLine();   // now works correctly

        System.out.println(age + " — " + name);
        sc.close();
    }
}

A Complete Interactive Example

import java.util.Scanner;

public class SimpleCalculator {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.println("=== Simple Calculator ===");
        System.out.print("Enter first number : ");
        double a = sc.nextDouble();

        System.out.print("Enter second number: ");
        double b = sc.nextDouble();

        double sum        = a + b;
        double difference = a - b;
        double product    = a * b;
        double quotient   = (b != 0) ? a / b : 0; // guard against division by zero

        System.out.println("Sum        : " + sum);
        System.out.println("Difference : " + difference);
        System.out.println("Product    : " + product);
        System.out.println("Quotient   : " + quotient);

        sc.close();
    }
}

Summary

Here is a quick recap of everything covered in this topic:

  • JVM executes bytecode on any platform; JRE = JVM + libraries; JDK = JRE + compiler and dev tools.
  • Every Java program needs a public class with a public static void main(String[] args) method as the entry point.
  • Java has 8 primitive types: byte, short, int, long, float, double, char, and boolean.
  • Variables follow camelCase naming; constants follow UPPER_SNAKE_CASE and use the final keyword.
  • Widening (implicit) casting is automatic; narrowing (explicit) casting requires a cast operator and may lose data.
  • Scanner from java.util reads user input — always call sc.nextLine() to flush the newline after nextInt() or similar methods.
