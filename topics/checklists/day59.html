Kafka — Rapid Recall

Question

Answer — cover and say from memory

acks=1 risk?

Leader acknowledges, crashes before replication → message lost. Fix: acks=all + min.insync.replicas=2.

Idempotent producer mechanism?

PID (producer ID) + sequence number per partition. Broker silently drops duplicates with same PID+seq.

Cooperative-sticky vs eager rebalance?

Cooperative: only partitions that need to move are revoked. Other consumers keep processing. Eager: all stop, all revoke, full reassign.

Static membership purpose?

Auto-commit risk?

Commits before processing completes. If crash mid-processing, message is lost. Always use manual commit for at-least-once.

Exactly-once requires?

Idempotent producer + Kafka transactions + isolation.level=read_committed on consumers.

DLT — what triggers immediate DLT?

Permanent errors: invalid schema, business rule violation, non-retryable exceptions in @RetryableTopic exclude list.

Consumer lag per partition — diagnose?

One partition lagging: poison pill or slow consumer thread. All lagging: producer outpacing consumers, need more consumers or faster processing.

AWS and Security — Rapid Recall

Question

Answer — cover and say from memory

EC2 instance type for memory-intensive JVM?

r-series (memory optimised). Not t-series (CPU credits depleted under sustained load).

S3 Standard-IA vs Glacier?

Standard-IA: immediate retrieval, infrequent access, per-GB retrieval fee. Glacier: hours retrieval, compliance archives, much cheaper storage.

IAM Deny vs Allow?

Deny always wins. An explicit Deny overrides any Allow, including from another policy.

EC2 IAM credentials — where come from?

Instance metadata service (IMDS). Role attached via instance profile. Credentials rotated automatically. No access keys in config.

JWT: 4 claims to validate beyond signature?

exp (not expired), iss (correct issuer), aud (correct audience), alg (whitelist — never accept 'none').

RS256 advantage over HS256?

Private key only on auth server. Services hold public key only. Compromised service cannot forge tokens.

Refresh token — where stored in browser?

HttpOnly, Secure, SameSite=Strict cookie. Not localStorage (XSS risk).

BOLA — two prevention layers?

Query layer: WHERE owner_id = currentUser. Security layer: @PostAuthorize ownership check. Test: user A cannot access user B's resource.

Mass assignment fix?

Dedicated request DTO with only client-settable fields. Never use JPA entity as @RequestBody.

OAuth2 PKCE — what it prevents?

Authorization code interception on mobile/SPA. Attacker without code_verifier cannot exchange intercepted code for tokens.

REST and System Design — Rapid Recall

Question

Answer — cover and say from memory

400 vs 422 vs 409?

400: malformed/unparseable. 422: valid format, business rule fail. 409: state conflict (duplicate, version mismatch).

301 vs 302 in URL shortener?

302 is default. Every redirect contacts server → click counted. 301 is browser-cached → click count stops.

Cursor pagination advantage?

WHERE id > last_seen — hits index. OFFSET N — scans N rows. Cursor also stable: new inserts don't shift pages.

Idempotency key for POST?

UUID in header. Store result in Redis 24h. On duplicate key: return original result, do NOT process again.

RFC 7807 required fields?

type (URI), title (human summary), status (HTTP code), detail (this occurrence), instance (request URI).

Cache-aside on write?

Update database → delete Redis key. Next read loads fresh from DB and repopulates cache.

Consistent hashing — add node impact?

Only 1/N of keys move (between new node and its predecessor). vs hash%N where nearly all keys move.

Saga choreography risk?

Outbox atomicity guarantee?

Write event row in same DB transaction as business data. DB transaction = either both commit or neither.

CQRS read model consistency?

Eventually consistent. Write commits, then event fires, then projection updates. Small window of stale reads.
