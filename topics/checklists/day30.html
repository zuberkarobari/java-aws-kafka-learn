SQL window function to rank rows with no gaps in ranking?

What does EXPLAIN ANALYZE actually run in PostgreSQL?

It executes the query and shows actual row counts and actual timing alongside the estimated plan.

Index type for LIKE 'pay%' queries in PostgreSQL?

B-tree index. The prefix anchor allows index range scan. LIKE '%pay' cannot use a B-tree — it requires a GIN index with pg_trgm.

What does a Seq Scan in EXPLAIN mean?

Full table scan — the planner decided the index was not worth using, usually because estimated selectivity is low or the table is small.

Two Sum — unsorted array. Pattern and complexity?

HashMap complement search. O(n) time, O(n) space.

Longest substring without repeating characters — pattern?

Variable sliding window with a HashMap storing last seen index. O(n) time.

Subarray sum equals k — why not sliding window?

Array can have negative numbers. Shrinking the window does not guarantee sum decreases. Use prefix sum + HashMap instead.

Reverse a linked list — what three pointers do you need?

Detect cycle in linked list — pattern?

Fast-slow pointers. Fast moves 2 steps, slow moves 1. If they meet, there is a cycle.

Inorder traversal visits BST nodes in what order?

Sorted ascending order. Left → Root → Right.

Lowest Common Ancestor in a BST — decision at each node?

If both target values are less than the node, go left. If both are greater, go right. Otherwise the current node is the LCA.

BFS vs DFS — which uses a queue and which uses a stack?

BFS uses a Queue (level-by-level). DFS uses a Stack (implicit call stack in recursion, or explicit with Deque).

Topological sort — what does it require and what algorithm?

Directed Acyclic Graph (DAG). Kahn's algorithm uses in-degree counts + BFS queue. DFS-based post-order reversal also works.

DP memoisation vs tabulation — one-line difference?

Memoisation is top-down recursion with a cache. Tabulation is bottom-up iteration filling a table.

Coin change — what DP recurrence?

dp[amount] = min(dp[amount], dp[amount - coin] + 1) for each coin. Base: dp = 0.

Longest Common Subsequence — recurrence?

If chars match: dp[i][j] = dp[i-1][j-1] + 1. If not: dp[i][j] = max(dp[i-1][j], dp[i][j-1]).

QuickSort worst case — when does it occur?

Already sorted (or reverse sorted) array with last-element pivot. Every partition is maximally unbalanced. Fix: randomise the pivot.

Binary search: find first occurrence — what changes vs standard?

When target is found at mid, record it and set hi = mid - 1 to keep searching left. Do not return immediately.

Search in rotated sorted array — key insight?

At any midpoint, at least one half is always normally sorted. Identify which half, then check if target is in its range.

Binary search on answer space — when does it apply?

When the feasibility function is monotonic — once an answer is feasible, all larger (or smaller) answers are also feasible.

Day 23 — Advanced SQL (Questions 1–10)

1. ROW_NUMBER vs RANK vs DENSE_RANK — what is the difference?

ROW_NUMBER: unique sequential number, no ties. RANK: ties get same number, next rank skips. DENSE_RANK: ties get same number, next rank does not skip.

2. What does PARTITION BY do in a window function?

Resets the window function calculation for each distinct group, similar to GROUP BY but without collapsing rows.

3. How do you find the second highest salary per department?

Use DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) and filter where rank = 2.

4. What is a covering index?

An index that contains all columns needed by a query — the query is answered entirely from the index without touching the main table (index-only scan).

5. INNER JOIN vs LEFT JOIN — one concrete difference?

INNER JOIN returns only matching rows. LEFT JOIN returns all rows from the left table and NULLs for unmatched right rows.

6. What does a Hash Join do and when does PostgreSQL choose it?

Builds a hash table from the smaller table, then probes it for each row from the larger table. Chosen when tables are large and there is no useful index.

7. What is a CTE and when would you use it over a subquery?

Common Table Expression — a named temporary result set defined with WITH. Use it when the same subquery is referenced more than once, or for readability in recursive queries.

8. READ COMMITTED vs REPEATABLE READ — what extra guarantee does Repeatable Read give?

Repeatable Read prevents non-repeatable reads — the same SELECT returns the same rows within the same transaction, even if another transaction commits updates between the reads.

9. HikariCP connectionTimeout vs idleTimeout — one sentence each?

connectionTimeout: how long a client waits to get a connection from the pool before throwing. idleTimeout: how long an idle connection stays in the pool before being closed.

10. What does Flyway do and what is its baseline concept?

Flyway manages database schema migrations using versioned SQL scripts. Baseline marks an existing database as already at a specific version so Flyway does not re-run old migrations.

Day 24 — Arrays and Strings (Questions 11–20)

11. Two Sum unsorted — O(n) approach?

HashMap: for each element x, check if target - x is in the map. If yes, return both indices. Otherwise add x to the map. O(n) time, O(n) space.

12. Container With Most Water — why do you move the shorter pointer?

The area is limited by the shorter side. Moving the taller side inward can only decrease or keep the width the same while the height ceiling stays. Only moving the shorter side gives a chance of finding a taller boundary.

13. Fixed sliding window — what two operations happen on each slide?

Add the new right element to the window aggregate. Remove the element that just exited on the left (the element at index i - k).

14. Prefix sum: to get sum from index i to j, what is the formula?

prefix[j+1] - prefix[i], where prefix[i] = sum of nums[0..i-1] and prefix = 0.

15. Group Anagrams — what is used as the HashMap key?

The sorted version of each string. All anagrams sort to the same string, so they map to the same key.

16. Dutch National Flag — three pointer invariants?

Everything left of low is 0. Everything between low and mid is 1. Everything right of high is 2. mid scans forward; low and high mark the boundaries.

17. Longest palindromic substring — approach and complexity?

Expand from centre: for each index try both odd (centre at i) and even (centre between i and i+1) expansions. O(n^2) time, O(1) space.

18. Subarray sum equals k — why initialise the HashMap with {0: 1}?

19. Fast-slow pointer: where does slow point when fast reaches the end of a list?

The middle node. For even length lists, it points to the second of the two middle nodes.

20. Check if one string is a rotation of another — one-line insight?

If s2 is a rotation of s1, then s2 is always a substring of s1 + s1. Check with contains().

Day 25 — Linked Lists, Stacks and Queues (Questions 21–30)

21. Reverse a linked list iteratively — three variables needed?

prev (null), curr (head), next (temp). Loop: save next, flip curr.next to prev, advance prev and curr.

22. Merge two sorted linked lists — base case?

If either list is null, return the other. Then recursively compare heads and attach the smaller one.

23. Remove nth node from end of list — one-pass approach?

Two pointers n+1 apart. Advance both until the fast pointer reaches the end. The slow pointer is then at the node before the one to remove.

24. LRU Cache — which two data structures implement O(1) get and put?

HashMap (key to node) + Doubly Linked List (nodes in access order). HashMap gives O(1) lookup. The list maintains recency order.

25. Monotonic stack — what does it maintain?

A stack where elements are always in increasing (or decreasing) order. Elements that violate the order are popped before the new element is pushed.

26. Next Greater Element — what data structure and what do you do when you pop?

Monotonic decreasing stack. When a new element is greater than the stack top, pop the top and record the new element as its next greater element.

27. Valid Parentheses — what goes on the stack?

Opening brackets. When a closing bracket is seen, pop from the stack and check if it matches. Return false if the stack is empty or the top does not match.

28. Sliding window maximum — what data structure gives O(n) total?

Monotonic decreasing deque. The front always holds the maximum. Elements outside the window are removed from the front. Smaller elements displaced by the new element are removed from the back.

29. Implement a queue using two stacks — how does dequeue work?

If the output stack is empty, pop all elements from the input stack onto the output stack. Then pop from the output stack. Each element moves at most once, giving amortised O(1).

30. Min Stack — how do you get O(1) getMin()?

Maintain a second stack that stores the current minimum at each level. Push the current min alongside every push. Pop from both stacks on pop.

Day 26 — Trees and BST (Questions 31–40)

31. Inorder traversal of a BST gives what output?

Nodes in sorted ascending order.

32. Maximum depth of binary tree — recursive definition?

1 + max(maxDepth(left), maxDepth(right)). Base case: null node returns 0.

33. Balanced binary tree — definition of balanced?

A tree where for every node, the height difference between its left and right subtrees is at most 1.

34. Validate BST — what additional info must be passed in recursion?

A min bound and a max bound. For the root: min = -inf, max = +inf. Left child: max = parent value. Right child: min = parent value.

35. Level-order traversal — what data structure?

Queue. Add root, then for each node dequeued add its left and right children to the queue.

36. Lowest Common Ancestor in a BST — decision at each node?

Both values less than node: go left. Both greater: go right. Otherwise the current node is the LCA.

37. Path sum in binary tree — recursive approach?

At each node subtract node.val from target. If it is a leaf and target is now 0, return true. Recurse on left and right.

38. Serialize and deserialize a binary tree — approach?

Preorder traversal with null markers. Serialize writes values with null for missing nodes. Deserialize reads values and reconstructs using a queue or index pointer.

39. BST insert — where does the new node always go?

At a null leaf position. BST property guides you: if value < node go left, if value > node go right, until you reach null.

40. Diameter of binary tree — what value to compute at each node?

leftHeight + rightHeight (the longest path through this node). Track the global maximum. Return max(leftHeight, rightHeight) + 1 to the parent.

Day 27 — Graphs (Questions 41–50)

41. BFS gives what guarantee that DFS does not?

Shortest path in terms of number of edges (hops) in an unweighted graph.

42. DFS cycle detection in undirected graph — what does the visited check look for?

A neighbour that is already visited AND is not the parent of the current node. Reaching a visited non-parent node confirms a cycle.

43. Topological sort — what is Kahn's algorithm?

Compute in-degrees for all nodes. Add all nodes with in-degree 0 to a queue. Process queue: for each node, reduce neighbour in-degrees by 1 and add any that reach 0 to the queue. If all nodes processed: valid topological order. If not: cycle exists.

44. Number of islands — DFS approach?

For each unvisited land cell, run DFS to mark all connected land cells as visited and increment the island count by 1.

45. Dijkstra's algorithm — what data structure and what does it find?

Min-heap (priority queue). Finds the shortest path from one source node to all other nodes in a weighted graph with non-negative edge weights.

46. What is a bipartite graph and how do you check for it?

A graph whose nodes can be coloured with two colours such that no two adjacent nodes share the same colour. Check with BFS/DFS: try to 2-colour the graph. If a conflict is found, it is not bipartite.

47. Union-Find: what are the two operations and their complexity with path compression and union by rank?

find(x): returns root of x's set. union(x, y): merges the sets containing x and y. With both optimisations: nearly O(1) amortised — O(alpha(n)) where alpha is the inverse Ackermann function.

48. Clone a graph — what extra structure do you need?

A HashMap from original node to cloned node. Prevents infinite loops on cycles and avoids creating duplicate clones for nodes visited multiple times.

49. Word Ladder — what graph problem is this and what algorithm?

Shortest path in an unweighted graph where nodes are words and edges connect words differing by one letter. BFS gives the shortest transformation sequence.

50. When does Dijkstra fail and what is the alternative?

Dijkstra fails with negative edge weights. Use Bellman-Ford, which relaxes all edges V-1 times and can detect negative cycles.

Days 28 + 29 — DP, Sorting and Binary Search (Questions 51–60)

51. Optimal substructure — definition?

A problem has optimal substructure if an optimal solution to the whole problem contains optimal solutions to its subproblems. This is a prerequisite for DP.

52. Fibonacci memoisation — what is the space complexity?

O(n) for the memo array (or HashMap). O(n) call stack depth in recursive version. Tabulation bottom-up also uses O(n) space, or O(1) if you only keep the last two values.

53. 0/1 Knapsack — recurrence?

dp[i][w] = max(dp[i-1][w], dp[i-1][w - weight[i]] + value[i]) if weight[i] <= w, else dp[i-1][w]. i = item index, w = current capacity.

54. Coin change: what is the base case and what does dp equal?

dp = 0 (zero coins needed to make amount 0). dp[i] = infinity for all i > 0, before filling.

55. Longest Increasing Subsequence — two approaches and their complexities?

DP with dp[i] = length of LIS ending at index i: O(n^2). Patience sorting with binary search to find insertion point: O(n log n).

56. MergeSort is stable but QuickSort is not — what does stable mean?

Equal elements preserve their original relative order after sorting. If two elements are equal, the one that appeared first in the input still appears first in the output.

57. Binary search loop condition: when do you use lo < hi instead of lo <= hi?

Use lo < hi for the insertion point (lower bound) variant, where hi = mid (not mid - 1) when the condition is satisfied. The loop terminates when lo == hi, which is the answer.

58. Find peak element — why does binary search work on an unsorted array?

Moving toward the higher neighbour (nums[mid+1] > nums[mid]) always leads to a peak — either nums[mid+1] is a peak or the array rises further and eventually peaks. This monotonicity is enough.

59. Top K frequent elements — approach and complexity?

Count frequencies with a HashMap, then use a min-heap of size k. Push each (element, frequency) pair; pop when size exceeds k. O(n log k).

Rules:

Given a string s and a dictionary of strings wordDict, return true if s can be segmented into a space-separated sequence of dictionary words.

Example: s = 'leetcode', wordDict = ['leet','code'] → true. s = 'applepenapple', wordDict = ['apple','pen'] → true.

Your solution:

Pattern: _______________ Time complexity: _______________ Space: _______________

Reference solution:

This is a 1D DP problem. dp[i] = true if s[0..i-1] can be segmented. For each index i, check all substrings s[j..i-1] where dp[j] is true and s[j..i-1] is in the dictionary. Base case: dp = true (empty string). Use a HashSet for O(1) word lookup. Time O(n^2 * L) where L is max word length for the substring check. Space O(n). Key insight: you do not need to find the segmentation — only whether it exists.

Problem 2 — Merge K Sorted Lists

Given an array of k linked lists, each sorted in ascending order, merge all of them into one sorted linked list and return it.

Example: lists = [[1,4,5],[1,3,4],[2,6]] → [1,1,2,3,4,4,5,6].

Your solution:

Pattern: _______________ Time complexity: _______________ Space: _______________

Reference solution:

Use a min-heap (PriorityQueue) of ListNode objects, ordered by node value. Add the head of each non-null list to the heap. Then repeatedly: poll the smallest node, append it to the result, and if it has a next node push that next node into the heap. Time O(N log k) where N is total nodes and k is the number of lists — each of the N nodes is pushed and popped from the heap once, each operation costs O(log k). Space O(k) for the heap. The naive approach of merging two lists at a time is O(kN) — the heap approach is significantly better when k is large.

Q1: A table has millions of rows. A query on it is slow. Walk me through how you would diagnose and fix it. [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

The first thing I do is run EXPLAIN ANALYZE on the slow query and look at the output top-down. I am looking for sequential scans on large tables where I would expect an index scan, high actual row counts that differ significantly from estimated row counts (which indicates stale statistics), and any nested loop joins where the inner side is being executed many times. If I see a sequential scan where I expect a filter condition to be selective, I check whether an index exists on that column, whether the data types match between the filter value and the column (a type mismatch silently disables index use), and whether the query planner's row estimate is far off from the actual count. If statistics are stale, running ANALYZE on the table updates them. If an index is missing, I add one. If the query has a LIKE '%value' pattern, a standard B-tree index will not help — that requires a GIN index with pg_trgm. If it is a complex multi-table join, I check whether the join order makes sense and whether adding a covering index would allow an index-only scan. For our Deloitte CloudWatch log lookup work, we had a query that was slow because it was joining on a varchar column without an index. Adding the index dropped the query from several seconds to under 10ms.

Q2: Given an array of integers, find two numbers that sum to a target. Then follow up: what if the array is sorted? What if you need all pairs? What if the array has duplicates? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

For the base case with an unsorted array, I use a HashMap. For each element I check whether target minus that element is already in the map. If yes, I found the pair. If no, I add the element to the map. This is O(n) time and O(n) space. If the array is sorted, I switch to two pointers — one at each end. If the sum is too small, advance the left pointer. If too large, move the right pointer left. O(n) time and O(1) space. For all pairs, I extend either approach: with the HashMap, when I find a complement I record the pair and continue instead of returning. With sorted two pointers, when I find a pair, I record it and advance both pointers while skipping duplicates on each side. For duplicates in the all-pairs case, I skip consecutive equal values after recording a pair to avoid duplicate pairs in the output. The sorted two-pointer approach handles this most cleanly because equal elements are adjacent and easy to skip.

Q3: When would you use BFS instead of DFS for a graph problem? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

BFS is the right choice when you need the shortest path in an unweighted graph — BFS naturally explores all nodes at distance 1 before any at distance 2, so the first time it reaches a target node is guaranteed to be via the shortest path. Word Ladder, shortest path between two nodes, and minimum jumps problems are all BFS. DFS is better when you need to explore all possible paths (permutations, all paths from source to target), when you are looking for any reachable destination rather than the nearest one, or when you need topological ordering. DFS uses less memory than BFS in practice for deep graphs because the recursion stack only holds one path at a time, while BFS needs to hold the entire frontier. The concrete signal: if the problem says shortest or minimum steps, reach for BFS. If it says all paths, exists a path, or topology, reach for DFS.

Q4: Explain dynamic programming. How do you recognise a DP problem and how do you approach solving one? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

DP is a technique for solving problems that have two properties: overlapping subproblems — the same subproblem is computed multiple times in a naive recursive solution — and optimal substructure — the optimal solution to the whole problem is built from optimal solutions to subproblems. I recognise a DP problem by looking for these signals: the problem asks for maximum, minimum, or count of ways to achieve something; the decision at each step affects future options; and there is a natural recurrence where the answer at position i depends on answers at earlier positions. My approach is always: first write a brute force recursive solution and verify it is correct. Then identify the repeated subproblems — these are the parameters of the recursion that change. Add memoisation to cache results by those parameters. Finally, if needed, convert to a bottom-up tabulation to eliminate the call stack overhead. The most important step is getting the recurrence right — everything else is mechanical.

Q5: You need to find the kth largest element in a stream of incoming integers. How do you design the data structure? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

I maintain a min-heap of exactly k elements. For each incoming integer, I add it to the heap. If the heap grows beyond k elements, I pop the minimum — removing the smallest element and keeping only the k largest seen so far. At any point, the minimum of the heap is the kth largest element overall. Adding an element is O(log k) since the heap never exceeds size k. The kth largest query is O(1) — just peek at the heap root. This is the KthLargest class pattern in LeetCode. The reason a min-heap works here: by always keeping the k largest elements and maintaining their minimum at the top, the top of the heap is always exactly the kth largest. If a new element is larger than the kth largest, it displaces the current minimum and the kth largest updates accordingly.

Q6: Given a sorted array that has been rotated at an unknown pivot, find a target value. Walk me through your solution. [Hard] ▼
💡 Ideal Answer (Amazon/GS Standard)

The naive approach is a linear scan — O(n). But since the array was originally sorted, I can do better. The key insight is that even in a rotated array, when I split at any midpoint, at least one of the two halves is always normally sorted. I compare nums[lo] with nums[mid] to identify which half is sorted. If nums[lo] is at most nums[mid], the left half from lo to mid is sorted in normal order. I then check whether the target falls within that range — between nums[lo] and nums[mid]. If yes, I search the left half. If no, the target must be in the right half. If nums[lo] is greater than nums[mid], the right half is the sorted one, and I do the same range check. Either way I eliminate half the search space at each step, giving O(log n) total. The boundary conditions are where it gets subtle — for the left half check I use target >= nums[lo] and target < nums[mid], making sure not to accidentally include or exclude the boundaries.

Pattern / Topic

The single most important thing to remember

SQL window functions

DENSE_RANK has no gaps. RANK has gaps. ROW_NUMBER has no ties. PARTITION BY resets per group.

Index design

Composite index: column order matters. Put the equality column first, range column second. Covering index avoids table lookup.

Two pointers (sorted)

Sorted array + pair/triplet → opposite ends. If sum too low, advance left. If too high, advance right.

Sliding window

Fixed size: add right, remove left on each slide. Variable: expand right, shrink left when constraint violated.

Prefix sum + HashMap

prefixSum[j] - prefixSum[i] = k → look for prefixSum[j] - k in the map. Initialise {0:1}.

HashMap patterns

Complement search (two sum), frequency count (anagrams), index tracking (longest subarray).

Linked list reversal

Three pointers: prev=null, curr=head, next=temp. Loop: save next, flip curr.next, advance both.

Fast-slow pointer

Find middle, detect cycle, happy number. Fast moves 2, slow moves 1.

Monotonic stack

Next greater element: decreasing stack — pop when new element is greater, record the new element as answer for popped.

BST traversal

Inorder = sorted. Validate BST: pass min/max bounds down the recursion.

Tree DP

Diameter: leftH + rightH at each node, return max(leftH, rightH) + 1 upward.

BFS vs DFS

BFS = shortest hops, level-order, uses Queue. DFS = all paths, topological sort, uses Stack or recursion.

Union-Find

find() with path compression. union() with rank. Detects cycles in undirected graphs in nearly O(1).

DP approach

Brute force recursion → identify repeated subproblems → memoise → tabulate if needed.

Binary search

For first/last occurrence: find target then keep searching in that direction. For answer space: define feasibility function, binary search bounds.

QuickSort vs MergeSort

QuickSort: faster in practice, in-place, not stable, O(n^2) worst case. MergeSort: stable, guaranteed O(n log n), O(n) space.
