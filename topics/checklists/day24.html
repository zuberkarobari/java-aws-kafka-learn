Section 2 — Two Pointers

Pattern 1: Opposite Ends — Sorted Array

When to use:

The array is sorted (or you can sort it). You need to find a pair, triplet, or validate a property that benefits from closing in from both ends. If the sum is too large, move the right pointer left. If too small, move the left pointer right. Each step eliminates an element — total O(n).

// TEMPLATE: Two Sum on sorted array
// Time: O(n)  Space: O(1)
public int[] twoSumSorted(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int sum = nums[left] + nums[right];
        if      (sum == target) return new int[]{left, right};
        else if (sum < target)  left++;   // need bigger sum
        else                    right--;  // need smaller sum
    }
    return new int[]{};
}
// PROBLEM: Three Sum — find all unique triplets that sum to zero
// Approach: sort, then fix one element and run two-pointer on the rest
// Time: O(n^2)  Space: O(1) excluding output
public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);
    List<List<Integer>> result = new ArrayList<>();
    for (int i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] == nums[i-1]) continue; // skip duplicate fixed element
        int left = i + 1, right = nums.length - 1;
        while (left < right) {
            int sum = nums[i] + nums[left] + nums[right];
            if (sum == 0) {
                result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                while (left < right && nums[left]  == nums[left+1])  left++;  // skip dupes
                while (left < right && nums[right] == nums[right-1]) right--; // skip dupes
                left++; right--;
            } else if (sum < 0) left++;
            else                right--;
        }
    }
    return result;
}
// PROBLEM: Container With Most Water
// Array heights — find two lines that form a container holding max water
// Key insight: always move the shorter pointer inward
// Time: O(n)  Space: O(1)
public int maxArea(int[] height) {
    int left = 0, right = height.length - 1, max = 0;
    while (left < right) {
        int area = Math.min(height[left], height[right]) * (right - left);
        max = Math.max(max, area);
        if (height[left] < height[right]) left++;   // shorter side limits area — move it
        else                              right--;
    }
    return max;
}
// PROBLEM: Valid Palindrome (ignoring non-alphanumeric)
// Time: O(n)  Space: O(1)
public boolean isPalindrome(String s) {
    int left = 0, right = s.length() - 1;
    while (left < right) {
        while (left < right && !Character.isLetterOrDigit(s.charAt(left)))  left++;
        while (left < right && !Character.isLetterOrDigit(s.charAt(right))) right--;
        if (Character.toLowerCase(s.charAt(left)) !=
            Character.toLowerCase(s.charAt(right))) return false;
        left++; right--;
    }
    return true;
}

Pattern 2: Same Direction — Read/Write Pointer

When to use:

You need to modify an array in-place, removing or partitioning elements without extra space. One pointer reads every element; the other pointer only advances when a valid element is written. The write pointer is always at or behind the read pointer.

// TEMPLATE: Remove duplicates from sorted array in-place
// write pointer moves only when a new unique value is found
// Time: O(n)  Space: O(1)
public int removeDuplicates(int[] nums) {
    if (nums.length == 0) return 0;
    int write = 1;                          // first element is always unique
    for (int read = 1; read < nums.length; read++) {
        if (nums[read] != nums[write - 1]) { // new unique value
            nums[write++] = nums[read];
        }
    }
    return write; // new length
}
// PROBLEM: Move all zeros to end, preserve relative order of non-zeros
// Time: O(n)  Space: O(1)
public void moveZeroes(int[] nums) {
    int write = 0;
    for (int read = 0; read < nums.length; read++) {
        if (nums[read] != 0) nums[write++] = nums[read]; // write non-zeros forward
    }
    while (write < nums.length) nums[write++] = 0; // fill rest with zeros
}
// PROBLEM: Dutch National Flag — sort array of 0s, 1s, 2s in one pass
// Three-way partition: low = boundary of 0s, high = boundary of 2s
// Time: O(n)  Space: O(1)
public void sortColors(int[] nums) {
    int low = 0, mid = 0, high = nums.length - 1;
    while (mid <= high) {
        if      (nums[mid] == 0) swap(nums, low++, mid++); // 0 goes to front
        else if (nums[mid] == 1) mid++;                    // 1 stays in place
        else                     swap(nums, mid, high--);  // 2 goes to end, recheck mid
    }
}
private void swap(int[] a, int i, int j) {
    int tmp = a[i]; a[i] = a[j]; a[j] = tmp;
}

Pattern 3: Fast-Slow Pointers

// PROBLEM: Find middle of a linked list
// Fast moves 2 steps, slow moves 1. When fast reaches end, slow is at middle.
// Time: O(n)  Space: O(1)
public ListNode middleNode(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow; // for even-length list, returns second middle
}
// PROBLEM: Detect cycle in linked list (Floyd's algorithm)
// Time: O(n)  Space: O(1)
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true; // they meet inside the cycle
    }
    return false;
}
// PROBLEM: Happy number — does repeated sum-of-squares reach 1?
// Key insight: if it is NOT a happy number, it will enter a cycle
// Use fast-slow on the sequence of sums to detect the cycle
// Time: O(log n)  Space: O(1)
public boolean isHappy(int n) {
    int slow = n, fast = sumOfSquares(n);
    while (fast != 1 && slow != fast) {
        slow = sumOfSquares(slow);
        fast = sumOfSquares(sumOfSquares(fast));
    }
    return fast == 1;
}
private int sumOfSquares(int n) {
    int sum = 0;
    while (n > 0) { int d = n % 10; sum += d * d; n /= 10; }
    return sum;
}
Q1: When do you use two pointers instead of nested loops? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

Two pointers replace the inner loop of an O(n^2) brute force when the data has a property that lets you move directionally rather than exhaustively. The classic case: sorted array with a pair sum target. The brute force checks every pair. With two pointers you start at opposite ends and each comparison either finds the answer or eliminates one element from consideration — you can prove no valid answer involves that element. The key question to ask yourself: if the current pair does not work, do I know which direction to move to get closer to the answer? If yes, two pointers works. If not, you may need a HashMap instead. A good signal in the problem statement: 'sorted array', 'find pair that sums to', 'find triplet', 'in-place without extra space', or 'remove duplicates from sorted array'. For problems where the array is not sorted and sorting would lose information, two pointers usually does not apply — reach for a HashMap.

Section 3 — Sliding Window

Fixed-Size Window

When to use:

The problem asks for max/min/average of exactly k consecutive elements. Maintain a running aggregate. On each step, add the new right element and subtract the element that just left the left side. No need to recompute the whole window — O(1) per step.

// TEMPLATE: Maximum sum of k consecutive elements
// Time: O(n)  Space: O(1)
public int maxSumSubarray(int[] nums, int k) {
    int windowSum = 0;
    for (int i = 0; i < k; i++) windowSum += nums[i]; // seed first window
    int maxSum = windowSum;
    for (int i = k; i < nums.length; i++) {
        windowSum += nums[i] - nums[i - k]; // slide: add right, remove left
        maxSum = Math.max(maxSum, windowSum);
    }
    return maxSum;
}
// PROBLEM: Find all anagrams of pattern p in string s
// Fixed window of size p.length(), compare frequency maps
// Time: O(n)  Space: O(1) — alphabet is bounded
public List<Integer> findAnagrams(String s, String p) {
    List<Integer> result = new ArrayList<>();
    if (s.length() < p.length()) return result;
    int[] pCount = new int[26];
    int[] wCount = new int[26];
    for (char c : p.toCharArray()) pCount[c - 'a']++;
    int k = p.length();
    for (int i = 0; i < s.length(); i++) {
        wCount[s.charAt(i) - 'a']++;           // add right char
        if (i >= k) wCount[s.charAt(i-k) - 'a']--; // remove left char
        if (Arrays.equals(pCount, wCount)) result.add(i - k + 1);
    }
    return result;
}

Variable-Size Window

When to use:

Find the longest or shortest subarray/substring satisfying a constraint. Expand the right pointer freely. When the constraint is violated, shrink from the left until the constraint holds again. The window always represents the current valid state.

// TEMPLATE: Longest substring without repeating characters
// Time: O(n)  Space: O(1) — bounded by charset size
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> lastSeen = new HashMap<>();
    int left = 0, maxLen = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        // If char was seen and is inside the current window, shrink from left
        if (lastSeen.containsKey(c) && lastSeen.get(c) >= left) {
            left = lastSeen.get(c) + 1;
        }
        lastSeen.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
// PROBLEM: Minimum window substring — smallest window in s containing all chars of t
// Time: O(n + m)  Space: O(m)
public String minWindow(String s, String t) {
    Map<Character, Integer> need  = new HashMap<>();
    Map<Character, Integer> have  = new HashMap<>();
    for (char c : t.toCharArray()) need.merge(c, 1, Integer::sum);
    int formed = 0, required = need.size();
    int left = 0, minLen = Integer.MAX_VALUE, minLeft = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        have.merge(c, 1, Integer::sum);
        // Check if this char's count in window meets the requirement
        if (need.containsKey(c) && have.get(c).equals(need.get(c))) formed++;
        // Shrink from left while the window is valid
        while (formed == required) {
            if (right - left + 1 < minLen) {
                minLen = right - left + 1;
                minLeft = left;
            }
            char lc = s.charAt(left);
            have.merge(lc, -1, Integer::sum);
            if (need.containsKey(lc) && have.get(lc) < need.get(lc)) formed--;
            left++;
        }
    }
    return minLen == Integer.MAX_VALUE ? "" : s.substring(minLeft, minLeft + minLen);
}
// PROBLEM: Longest subarray with at most k distinct characters
// Time: O(n)  Space: O(k)
public int longestSubarrayKDistinct(String s, int k) {
    Map<Character, Integer> freq = new HashMap<>();
    int left = 0, maxLen = 0;
    for (int right = 0; right < s.length(); right++) {
        freq.merge(s.charAt(right), 1, Integer::sum);
        // Shrink until we have at most k distinct chars
        while (freq.size() > k) {
            char lc = s.charAt(left++);
            freq.merge(lc, -1, Integer::sum);
            if (freq.get(lc) == 0) freq.remove(lc);
        }
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
Q2: How do you decide between a fixed and variable sliding window? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

Fixed window is when the problem specifies an exact size — max sum of k elements, all anagrams of a string with known length. The window moves exactly one step at a time, adding the new right element and removing the old left element. Variable window is when the problem says 'longest', 'shortest', or 'minimum' with a constraint you have to satisfy — longest substring without repeating characters, smallest window containing all required characters. You expand the right pointer freely and only shrink the left pointer when the constraint is violated. The shrink condition is what makes each variable window problem slightly different — for distinct character problems it is when the distinct count exceeds k; for sum problems it is when the sum exceeds the target. Once you identify that it is a variable window problem, the structure is always: expand right, check if valid, if not shrink from left until valid, update the answer at each step.

Section 4 — Prefix Sum

The Core Idea

What it gives you:

Build a prefix sum array once in O(n). Then answer any range sum query in O(1): sum(i, j) = prefix[j+1] - prefix[i]. The key insight: instead of summing elements between i and j each time, you store the running total and subtract. This is one of those techniques where once you see it, you see it everywhere.

// TEMPLATE: Prefix sum array
// prefix[i] = sum of nums[0..i-1]  (prefix[0] = 0 sentinel)
int[] prefix = new int[nums.length + 1];
for (int i = 0; i < nums.length; i++) {
    prefix[i+1] = prefix[i] + nums[i];
}
// Range sum query: sum from index i to j (inclusive)
int rangeSum = prefix[j+1] - prefix[i];
// PROBLEM: Subarray Sum Equals K — count subarrays whose sum equals k
// Brute force is O(n^2). With prefix sum + HashMap it is O(n).
// Key insight: sum(i..j) = prefix[j] - prefix[i-1]
// We want prefix[j] - prefix[i-1] == k → prefix[i-1] == prefix[j] - k
// For each j, check how many previous prefix sums equal prefix[j] - k
// Time: O(n)  Space: O(n)
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0, 1); // empty subarray has prefix sum 0
    int count = 0, runningSum = 0;
    for (int num : nums) {
        runningSum += num;
        // How many times has (runningSum - k) appeared as a prefix sum?
        count += prefixCount.getOrDefault(runningSum - k, 0);
        prefixCount.merge(runningSum, 1, Integer::sum);
    }
    return count;
}
// PROBLEM: Continuous Subarray Sum divisible by k
// Key: if prefix[i] % k == prefix[j] % k, then sum(i+1..j) is divisible by k
// Time: O(n)  Space: O(k)
public boolean checkSubarraySum(int[] nums, int k) {
    Map<Integer, Integer> remainderIndex = new HashMap<>();
    remainderIndex.put(0, -1); // remainder 0 seen before index 0
    int sum = 0;
    for (int i = 0; i < nums.length; i++) {
        sum = (sum + nums[i]) % k;
        if (remainderIndex.containsKey(sum)) {
            if (i - remainderIndex.get(sum) >= 2) return true; // subarray length >= 2
        } else {
            remainderIndex.put(sum, i); // only record first occurrence
        }
    }
    return false;
}
// PROBLEM: Product of array except self — no division, O(1) extra space
// Left pass: result[i] = product of all elements to the LEFT of i
// Right pass: multiply in product of all elements to the RIGHT of i
// Time: O(n)  Space: O(1) output only
public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    result[0] = 1;
    for (int i = 1; i < n; i++)
        result[i] = result[i-1] * nums[i-1];    // prefix product
    int right = 1;
    for (int i = n - 1; i >= 0; i--) {
        result[i] *= right;                     // multiply in suffix product
        right *= nums[i];
    }
    return result;
}
Q3: What is the prefix sum technique and when does it apply? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

Prefix sum is pre-computation that trades O(n) space for O(1) query time on range sum problems. You build an array where prefix[i] holds the sum of all elements from index 0 to i-1. Then any range sum from i to j is prefix[j+1] minus prefix[i] — a single subtraction. It applies whenever a problem asks you to find subarrays or ranges based on their sum. The classic pattern is subarray sum equals k — for each ending index, you want to know how many starting indices give a subarray that sums to k. You express this as prefix[j] minus prefix[i] equals k, which means you are looking for prefix[i] equals prefix[j] minus k. Keep a running prefix sum and a HashMap of how many times each prefix sum has been seen. For each new element, look up how many times the needed prefix sum has appeared. This brings what would be an O(n^2) problem down to O(n). The trick that trips people up: initialise the HashMap with the entry 0 → 1 to handle subarrays that start at index 0.

Section 5 — HashMap Patterns

Three HashMap Patterns for Arrays and Strings

// ── PATTERN 1: Frequency Count ──
// Use when: comparing character/element frequencies, finding duplicates, grouping
// PROBLEM: Group Anagrams
// Key insight: anagrams have the same sorted string — use that as the HashMap key
// Time: O(n * k log k)  k = average string length  Space: O(n * k)
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    for (String s : strs) {
        char[] chars = s.toCharArray();
        Arrays.sort(chars);
        String key = new String(chars); // sorted string is the canonical anagram form
        map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }
    return new ArrayList<>(map.values());
}
// PROBLEM: First non-repeating character in a string
// Time: O(n)  Space: O(1) — only 26 lowercase letters
public int firstUniqChar(String s) {
    int[] freq = new int[26];
    for (char c : s.toCharArray()) freq[c - 'a']++;
    for (int i = 0; i < s.length(); i++)
        if (freq[s.charAt(i) - 'a'] == 1) return i;
    return -1;
}
// ── PATTERN 2: Complement Search ──
// Use when: find two elements that combine to meet a target
// PROBLEM: Two Sum (unsorted array)
// Time: O(n)  Space: O(n)
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> seen = new HashMap<>(); // value → index
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (seen.containsKey(complement))
            return new int[]{seen.get(complement), i};
        seen.put(nums[i], i);
    }
    return new int[]{};
}
// ── PATTERN 3: Index Tracking ──
// Use when: need to know WHEN something was last seen, not just IF
// PROBLEM: Longest subarray with equal number of 0s and 1s
// Key: replace 0 with -1. Now the problem is longest subarray with sum = 0.
// Record first occurrence of each prefix sum. When we see it again,
// the elements between the two occurrences cancel out (sum = 0).
// Time: O(n)  Space: O(n)
public int findMaxLength(int[] nums) {
    Map<Integer, Integer> firstSeen = new HashMap<>();
    firstSeen.put(0, -1); // prefix sum 0 seen before index 0
    int sum = 0, maxLen = 0;
    for (int i = 0; i < nums.length; i++) {
        sum += (nums[i] == 0 ? -1 : 1); // treat 0 as -1
        if (firstSeen.containsKey(sum))
            maxLen = Math.max(maxLen, i - firstSeen.get(sum));
        else
            firstSeen.put(sum, i); // only store first occurrence
    }
    return maxLen;
}
Q4: When do you use a HashMap over sorting for an array problem? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

Sorting costs O(n log n) and is fine when you need a permanent ordering, like in three-sum where you fix one element and run two pointers on the sorted rest. Use a HashMap when you need O(n) and the problem requires remembering something about each element — its value, its index, its frequency, or its complement. Two Sum is the clearest example: sorting and two pointers would work but loses the original indices. A HashMap gives O(1) lookup and preserves index information. The general rule: if the question asks for something like 'have I seen x before', 'what is the index of y', or 'how many times does z appear', a HashMap is almost always the right choice. If the question involves comparing across sorted positions, two pointers on a sorted array is often cleaner. Knowing which one applies comes down to whether you need to preserve original indices and whether the extra O(n) space is acceptable.

Section 6 — String Manipulation Patterns

Common String Techniques

// PROBLEM: Reverse words in a string
// Input: "  hello   world  "  Output: "world hello"
// Time: O(n)  Space: O(n)
public String reverseWords(String s) {
    String[] words = s.trim().split("\\s+"); // split on any whitespace
    int left = 0, right = words.length - 1;
    while (left < right) {
        String tmp = words[left]; words[left] = words[right]; words[right] = tmp;
        left++; right--;
    }
    return String.join(" ", words);
}
// PROBLEM: Longest palindromic substring (expand from centre)
// For each character, try expanding outwards for odd and even length palindromes
// Time: O(n^2)  Space: O(1)
public String longestPalindrome(String s) {
    int start = 0, maxLen = 0;
    for (int i = 0; i < s.length(); i++) {
        // Odd length: centre is s[i]
        int len1 = expand(s, i, i);
        // Even length: centre is between s[i] and s[i+1]
        int len2 = expand(s, i, i + 1);
        int len  = Math.max(len1, len2);
        if (len > maxLen) {
            maxLen = len;
            start  = i - (len - 1) / 2;
        }
    }
    return s.substring(start, start + maxLen);
}
private int expand(String s, int left, int right) {
    while (left >= 0 && right < s.length() && s.charAt(left) == s.charAt(right)) {
        left--; right++;
    }
    return right - left - 1;
}
// PROBLEM: Check if one string is a rotation of another
// Key insight: if s2 is a rotation of s1, then s2 is always a substring of s1+s1
// Time: O(n)  Space: O(n)
public boolean isRotation(String s1, String s2) {
    if (s1.length() != s2.length()) return false;
    return (s1 + s1).contains(s2);
}
// PROBLEM: Roman to Integer
// Walk right to left. If current value < next value, subtract it; otherwise add.
// Time: O(n)  Space: O(1)
public int romanToInt(String s) {
    Map<Character, Integer> val = Map.of(
        'I',1, 'V',5, 'X',10, 'L',50, 'C',100, 'D',500, 'M',1000);
    int result = 0;
    for (int i = 0; i < s.length(); i++) {
        int curr = val.get(s.charAt(i));
        int next = (i+1 < s.length()) ? val.get(s.charAt(i+1)) : 0;
        result += (curr < next) ? -curr : curr;
    }
    return result;
}
// PROBLEM: Valid parentheses — {}, [], ()
// Time: O(n)  Space: O(n)
public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(' || c == '{' || c == '[') { stack.push(c); continue; }
        if (stack.isEmpty()) return false;
        char top = stack.pop();
        if ((c == ')' && top != '(') ||
            (c == '}' && top != '{') ||
            (c == ']' && top != '[')) return false;
    }
    return stack.isEmpty();
}

Hard problem — combines HashMap with careful math:

Interview Q&A — 10 Questions

How to use:

These are the approach questions. Before saying a single line of code, say the pattern name, the time complexity, and why the naive approach is worse. That framing is what interviewers listen for.

Q5: Given a sorted array and a target, find two numbers that sum to the target. What is your approach? [Easy] ▼
💡 Ideal Answer (Amazon/GS Standard)

This is a classic two-pointer problem on a sorted array. I place one pointer at the start and one at the end. If the sum of the two elements equals the target, I return them. If the sum is less than the target, I move the left pointer right to increase the sum. If greater, I move the right pointer left to decrease it. Each comparison either finds the answer or eliminates one element, so the whole thing runs in O(n) with O(1) extra space. The reason this works is the sorted order — in a sorted array, moving the left pointer right always increases the sum and moving the right pointer left always decreases it, so every move makes progress. If the array were unsorted and I could not sort it, I would use a HashMap for O(n) time at the cost of O(n) space.

Q6: Find the longest substring without repeating characters. Walk me through your approach. [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

This is a variable sliding window problem. I maintain a window defined by a left and right pointer, and a HashMap that stores the last seen index of each character. I expand the right pointer across the string. When I encounter a character that is already in the window — meaning its last seen index is at or after the left pointer — I move the left pointer to one position past where that character was last seen. This shrinks the window to remove the duplicate. At each step, I update the maximum length as right minus left plus one. Time is O(n) and space is O(1) if we consider the charset bounded at 128 ASCII characters. The key detail that trips people up: you do not just move left forward by one when you see a duplicate. You jump left to lastSeen[c] plus one, because the duplicate could be several positions back and moving one at a time would be O(n^2).

Q7: Count subarrays that sum to k. The array may have negative numbers. [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

Because the array has negative numbers, a sliding window will not work — shrinking the window does not guarantee the sum decreases. Sorting is also not applicable here. The right approach is prefix sum combined with a HashMap. For each position j, I maintain the running sum prefix[j]. The sum of a subarray from i to j is prefix[j] minus prefix[i-1]. I want this to equal k, which means I am looking for prefix[i-1] equals prefix[j] minus k. For every new element, I look up how many times that value has appeared as a previous prefix sum — each occurrence corresponds to one valid subarray ending at j. I use a HashMap to count prefix sum occurrences and initialise it with 0 → 1 to handle subarrays starting at index 0. Time is O(n), space is O(n). The initialisation of 0 → 1 is the detail most people miss on first attempt.

Q8: Find all anagrams of a pattern in a string. What is your approach? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

This is a fixed-size sliding window problem. The window size is the length of the pattern. I count the character frequencies of the pattern in an int array of size 26. Then I slide a window of the same size across the string, maintaining a frequency count of the current window. At each position I add the new right character and remove the character that just left the left side. If the two frequency arrays match, the current window is an anagram. Comparing two int arrays is O(1) since the size is fixed at 26. Total time is O(n) where n is the length of the string, space is O(1). The clean approach uses Arrays.equals() for the comparison.

Q9: What is the two-pointer technique and when does it not work? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

Two pointers is a technique for traversing an array or string with two indices that move toward each other or in the same direction, eliminating the need for a nested loop. It works when the data has an ordering property that lets you make a directional decision at each step — typically when the array is sorted, or when you can derive that moving one pointer in a specific direction always makes progress toward the answer. It does not work when there is no such ordering property and you cannot eliminate candidates after each comparison. For example, finding a pair that sums to a target in an unsorted array cannot be solved with two pointers without sorting first — and sorting might not be permitted if the problem requires returning original indices. Two pointers also does not work for problems involving non-contiguous elements or when the window is not defined by two adjacent boundaries.

Q10: How do you find the longest consecutive sequence in an unsorted array in O(n)? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

The naive approach is to sort the array and scan for consecutive runs — O(n log n). To get O(n), I put all elements into a HashSet for O(1) lookup. Then I iterate over the array. For each element, I only start counting a sequence if element minus one is NOT in the set — meaning this element is the start of a potential sequence. From there I count upward as long as element plus one is in the set. This ensures each element is visited at most twice across all sequences, giving O(n) total. The key insight is the check for element minus one. Without it, every element would try to start a sequence and you would revisit elements multiple times.

Q11: You have an array of 0s and 1s. Find the maximum length subarray with equal numbers of 0s and 1s. [Hard] ▼
💡 Ideal Answer (Amazon/GS Standard)

I replace every 0 with -1. Now equal numbers of 0s and 1s means the subarray sum is zero. This transforms the problem into finding the longest subarray with sum zero, which is a classic prefix sum problem. I maintain a running sum and a HashMap that stores the first index at which each prefix sum was seen. If the same prefix sum appears again at index j, then the subarray between the first occurrence and j has sum zero — meaning equal 0s and 1s. I track the maximum length across all such pairs. Initialise the map with 0 → -1 to handle subarrays starting at index 0. Time is O(n), space is O(n). The transformation from 0s and 1s to +1 and -1 is the insight that unlocks the solution.

Q12: Explain the Dutch National Flag problem and your solution. [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

The problem is to sort an array containing only three distinct values — in the original problem 0, 1, and 2 — in a single pass without using a counting sort. The three-pointer approach maintains three regions: everything to the left of a low pointer is 0s, everything to the right of a high pointer is 2s, and the middle is 1s. A mid pointer scans from left to right. If nums[mid] is 0, swap it with nums[low] and advance both low and mid. If it is 1, just advance mid. If it is 2, swap with nums[high] and advance high backward, but do not advance mid because the swapped element at mid has not been examined yet. The loop terminates when mid passes high. Time is O(n), space is O(1), and the array is sorted in a single pass.

Q13: What is the expand-from-centre approach for palindrome problems? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

Instead of checking every possible substring to see if it is a palindrome — O(n^3) — I treat each position as a potential centre of a palindrome and expand outward as long as the characters match. For odd-length palindromes the centre is a single character. For even-length palindromes the centre is between two characters. By checking both cases at each position and tracking the longest expansion found, I get O(n^2) time with O(1) space. The expand helper function returns the length of the longest palindrome centred at two given indices. The implementation detail: for the even-length case, start with left equal to i and right equal to i plus one.

Q14: When should you sort an array versus using a HashMap to solve a problem? [Medium] ▼
💡 Ideal Answer (Amazon/GS Standard)

Sort the array when: the problem inherently involves ordering (find the kth largest, find a pair in sorted order, binary search), when sorting does not lose information you need (you do not need original indices), and when O(n log n) is acceptable. Use a HashMap when: you need O(n) and cannot afford O(n log n), when you need to preserve original indices (two sum returning indices), when you need fast O(1) lookups for complements or frequencies, or when the array has no natural ordering that helps. A rough heuristic: if the problem says 'return indices', use a HashMap. If it says 'return values' and the array can be sorted, sort and consider two pointers or binary search. If the problem involves frequencies or has negative numbers making windowing approaches break, use a HashMap with prefix sums.
