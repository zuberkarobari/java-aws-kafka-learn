STAR Stories for Common Interview Questions

Story 1 — Debugging a Production Performance Issue

When asked:

'Tell me about a time you diagnosed a production performance problem' or 'describe a challenging technical problem you solved'.

// Your STAR story (write your own version using these prompts)
// Situation: What was the system? What was the symptom?
// e.g. The CloudWatch log lookup service at Deloitte was showing p99 latency
//      climbing to 8 seconds for log group queries. Users were timing out.
// Task: What was your specific responsibility?
// e.g. I owned the backend service. I needed to diagnose and fix the issue
//      without taking the service offline.
// Action: What specific steps did you take?
// e.g. I added distributed tracing (OpenTelemetry spans) to identify which
//      downstream call was slow. Found the compliance service was timing out
//      at 7 seconds on 30% of requests. Added Resilience4j CircuitBreaker with
//      2-second TimeLimiter. Added Bulkhead to isolate compliance calls from
//      other requests. Added Micrometer metrics to CloudWatch.
// Result: What was the measurable outcome?
// e.g. p99 latency dropped from 8s to 340ms. Circuit breaker opens when
//      compliance service is degraded, returning cached response in <10ms.
//      Zero timeout errors in the 6 weeks since deployment.
