Architecture
The Doorman of Microservices
The API Gateway is the most visible part of your architecture. It handles cross-cutting concerns: Auth, Rate Limiting, and Routing. A fat gateway is untestable and a shared liability. Keep it thin.
✅ What belongs here
• SSL/TLS Termination
• JWT Validation
• Rate Limiting
• Centralized CORS
❌ What DOES NOT
• Database Calls
• Business Logic (if/else)
• Heavy Data Aggregation
• Complex Orchestration
Stage 1
Reactive Routing (Netty)
Spring Cloud Gateway uses non-blocking Netty. It queries the Service Registry (Eureka/K8s) to load-balance traffic, avoiding hardcoded IPs via the lb:// protocol.

Stage 2
The Filter Chain
GlobalFilters apply to ALL routes (e.g., Logging, JWT validation). GatewayFilters apply to specific routes (e.g., Rate Limiting per service). They execute sequentially.
Global: RequestLoggingFilter
↓
Global: JwtAuthenticationFilter
↓
Route: RequestRateLimiter
↓
Route: CircuitBreaker
JAVAGlobalFilter Interface
public class RequestLoggingFilter implements GlobalFilter { @Override public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) { log.info("Path requested: " + exchange.getRequest().getPath()); return chain.filter(exchange); } }
Stage 3
JWT Mutation (Zero Trust)
The gateway validates the token signature via JWKS. On success, it extracts claims and injects trusted internal headers (X-User-Id). Crucial: It MUST strip external forged headers first!
Incoming Request

Auth: Bearer eyJ...
X-User-Id: 9999 (Forged!)
Host: api.app.com
➡️
Mutated For Backend

Auth: [Stripped]
X-User-Id: 9821 (Trusted)
X-Roles: ADMIN
JAVAMutating the Request
// Inject validated claims as trusted headers ServerHttpRequest mutated = request.mutate() .header("X-User-Id", claims.getSubject()) .header("X-Roles", claims.getRoles()) .build(); return chain.filter(exchange.mutate() .request(mutated).build());
Stage 4
Resilience4j Circuit Breaker
Prevents cascading failures. If the downstream service times out, the circuit OPENS and returns a Fallback immediately to save Gateway threads.
Gateway
user-service
Fallback Triggered: 503 Service Unavailable
YAMLRoute Config
filters: - name: CircuitBreaker args: name: user-cb fallbackUri: forward:/fallback/users
Stage 5
Global CORS & Metrics
Handle CORS centrally at the gateway so downstream services don't duplicate config. Spring Gateway natively exports Micrometer metrics for Grafana/Prometheus.
YAMLGlobal Configuration
spring: cloud: gateway: globalcors: cors-configurations: '[/**]': allowed-origins: https://frontend.app.com allowed-methods: "*" allow-credentials: true
