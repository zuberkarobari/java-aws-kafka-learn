Request Mapping

Mastering @RequestMapping, composed shortcuts like @GetMapping, and the mechanics of URL routing.

1. Definition

Simple English

Imagine an office building's receptionist. When you say "I have a package for IT," they direct you to Room 101. Request mappings are the "receptionists" of Spring Boot—they look at the incoming HTTP request and direct it to the exact Java method designed to handle it.

Technical

@RequestMapping and its specialized variants (@GetMapping, @PostMapping, etc.) map web requests to specific handler classes and/or handler methods in a Controller, populating the RequestMappingHandlerMapping registry.

Purpose

To define the exact API Contract. These annotations dictate the URL path, the allowed HTTP methods, what Content-Type headers the method accepts, and what format it returns.

4. Key Points

1. Class-Level Mapping

You can place @RequestMapping("/api/users") at the class level to establish a base URI. All method-level mappings are appended to this base path.

2. Composed Annotations

@GetMapping is just a shortcut for @RequestMapping(method = RequestMethod.GET). Introduced in Spring 4.3 to reduce boilerplate.

3. Consumes & Produces

You can restrict requests. consumes = "application/json" requires the client to send JSON. produces = "application/xml" tells the client this method returns XML.

4. Ambiguous Mappings

If two methods map to the exact same URI and HTTP method, Spring throws an AmbiguousHandlerResolutionException and crashes on startup.

5. Wildcards and Ant-style Paths

Mappings support Ant-style patterns. @GetMapping("/users/*") matches one path segment (/users/123). @GetMapping("/users/**") matches zero or more recursively (/users/123/profile).

6. Code Example

import org.springframework.web.bind.annotation.*;
import org.springframework.http.ResponseEntity;

@RestController
/* 1. CLASS-LEVEL MAPPING: Pre-pends "/api/v1/products" to all methods below */
@RequestMapping("/api/v1/products")
public class ProductController {

    /* 2. COMPOSED ANNOTATION (GET) */
    @GetMapping
    public ResponseEntity<List<Product>> getProducts() {
        return ResponseEntity.ok(productList);
    }

    /* 3. PATH VARIABLE extraction */
    @GetMapping("/{id}")
    public ResponseEntity<Product> getProductById(@PathVariable("id") Long productId) {

    }

    /* 4. CONTENT NEGOTIATION (consumes/produces) */

7. Senior Developer Answer

"Request mappings define the exact API contracts of our system. I strictly enforce the use of composed annotations like @GetMapping and @PostMapping to ensure endpoints map perfectly to HTTP verbs and prevent developers from exposing broad, unconstrained endpoints. In an enterprise architecture, I use the advanced attributes of these annotations extensively. Specifically, I leverage the 'produces' and 'consumes' attributes to strictly enforce Content-Type negotiation, and the 'headers' attribute for robust API versioning (e.g., mapping X-API-VERSION=2 to a newer Java method) to maintain backwards compatibility seamlessly."
