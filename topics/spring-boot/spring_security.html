1. Definition

Simple English

It's a highly customizable bouncer for your application. It stands at the door, checks IDs to see who you are (Authentication), and checks a VIP list to see what rooms you are allowed to enter (Authorization).

Technical

A powerful and highly customizable authentication and access-control framework. It operates fundamentally via a chain of standard Servlet Filters (`FilterChainProxy`) that intercept requests before they ever reach the `DispatcherServlet`.

Purpose

To secure applications comprehensively. It provides out-of-the-box protection against common attacks like CSRF, Session Fixation, and Clickjacking, while offering deep hooks for OAuth2, OIDC, and JWT integrations.

3. Real-World Project Example

Stateless JWT API

How a React frontend talks securely to a Spring Boot backend.

1. The Login Phase

Client POSTs credentials to /auth/login. The AuthenticationManager checks the DB. If valid, we generate a cryptographically signed JSON Web Token (JWT) and return it to the client.

2. Subsequent Requests

Client requests /api/orders, passing the JWT in the Authorization: Bearer <token> header. Spring Security completely ignores session cookies.

3. Custom JWT Filter

Our custom JwtAuthFilter sits in the SecurityFilterChain. It intercepts the request, verifies the JWT signature mathematically (without hitting the database), extracts the username and roles, and manually populates the SecurityContextHolder.

4. 7 Key Points For senior-level depth

1

AuthN vs AuthZ

Authentication (AuthN) is verifying identity (Are you who you say you are?). Authorization (AuthZ) is checking permissions (Are you allowed to do this?).

2

Spring Boot 3 Changes

WebSecurityConfigurerAdapter is completely deprecated. Configuration is now component-based. You create a SecurityFilterChain bean using the fluent builder API.

3

Password Encoding

Never store plain text. Spring provides PasswordEncoder. BCryptPasswordEncoder is the standard, hashing passwords with an embedded salt to defeat rainbow table attacks.

4

Stateless Sessions

In REST APIs, you must explicitly tell Spring Security not to create HTTP Sessions (SessionCreationPolicy.STATELESS). This prevents memory leaks and CSRF vulnerabilities.

5

Method Security

Using @EnableMethodSecurity, you can add @PreAuthorize("hasRole('ADMIN')") directly above Service or Controller methods for fine-grained, AOP-based authorization.

6

CORS & CSRF

Cross-Origin Resource Sharing (CORS) must be explicitly configured if your UI and API live on different domains. CSRF (Cross-Site Request Forgery) protection should be disabled for stateless APIs but enabled for SSR (Thymeleaf).

7

OAuth2 & OIDC

Modern apps often delegate AuthN entirely to Identity Providers (Okta, Auth0, Google) using OAuth2. Spring Security provides extensive native integration for acting as an OAuth2 Resource Server.

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configurers.AbstractHttpConfigurer;
import org.springframework.security.config.http.SessionCreationPolicy;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;
import org.springframework.security.web.SecurityFilterChain;
import org.springframework.security.web.authentication.UsernamePasswordAuthenticationFilter;

@Configuration
@EnableWebSecurity
public class SecurityConfig {

    private final JwtAuthenticationFilter jwtAuthFilter;

    public SecurityConfig(JwtAuthenticationFilter jwtAuthFilter) {
        this.jwtAuthFilter = jwtAuthFilter;
    }

    /* 1. Component-based Security Configuration (Spring 6 / Boot 3) */
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        
        http
            // Disable CSRF for stateless REST APIs
            .csrf(AbstractHttpConfigurer::disable)
            
            // Disable HTTP Sessions completely (No JSESSIONID)
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            
            // Configure Authorization Rules
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/v1/auth/**").permitAll() // Login/Signup
                .requestMatchers("/api/v1/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated() // Everything else requires auth
            )
            
            // Insert our custom JWT filter BEFORE the standard auth filter
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    /* 2. Password Encoding */
    @Bean
    public PasswordEncoder passwordEncoder() {
        // BCrypt handles salt generation automatically
        return new BCryptPasswordEncoder();
    }
}

7. Senior Developer Answer

"Spring Security is fundamentally an elaborate chain of Servlet Filters coordinated by the `FilterChainProxy`. Before a request ever hits our Spring MVC layer, it passes through this chain where the `AuthenticationManager` verifies identity, and subsequent filters verify permissions.

In modern architectures, especially with the shift in Spring Boot 3, we have completely abandoned the deprecated `WebSecurityConfigurerAdapter`. Instead, we define our security rules via component-based `SecurityFilterChain` beans. Furthermore, because we build highly scalable React/Angular frontends, we exclusively use Stateless architectures. This means explicitly disabling CSRF, setting `SessionCreationPolicy.STATELESS`, and managing authentication entirely via cryptographically verified JSON Web Tokens (JWTs).

To achieve this, we inject a custom `OncePerRequestFilter` ahead of the `UsernamePasswordAuthenticationFilter` to manually validate incoming tokens and populate the `SecurityContextHolder`. For enterprise clients, we frequently shift authentication entirely off our database by configuring Spring Security to act as an OAuth2 Resource Server validating JWTs issued by Okta or Auth0."
