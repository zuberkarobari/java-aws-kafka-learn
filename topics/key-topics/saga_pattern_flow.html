Architecture
The Distributed Data Problem
In a monolith, transferring money uses one ACID database transaction. In microservices, Order, Payment, and Inventory have separate databases. You cannot lock them all across a network.
✅ Monolith (ACID)
1 DB. 1 Connection. Absolute consistency. Rollbacks are instant and mathematical.
❌ Microservices
3 DBs. 3 Connections over HTTP/TCP. If DB 3 fails, how do you rollback DB 1?
Anti-Pattern
Two-Phase Commit (2PC)
The naive approach is a distributed lock. The coordinator locks ALL databases, executes the logic, and commits. This guarantees consistency but destroys availability. If a network drops, the system freezes permanently.

Pattern 1
Choreography (Event-Driven)

Pattern 2
Orchestration (The Brain)
We introduce a central brain: The Saga Execution Coordinator (SEC). It issues explicit Commands (not events) and tracks the exact state in its own database. Perfect for complex flows with money.

The Happy Path
The Coordinator breaks the transaction into a sequence of Local Transactions. Each service commits locally, releases its lock instantly, and replies. High availability is maintained.
JAVACoordinator Logic
// Step 1 orderClient.createPendingOrder(req); state = ORDER_PENDING; // Step 2 paymentClient.charge(req.amount()); state = PAYMENT_APPROVED; // Step 3 inventoryClient.reserve(req.item()); state = COMPLETED;
Failure
The Distributed Failure
The network is hostile. Order is created. Payment is charged. But when reserving Inventory, it returns "Out of Stock". The user lost money, but gets no item. We cannot do an SQL rollback on the Payment Service!
JAVAInventory Call
try { inventoryClient.reserve(req.item()); state = COMPLETED; } catch (OutOfStockException e) { // FATAL: Payment was already committed! state = INVENTORY_FAILED; triggerCompensation(); }
Recovery
Compensating Transactions
The genius of Saga: Semantic Rollbacks. The Coordinator shifts into reverse and issues Compensation Commands. It tells Payment: 'Refund $2500'. It tells Order: 'Cancel'. Perfect financial consistency restored.
JAVACompensation Logic
private void triggerCompensation() { state = COMPENSATING; // Reverse Step 2 paymentClient.refund(req.amount()); // Reverse Step 1 orderClient.cancelOrder(req.orderId()); state = ABORTED_SAFELY; }
